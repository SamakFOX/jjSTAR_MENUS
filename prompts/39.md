# 모든 메뉴 레벨에 설명(desc) 필드 추가 및 Hover 툴팁 표시 요청

## 작업 목표

현재 메뉴 편집 화면에서 각 메뉴 항목에 마우스를 올렸을 때, 해당 메뉴에 대한 설명을 툴팁 형태로 표시하고 싶습니다.

예시는 아래와 같은 형태입니다.

```txt
학생들이 JJSTAR를 처음 이용할 때 학교생활에 필요한 정보를 얻고 학교와 친해지기 위해 사용할 수 있는 메뉴로 구성되어 있어요
```

이 설명은 모든 레벨 메뉴에 적용되어야 합니다.

```txt
Level 1
Level 2
Level 3
```

---

## 데이터 구조 수정 요청

현재 메뉴 리스트를 로딩하는 데이터 파일에 각 메뉴 항목별로 `desc` 필드를 추가해 주세요.

예상 대상 파일은 현재 메뉴 데이터를 관리하는 파일입니다.

```txt
src/data/menus.js
```

또는 현재 Level 리스트를 로딩하는 실제 데이터 파일에 적용해 주세요.

---

## desc 필드 추가 방식

모든 메뉴 객체에 `desc` 필드를 추가해 주세요.

초기값은 빈 문자열로 두면 됩니다.

```js
desc: ""
```

제가 직접 나중에 설명 문구를 채워 넣을 예정이므로,
자동으로 임의 설명을 생성하지 말아 주세요.

---

## 데이터 예시

### Level 1 예시

```js
{
  id: "star-support",
  title: "star 학생지원",
  level: 1,
  desc: "",
  children: [
    // ...
  ]
}
```

### Level 2 예시

```js
{
  id: "star-manage",
  title: "star 관리",
  level: 2,
  desc: "",
  children: [
    // ...
  ]
}
```

### Level 3 예시

```js
{
  id: "course-guide",
  title: "수강편람",
  level: 3,
  desc: "",
}
```

현재 데이터 구조에서 `id`, `code`, `title`, `children`, `level` 등의 필드명이 다르다면 기존 구조는 그대로 유지하고,
각 메뉴 객체에 `desc: ""`만 추가해 주세요.

---

## 적용 범위

`desc` 필드는 모든 메뉴 레벨에 추가되어야 합니다.

```txt
Level 1 메뉴 전체
Level 2 메뉴 전체
Level 3 메뉴 전체
```

하위 메뉴가 있는 메뉴에도 추가하고,
하위 메뉴가 없는 메뉴에도 추가해 주세요.

## Hover 툴팁 표시 요청

메뉴 항목에 마우스를 올렸을 때 해당 메뉴의 `desc` 값이 있으면 툴팁을 표시해 주세요.

### 표시 조건

```txt
menu.desc가 빈 문자열이 아니면 hover 시 툴팁 표시
menu.desc가 빈 문자열이면 툴팁 표시하지 않음
```

즉, 아래 값이면 표시하지 않습니다.

```js
desc: ""
```

아래처럼 값이 들어 있으면 표시합니다.

```js
desc: "수강편람 메뉴로 이동하기"
```

---

## 툴팁 적용 대상

툴팁은 다음 화면/모드에서 표시되면 좋겠습니다.

```txt
리스트 편집 모드
화면에서 편집 / 비주얼모드
미리보기 화면
```

단, 우선 적용이 복잡하다면 아래 순서로 적용해 주세요.

```txt
1순위: 리스트 편집 모드
2순위: 비주얼모드
3순위: 미리보기 화면
```

---

## 툴팁 위치

툴팁은 hover한 메뉴 항목 근처에 표시해 주세요.

기본 위치는 메뉴 항목의 위쪽 또는 오른쪽 위를 권장합니다.

예시:

```txt
[수강편람]  ← hover
  ↑
[수강편람 메뉴로 이동하기]
```

또는 공간이 부족하면 오른쪽으로 표시해도 됩니다.

```txt
[수강편람] [수강편람 메뉴로 이동하기]
```

---

## 툴팁 스타일

현재 UI와 어울리도록 어두운 배경의 작은 툴팁으로 표시해 주세요.

예시 스타일:

```css
.menu-desc-tooltip {
  background: #f0f9ff;
  font-size: 12px;
  line-height: 1.4;
  padding: 8px 10px;
  border-radius: 8px;
  box-shadow: 0 8px 20px rgba(15, 23, 42, 0.18);
  max-width: 280px;
  white-space: normal;
  word-break: keep-all;
  z-index: 1000;
}
```

짧은 설명은 한 줄로 보이고, 긴 설명은 자연스럽게 줄바꿈되면 됩니다.

## 툴팁 내용 처리

툴팁에는 `menu.desc` 값을 그대로 표시해 주세요.

```jsx
{menu.desc}
```

HTML로 해석하지 말고 일반 텍스트로 표시해 주세요.

보안상 아래처럼 HTML 삽입은 사용하지 말아 주세요.

```jsx
dangerouslySetInnerHTML
```

## Hover 동작

마우스를 올리면 툴팁이 표시되고,
마우스가 벗어나면 사라져야 합니다.

```txt
mouseenter → tooltip 표시
mouseleave → tooltip 숨김
```

가능하면 약간의 delay를 줄 수 있습니다.

```txt
표시 delay: 150ms 정도
숨김 delay: 0~100ms 정도
```

delay가 없어도 괜찮지만, 빠르게 지나갈 때 툴팁이 너무 많이 깜빡이지 않으면 좋겠습니다.

---

## 드래그 중 툴팁 표시 방지

메뉴를 드래그하는 중에는 툴팁이 표시되지 않도록 해 주세요.

드래그 중 툴팁이 뜨면 드래그앤드랍 조작을 방해할 수 있습니다.

```txt
isDragging === true → tooltip 표시 안 함
```

또는 dnd-kit / hello-pangea-dnd에서 제공하는 dragging 상태가 있다면 해당 값을 사용해 주세요.

---

## 빈 desc 처리

`desc`가 비어 있으면 툴팁을 렌더링하지 마세요.

```js
if (!menu.desc?.trim()) {
  return null;
}
```

공백만 있는 경우도 표시하지 않아야 합니다.

---

## 컴포넌트 분리 제안

가능하면 공통 툴팁 컴포넌트를 만들어 재사용해 주세요.

예시:

```jsx
function MenuDescTooltip({ desc, children, disabled }) {
  const shouldShowTooltip = desc && desc.trim().length > 0 && !disabled;

  return (
    <div className="relative inline-flex">
      {children}

      {shouldShowTooltip && (
        <div className="menu-desc-tooltip">
          {desc}
        </div>
      )}
    </div>
  );
}
```

실제 구현은 현재 프로젝트 구조에 맞게 조정해 주세요.

## 최종 기대 결과

| 항목          | 기대 결과                        |
| ----------- | ---------------------------- |
