# Visual Editor Mode 상단 헤더 높이 변동 문제 수정 요청

## 현재 문제

Visual Editor Mode에서 상단 메뉴를 드래그하거나 MOVE 탭이 표시될 때,  
상단 헤더 영역의 높이가 달라지는 문제가 있습니다.

현재 화면에서 확인되는 증상:

- Visual Editor Mode 상단 메뉴바 높이가 상황에 따라 줄어들거나 늘어남
- MOVE 라벨 또는 드래그 핸들 탭이 표시될 때 상단 네비 영역이 위아래로 밀림
- 탭이 열렸을 때 상단 메뉴 영역이 압축되어 보임
- 실제 콘텐츠 시작 위치도 달라지는 것처럼 보임

즉, 드래그용 UI 요소가 헤더의 일반 레이아웃 흐름에 포함되어  
상단 영역 높이를 변경시키는 것으로 보입니다.

---

# 수정 범위

이번 수정은 **Visual Editor Mode 전용 UI 수정**입니다.

아래 영역은 수정하지 않습니다.

- List Mode
- 실제 메뉴 데이터 구조
- drag/drop 저장 로직
- 메뉴 level 변경 로직
- 기존 List Mode 레이아웃

이번에는 Visual Mode의 상단 드래그 탭 / MOVE 라벨 / drag handle preview 위치만 수정해주세요.

---

# 핵심 수정 방향

Visual Editor Mode에서 표시되는 MOVE 라벨, 드래그 핸들, 선택 탭, floating tab 등은  
절대 헤더의 높이에 영향을 주면 안 됩니다.

따라서 해당 요소들은 일반 문서 흐름에 포함하지 말고,  
`position: absolute` 또는 `position: fixed`로 띄워주세요.

```txt
잘못된 방식:
MOVE 탭이 header 내부 flex/grid 흐름에 포함됨
→ header height가 변함
→ 메뉴바가 밀림

올바른 방식:
MOVE 탭은 header 위에 overlay로 표시
→ header height는 항상 고정
→ 메뉴바 위치 변화 없음
````

---

# 상단 헤더 고정 조건

Visual Editor Mode에서도 상단 헤더 높이는 항상 일정해야 합니다.

예시 기준:

```css
.visual-editor-header {
  height: 72px;
  min-height: 72px;
  max-height: 72px;
}
```

또는 현재 프로젝트에서 사용하는 실제 헤더 높이에 맞춰
고정값을 유지해주세요.

중요한 점은 아래입니다.

```txt
MOVE 탭이 보이든 안 보이든
Visual Editor Mode가 켜져 있든 꺼져 있든
드래그 중이든 아니든
상단 헤더 높이는 변하면 안 됨
```

---

# MOVE 탭 / 드래그 탭 위치 처리

MOVE 라벨이나 드래그용 탭은 헤더 내부 레이아웃을 밀지 않고
겹쳐서 표시되도록 해주세요.

예시:

```css
.visual-move-badge {
  position: absolute;
  top: 0;
  left: 48px;
  z-index: 50;
  pointer-events: none;
}
```

또는 상단 전체 기준으로 띄워야 한다면:

```css
.visual-move-badge {
  position: fixed;
  top: 72px;
  left: 48px;
  z-index: 999;
  pointer-events: none;
}
```

단, `position: relative` 기준 부모를 명확히 지정해주세요.

```css
.visual-editor-wrapper {
  position: relative;
}
```

---

# 금지 사항

아래 방식은 사용하지 말아주세요.

```txt
MOVE 탭을 header flex row 내부에 삽입
MOVE 탭 때문에 padding-top / margin-top 변경
MOVE 탭 표시 여부에 따라 header height 변경
드래그 중에 nav bar top 값 변경
드래그 중에 main content margin-top 변경
```

특히 아래와 같은 방식은 피해주세요.

```css
.header {
  height: auto;
}

.header.has-move-tab {
  padding-top: 20px;
}
```

이렇게 되면 탭이 보일 때마다 헤더 높이가 바뀝니다.

# Visual Mode 상단 레이아웃 기준

Visual Editor Mode의 상단 구조는 아래처럼 유지되어야 합니다.

```txt
[고정된 상단 헤더]
- 로고
- 검색창
- VISUAL EDITOR MODE 라벨
- 우측 아이콘

[고정된 상단 메뉴바]
- star 학생지원
- 학사정보
- 대학생활
- 진로개발
- 취업지원
- SUPER STAR
- 정보공유
- My Page

[본문 영역]
- 메뉴 카드 / 미리보기 영역
```

MOVE 라벨이나 드래그 탭은 위 구조를 밀어내면 안 되고,
필요한 위치에 overlay로만 표시되어야 합니다.

---

# 예상 원인

현재 문제는 아마 아래 중 하나로 보입니다.

## 1. MOVE 라벨이 nav item 내부에 normal flow로 삽입됨

```html
<li class="nav-item">
  <span class="move-label">MOVE</span>
  star 학생지원
</li>
```

이 경우 `move-label`이 메뉴 높이에 영향을 줄 수 있습니다.

수정 방향:

```css
.nav-item {
  position: relative;
}

.move-label {
  position: absolute;
  top: -14px;
  left: 0;
}
```

---

## 2. 드래그 핸들 영역이 메뉴 아이템 높이를 키움

현재 드래그 핸들이 메뉴 텍스트와 같은 줄 구조에 들어가면서
아이템 height를 변경하는 경우가 있습니다.

수정 방향:

```css
.drag-handle {
  position: absolute;
  left: 8px;
  top: 50%;
  transform: translateY(-50%);
}
```

그리고 메뉴 텍스트는 기존 높이를 유지합니다.

---

## 3. Visual Mode 활성화 시 header padding이 바뀜

Visual Mode일 때 별도 class가 붙으면서
header나 nav에 padding/margin이 추가되는지 확인해주세요.

예시:

```css
.visual-mode .top-header {
  padding-top: 12px;
}
```

이런 코드는 제거하거나, 내부 요소만 absolute로 처리해주세요.

---

# 수정 요청 상세

## 1. Header height 고정

Visual Mode 여부와 관계없이 header 높이를 고정해주세요.

```css
.top-header {
  height: var(--header-height);
  min-height: var(--header-height);
  max-height: var(--header-height);
}
```

---

## 2. Navigation bar height 고정

상단 메뉴바도 높이가 변하지 않도록 고정해주세요.

```css
.top-nav {
  height: var(--nav-height);
  min-height: var(--nav-height);
  max-height: var(--nav-height);
  overflow: visible;
}
```

`overflow: visible`은 MOVE 라벨이 위로 살짝 떠야 할 경우 필요합니다.

---

## 3. MOVE badge는 absolute overlay 처리

```css
.nav-item {
  position: relative;
}

.move-badge {
  position: absolute;
  top: -14px;
  left: 12px;
  z-index: 20;
  pointer-events: none;
}
```

---

## 4. Drag handle은 메뉴 높이에 영향 없게 처리

```css
.visual-drag-handle {
  position: absolute;
  left: 8px;
  top: 50%;
  transform: translateY(-50%);
}
```

또는 아이콘이 inline으로 들어가야 한다면
아이콘 크기와 line-height를 고정해주세요.

```css
.visual-drag-handle {
  width: 16px;
  height: 16px;
  line-height: 16px;
  flex-shrink: 0;
}
```

---

## 5. 드래그 중 placeholder가 header 안에 생기지 않게 하기

상단 메뉴를 드래그할 때 placeholder 또는 preview element가
header/nav 내부에 삽입되어 높이를 바꾸면 안 됩니다.

드래그 preview는 portal 또는 overlay layer로 분리해주세요.

예시:

```txt
드래그 preview 위치:
body > visual-drag-overlay

금지:
header > nav > drag-preview
```

# 최종 요약

Visual Editor Mode에서 MOVE 탭 또는 드래그 관련 UI가
상단 헤더/네비의 일반 레이아웃 흐름에 포함되면서
헤더 높이를 바꾸고 있는 것으로 보입니다.

따라서 해당 요소는 반드시 overlay 방식으로 분리해주세요.

```txt
MOVE 탭 / drag handle / drag preview는
header height에 영향을 주면 안 된다.
```

핵심 요구사항:

```txt
Visual Editor Mode 상단 헤더와 메뉴바 높이는 항상 고정
MOVE 탭은 absolute/fixed overlay로 표시
드래그 preview는 header 내부가 아니라 별도 overlay layer 사용
List Mode는 수정하지 않음
```
