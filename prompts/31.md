# 비주얼모드 드래그앤드랍 UX 단순화 요청

## 핵심 방향

현재 비주얼모드에서 드래그앤드랍 시 금지 표시가 자주 뜨고, 금지 표시 위치도 실제 마우스 위치와 어긋나는 문제가 있습E니다.

이 방식은 사용자 입장에서 복잡하고 조잡하게 느껴집니다.

따라서 “잘못된 위치에 올리면 금지 표시를 보여주는 방식”이 아니라,  
**처음부터 잡은 메뉴 레벨에 따라 이동 가능한 영역만 활성화하는 방식**으로 단순화해 주세요.

즉, 금지 표시를 중심으로 구현하지 말고  
**레벨별 허용 동작만 가능하게 제한**하는 방식으로 변경해 주세요.

---

## 기본 원칙

드래그를 시작한 아이템의 level에 따라 가능한 이동 범위를 고정합니다.

```txt
Level 1을 잡음 → Level 1끼리만 정렬
Level 2를 잡음 → Level 1 그룹 안에서 Level 2끼리만 정렬
Level 3을 잡음 → Level 2 그룹 안에서 Level 3끼리만 정렬
````

잘못된 위치에 드래그했을 때 금지표시를 보여주는 것보다,
애초에 잘못된 위치는 드롭 후보로 계산하지 않도록 해 주세요.

---

# 1. Level 1 드래그 동작

## 허용 동작

Level 1 메뉴를 잡았을 때는 무조건 Level 1 메뉴들 사이에서만 정렬되게 해 주세요.

예시:

```txt
[star 학생지원] [학사정보] [대학생활] [진로개발]
```

Level 1을 드래그하면 이 상단 Level 1 메뉴들 사이에서만 이동 가능합니다.

## 금지/비활성 처리

Level 1을 잡은 상태에서는 아래 영역을 드롭 대상으로 보지 않습니다.

```txt
Level 2 영역
Level 3 영역
본문 빈 공간
하위 메뉴 카드
```

즉, Level 1이 Level 2나 Level 3 아래로 들어갈 가능성 자체가 없어야 합니다.

## 기대 동작

```txt
Level 1 드래그 시작
→ Level 1 정렬 모드로 진입
→ Level 1 메뉴들만 드롭 후보
→ 다른 영역은 드롭 후보 아님
→ 금지표시 불필요
```

---

# 2. Level 2 드래그 동작

## 허용 동작

Level 2 메뉴를 잡았을 때는 Level 2 정렬/이동 모드로 동작해야 합니다.

허용되는 동작은 두 가지입니다.

```txt
1. 현재 Level 1 안에서 Level 2 순서 정렬
2. 다른 Level 1 레이블 위로 이동하면 해당 Level 1 메뉴로 전환 후 그 안에서 Level 2 정렬
```

예시:

```txt
취업지원 > 취업정보
```

를 잡고

```txt
정보공유
```

Level 1 레이블 위로 이동하면,
화면이 정보공유 Level 1 메뉴로 전환되고,
그 안의 Level 2 목록 사이에서 정렬할 수 있어야 합니다.

## 중요한 점

Level 2를 잡았을 때는 Level 3 영역을 드롭 대상으로 볼 필요가 없습니다.

즉, 사용자가 Level 2를 Level 3 위에 끌고 가더라도:

```txt
금지표시를 보여줄 필요 없음
Level 3을 드롭 후보로 계산하지 않음
Level 2 정렬 후보만 계속 유지
```

## 기대 동작

```txt
Level 2 드래그 시작
→ Level 2 정렬 모드로 진입
→ 현재 Level 1의 Level 2 목록에서 정렬 가능
→ 다른 Level 1 레이블 hover 시 해당 Level 1로 전환
→ 전환된 Level 1 내부에서 Level 2 정렬 가능
→ Level 3 영역은 드롭 후보에서 제외
```

---

# 3. Level 3 드래그 동작

## 허용 동작

Level 3 메뉴를 잡았을 때는 Level 3 정렬/이동 모드로 동작해야 합니다.

허용되는 동작은 두 가지입니다.

```txt
1. 같은 Level 2 내부에서 Level 3 순서 정렬
2. 다른 Level 2 아래로 이동하여 그 Level 2 내부에서 Level 3 정렬
```

예시:

```txt
star 관리 > 희망학과 선택
```

Level 3 메뉴를 잡고 다른 Level 2인

```txt
장학관리
```

위 또는 내부로 이동하면,
장학관리 하위 Level 3 목록 안에서 정렬할 수 있어야 합니다.

## Level 3의 이동 범위

Level 3을 잡은 상태에서는 Level 2 그룹만 드롭 후보가 됩니다.

아래 영역은 드롭 후보로 보지 않습니다.

```txt
Level 1 메뉴 자체
Level 1 빈 공간
다른 Level 3의 하위
본문 빈 공간
```

## 기대 동작

```txt
Level 3 드래그 시작
→ Level 3 정렬 모드로 진입
→ 현재 Level 2 내부에서 정렬 가능
→ 다른 Level 2 hover 시 해당 Level 2의 하위 목록으로 이동 가능
→ Level 1 바로 아래에는 들어가지 않음
→ 금지표시 불필요
```

---

# 4. 금지표시 제거 또는 최소화

현재처럼 금지표시가 여러 곳에 뜨거나, 마우스 위치와 다른 곳에 뜨는 방식은 제거해 주세요.

## 수정 요청

가능하면 비주얼모드에서 금지표시는 사용하지 않는 방향으로 변경해 주세요.

대신:

```txt
잡은 레벨에 맞는 드롭 후보만 활성화
잡은 레벨에 맞지 않는 영역은 드롭 계산에서 제외
```

하는 방식으로 구현해 주세요.

## 예외

정말 필요한 경우에만 간단한 안내 토스트를 사용할 수 있습니다.

예시:

```txt
이 메뉴는 현재 레벨 안에서만 이동할 수 있습니다.
```

하지만 일반적인 드래그 중에는 금지 표시가 계속 보이지 않는 것이 좋습니다.

# 5. 레벨별 드래그 모드 개념

드래그 시작 시점에 현재 드래그 모드를 결정해 주세요.

```js
function getDragMode(dragItem) {
  if (dragItem.level === 1) return "LEVEL_1_SORT";
  if (dragItem.level === 2) return "LEVEL_2_SORT_OR_MOVE";
  if (dragItem.level === 3) return "LEVEL_3_SORT_OR_MOVE";
}
```

드래그 중에는 이 모드에 맞는 대상만 후보로 계산합니다.

```txt
LEVEL_1_SORT
→ Level 1 슬롯만 활성

LEVEL_2_SORT_OR_MOVE
→ Level 1 레이블 hover 가능
→ 해당 Level 1 내부의 Level 2 슬롯만 활성

LEVEL_3_SORT_OR_MOVE
→ Level 2 hover 가능
→ 해당 Level 2 내부의 Level 3 슬롯만 활성
```

# 6. Level 2 이동 상세 동작

Level 2 이동은 아래 방식으로 처리해 주세요.

## 같은 Level 1 내부 정렬

```txt
현재 Level 1: 취업지원

[창업] [취업정보] [현장실습학기제]
```

`취업정보`를 `창업` 앞으로 이동하면:

```txt
[취업정보] [창업] [현장실습학기제]
```

## 다른 Level 1로 이동

```txt
Level 2: 취업정보
현재 부모: 취업지원
이동 대상 Level 1: 정보공유
```

동작:

```txt
1. Level 2 아이템을 잡음
2. 상단의 정보공유 Level 1 레이블 위로 이동
3. 화면이 정보공유 메뉴로 전환됨
4. 정보공유 안의 Level 2 목록이 보임
5. 원하는 위치에 드롭
6. 취업정보의 parent가 정보공유로 변경됨
7. 해당 위치에 삽입됨
```

이 과정에서 Level 3 영역은 드롭 후보가 아니어야 합니다.

---

# 7. Level 3 이동 상세 동작

Level 3 이동은 아래 방식으로 처리해 주세요.

## 같은 Level 2 내부 정렬

```txt
현재 Level 2: 커뮤니티

[Q&A] [공지사항] [설문조사] [시설 관련 요청]
```

`설문조사`를 `Q&A` 앞으로 이동하면:

```txt
[설문조사] [Q&A] [공지사항] [시설 관련 요청]
```

## 다른 Level 2 아래로 이동

```txt
Level 3: 설문조사
현재 부모: 커뮤니티
이동 대상 Level 2: 졸업관리
```

동작:

```txt
1. Level 3 아이템을 잡음
2. 다른 Level 2 그룹 위로 이동
3. 해당 Level 2 하위 목록이 드롭 후보가 됨
4. 원하는 위치에 드롭
5. Level 3의 parent가 해당 Level 2로 변경됨
6. 해당 위치에 삽입됨
```

# 8. 드롭 프리뷰

드래그 중에는 실제 들어갈 위치를 명확하게 보여주세요.

## 기대 UI

```txt
아이템이 들어갈 위치에 placeholder 표시
기존 아이템은 그 위치를 기준으로 밀려남
드롭 후 실제 위치는 프리뷰와 동일
```

중요한 원칙:

```txt
프리뷰 위치 = 실제 드롭 위치
```

드래그 중 계산한 preview parent/index와
drop 시 계산한 final parent/index가 달라지면 안 됩니다.

---

# 9. 드롭 계산 방식

드롭 계산은 현재 드래그 모드에 맞는 슬롯만 대상으로 해야 합니다.

## Level 1

```txt
대상 슬롯: Level 1 목록
계산 기준: Level 1 사이 index
```

## Level 2

```txt
대상 슬롯: 현재 활성 Level 1의 Level 2 목록
계산 기준: Level 2 사이 index
```

## Level 3

```txt
대상 슬롯: 현재 활성 Level 2의 Level 3 목록
계산 기준: Level 3 사이 index
```

잘못된 레벨의 슬롯은 계산 대상에서 제외합니다.

---

# 10. 마우스 위치와 무관한 금지표시 제거

현재 문제 중 하나는 금지표시가 마우스 위치가 아닌 다른 곳에 나타나는 것입니다.

이 문제를 해결하기 위해 금지표시 방식 자체를 제거하거나 최소화해 주세요.

드래그 중 표시되어야 하는 것은 아래 정도면 충분합니다.

```txt
현재 이동 가능한 대상 영역 강조
삽입될 위치 placeholder
기존 아이템 밀림 프리뷰
```

잘못된 영역은 굳이 금지표시를 띄우지 말고
그냥 아무 반응도 하지 않게 처리해 주세요.

# 11. 히스토리 연동

드롭이 성공했을 때만 히스토리에 1회 저장해 주세요.

```txt
드래그 시작 → 히스토리 변화 없음
드래그 중 프리뷰 변경 → 히스토리 변화 없음
드롭 성공 → commitMenuChange(nextMenuState) 1회 호출
드롭 취소 → 히스토리 변화 없음
잘못된 영역에 드롭 → 히스토리 변화 없음
```

중요:

```txt
금지표시 제거와 관계없이, 실제 상태 변경이 없는 경우 히스토리에 저장하지 말 것
```

# 13. 최종 기대 결과

| 드래그 대상                   | 동작                                |
| ------------------------ | --------------------------------- |
| Level 1                  | Level 1 목록 안에서만 정렬                |
| Level 2                  | 현재 Level 1의 Level 2 목록에서 정렬       |
| Level 2 → 다른 Level 1 레이블 | 해당 Level 1로 전환 후 그 안에서 Level 2 정렬 |
| Level 3                  | 현재 Level 2의 Level 3 목록에서 정렬       |
| Level 3 → 다른 Level 2     | 해당 Level 2 아래로 이동 후 Level 3 정렬    |
| 잘못된 영역                   | 드롭 후보가 아니므로 아무 반응 없음              |
| 금지표시                     | 가능하면 사용하지 않음                      |
| 프리뷰                      | 실제 드롭 결과와 동일                      |
| 히스토리                     | 성공한 드롭 1회당 되돌리기 +1                |

# 핵심 요약

현재처럼 드래그 중 금지표시를 띄우며 모든 영역을 판정하려고 하지 말고,
드래그 시작 시점에 잡은 아이템의 Level에 따라 가능한 드롭 후보만 활성화해 주세요.

```txt
Level 1 → Level 1 정렬만
Level 2 → Level 2 정렬/다른 Level 1로 이동만
Level 3 → Level 3 정렬/다른 Level 2로 이동만
```

잘못된 영역은 금지표시를 띄우는 것이 아니라
애초에 드롭 후보에서 제외하는 방식으로 처리해 주세요.

```
