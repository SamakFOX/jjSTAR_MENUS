# 현재 오류별 수정 방향

## 1. Level 1 순서변경 시 블러가 앞을 가리는 문제

현재 `Level 1 순서변경 모드`에서 블러 오버레이가 리스트 위에 올라와서 메뉴를 가리고 있어.

수정 방향:

```txt
Level 1 순서변경 모드에서는 배경 블러를 제거하거나,
블러 레이어의 z-index를 리스트보다 낮게 설정한다.

드래그 가능한 Level 1 카드와 안내 배지는 블러 위에 표시되어야 한다.
```

권장 방식은 **블러 제거**야.
순서변경 중이라는 강조는 노란 배지와 카드 테두리만으로 충분해.

```txt
대분류(Level 1) 순서 변경 중
```

배지는 상단 고정으로 두고, 리스트 자체는 선명하게 보여야 함.

---

## 2-3. Level 3을 다른 Level 2로 넣었다가 원래 Level 2로 되돌릴 때 순서정렬만 되는 문제

이건 아까 정한 규칙이 제대로 분리되지 않은 거야.

우리가 정한 건 이거였어.

```txt
장바구니 아이콘 드롭존 = 해당 메뉴 안으로 넣기
그 외 영역 = 순서정렬
```

즉 Level 3을 원래 Level 2 아래로 다시 넣고 싶으면, 반드시 **그 Level 2의 장바구니 아이콘 드롭존**에 넣었을 때 들어가야 해.

지금은 장바구니 드롭존이 제대로 작동하지 않거나, `drop-into:{id}`와 `sort:{id}`가 분리되지 않아서 전부 순서정렬로 처리되는 것 같아.

수정 방향:

```txt
Level 3을 Level 2의 장바구니 드롭존에 놓으면
반드시 해당 Level 2의 children으로 이동해야 한다.

Level 3을 Level 2의 행 사이 또는 일반 영역에 놓으면
같은 부모 안에서의 순서정렬로만 처리한다.
```

그리고 원래 부모로 되돌릴 때도 동일하게:

```txt
Level 3 → 원래 Level 2 장바구니 드롭존
= 원래 Level 2 하위로 다시 이동
```

---

## 3. 명시적 Level Down 버튼이 아무 반응 없는 문제

이건 명시적 Level 버튼 로직이 누락되었거나, 조건에 걸렸는데 안내 문구를 안 띄우는 상태야.

명시적 Level Down 규칙은 이렇게 되어야 해.

### Level 2 Down

```txt
Level 2를 Level Down 하면 Level 3이 된다.
단, 해당 Level 2에 하위메뉴가 없어야 한다.
```

위치 규칙:

```txt
같은 Level 1 안에서 바로 뒤에 Level 2가 있으면
→ 그 Level 2의 첫 번째 Level 3으로 이동

뒤에 Level 2가 없으면
→ 바로 앞 Level 2의 마지막 Level 3으로 이동

앞뒤에 받을 수 있는 Level 2가 없으면
→ 안내 문구 표시
```

예시:

```txt
A L2
B L2
  C L3
  D L3
```

A를 Level Down:

```txt
B L2
  A L3
  C L3
  D L3
```

필수 수정:

```txt
명시적 Level Down 버튼을 눌렀는데 조건 불가라면 아무 반응 없이 끝내면 안 된다.
반드시 안내 메시지를 표시한다.
```

문구 예시:

```txt
이 메뉴를 Level 3으로 변경할 수 없습니다.
같은 Level 1 안에 이동할 대상 Level 2가 없습니다.
```

또는:

```txt
"{메뉴명}"에는 하위메뉴가 있어 Level 3으로 변경할 수 없습니다.
먼저 하위메뉴를 이동하거나 삭제해 주세요.
```

---

## 4. Level 2를 다른 Level 1에 담기 기능으로 넣고 싶은데 장바구니 아이콘이 반응 없는 문제

이건 장바구니 아이콘이 실제 droppable로 등록되지 않았거나, 드롭 이벤트에서 구분을 못 하는 상태야.

수정 방향:

```txt
각 메뉴 행은 두 개의 드롭 영역을 가져야 한다.

1. sort:{menuId}
   - 일반 행 영역
   - 순서정렬 전용

2. drop-into:{menuId}
   - 왼쪽 장바구니 아이콘 영역
   - 하위 이동/통합 전용
```

예시:

```txt
Level 2 → Level 1의 drop-into 영역
= 해당 Level 1의 children으로 이동

Level 3 → Level 2의 drop-into 영역
= 해당 Level 2의 children으로 이동

Level 2 → Level 2의 drop-into 영역
= 통합 팝업 표시
```

Level 2를 Level 1에 넣는 경우 문구:

```txt
"{Level 2 이름}"의 하위메뉴도 함께 이동하시겠습니까?

[함께 이동] [취소]
```

취소하면 아무것도 변경하지 않음.

---

## 5-6. 드래그드롭 시 Level 2나 Level 3이 Level 1로 올라가버리는 문제

이건 반드시 막아야 해.
현재 드롭 후 레벨 보정 로직이 너무 느슨해서 `Level 2`, `Level 3`이 root로 올라가는 것 같아.

최종 구조 제한은 이거야.

```txt
Level 1은 root에만 존재
Level 2는 반드시 Level 1의 children에만 존재
Level 3은 반드시 Level 2의 children에만 존재
```

따라서 금지해야 하는 결과:

```txt
Level 2가 root로 올라감  → 금지
Level 3이 root로 올라감  → 금지
Level 3이 Level 1의 직접 자식이 됨 → 금지
Level 2가 Level 2의 자식이 됨 → 직접 이동 금지, 통합으로만 처리
```

수정 방향:

```txt
드롭 처리 후 level을 Math.max / Math.min으로 보정하지 말고,
아예 잘못된 구조가 만들어지는 이동을 사전에 차단한다.
```

즉 이런 식의 보정은 위험해.

```js
level: Math.max(1, Math.min(MAX_LEVEL, level))
```

이렇게 하면 Level 3이 잘못 root로 올라갔을 때도 그냥 L1/L2로 보정돼서 구조가 깨질 수 있음.

대신 tree 기준으로 처리해야 해.

```txt
Level 2를 이동하려면 target parent가 Level 1인지 확인
Level 3을 이동하려면 target parent가 Level 2인지 확인
조건이 맞지 않으면 이동 취소
```

# 에이전트에게 전달할 수정 요청 문구

아래 그대로 전달하면 돼.

````md
# 메뉴 편집기 오류 수정 요청

## 1. Level 1 순서변경 모드 블러 문제

현재 Level 1 순서변경 모드에서 블러 오버레이가 메뉴 리스트를 가리고 있습니다.

수정 요청:
- Level 1 순서변경 모드에서는 리스트가 선명하게 보여야 합니다.
- 블러 효과를 제거하거나, 블러 레이어의 z-index를 리스트보다 낮게 설정해 주세요.
- 안내 배지와 Level 1 카드만 강조되면 됩니다.
- 사용자가 순서를 변경할 메뉴 자체가 흐려지면 안 됩니다.

---

## 2. 장바구니 드롭존과 순서정렬 영역 분리

리스트모드 드래그는 다음 두 영역을 명확히 분리해야 합니다.

### sort 영역

```txt
sort:{menuId}
````

* 일반 행 영역
* 행 위/아래/사이 영역
* 순서정렬 전용

### drop-into 영역

```txt
drop-into:{menuId}
```

* 왼쪽 장바구니 아이콘 영역
* 하위 이동 또는 통합 전용

규칙:

* 장바구니 아이콘 영역에 드롭한 경우에만 “해당 메뉴 안으로 넣기 / 통합”으로 처리합니다.
* 장바구니 아이콘 외의 영역에 드롭한 경우에는 무조건 순서정렬로 처리합니다.

---

## 3. Level 3을 Level 2에 넣는 동작 수정

현재 Level 3을 다른 Level 2로 넣었다가 다시 원래 Level 2로 되돌리려 하면 순서정렬만 적용되고 하위로 들어가지 않는 문제가 있습니다.

수정 요청:

* Level 3을 Level 2의 `drop-into:{level2Id}` 영역에 놓으면 반드시 해당 Level 2의 children으로 이동해야 합니다.
* 원래 부모 Level 2로 되돌릴 때도 동일하게 작동해야 합니다.
* Level 3을 Level 2의 일반 행 영역이나 사이 영역에 놓으면 순서정렬만 처리합니다.

---

## 4. Level 2를 Level 1에 넣는 담기 기능 수정

현재 Level 2를 다른 Level 1의 장바구니 아이콘 쪽에 올려도 반응이 없습니다.

수정 요청:

* Level 2를 Level 1의 `drop-into:{level1Id}` 영역에 놓으면 해당 Level 1의 children으로 이동해야 합니다.
* 이동 시 Level 2의 하위 Level 3도 함께 이동해야 합니다.
* 드롭 시 확인 팝업을 표시합니다.

문구:

```txt
"{Level 2 이름}"의 하위메뉴도 함께 이동하시겠습니까?
```

버튼:

```txt
[함께 이동] [취소]
```

---

## 5. Level 2 → Level 2 드롭 시 통합 처리

Level 2는 다른 Level 2의 자식이 될 수 없습니다.

따라서 Level 2를 Level 2의 `drop-into:{level2Id}` 영역에 놓으면 이동이 아니라 통합 확인 팝업을 표시해야 합니다.

문구:

```txt
"{이동할 Level 2 이름}"은 Level 2 메뉴이므로 "{대상 Level 2 이름}"의 하위 메뉴가 될 수 없습니다.

대신 "{이동할 Level 2 이름}" 메뉴를 제거하고,
그 하위메뉴를 "{대상 Level 2 이름}"의 하위메뉴로 통합할 수 있습니다.

진행하시겠습니까?
```

버튼:

```txt
[통합하기] [취소]
```

처리:

* 이동할 Level 2 자체는 제거합니다.
* 이동할 Level 2의 하위 Level 3만 대상 Level 2의 children으로 이동합니다.
* 화면에 `[삭제됨]` 그룹은 남기지 않습니다.
* 변경 로그에는 통합 기록을 남깁니다.

---

## 6. Level 2 / Level 3이 Level 1로 올라가는 문제 차단

현재 드래그드롭 시 Level 2나 Level 3이 Level 1로 올라가버리는 문제가 있습니다.

수정 요청:

* Level 2는 반드시 Level 1의 children으로만 존재해야 합니다.
* Level 3은 반드시 Level 2의 children으로만 존재해야 합니다.
* Level 2가 root로 올라가는 이동은 금지합니다.
* Level 3이 root로 올라가는 이동은 금지합니다.
* Level 3이 Level 1의 직접 자식이 되는 이동도 금지합니다.

금지된 드롭이면 이동하지 않고 안내 문구를 표시합니다.

예시 문구:

```txt
Level 2 메뉴는 Level 1 하위에만 배치할 수 있습니다.
```

```txt
Level 3 메뉴는 Level 2 하위에만 배치할 수 있습니다.
```

---

## 7. 레벨 보정 방식 금지

드롭 후에 level 값을 `Math.max`, `Math.min` 등으로 보정해서 구조를 맞추는 방식은 사용하지 않습니다.

잘못된 구조가 만들어진 뒤 보정하지 말고, 애초에 잘못된 이동을 차단해야 합니다.

금지 예시:

```js
level: Math.max(1, Math.min(MAX_LEVEL, nextLevel))
```

대신 tree 기준으로 다음을 검증합니다.

```txt
Level 1 이동 가능 위치: root
Level 2 이동 가능 위치: Level 1.children
Level 3 이동 가능 위치: Level 2.children
```

---

## 8. 명시적 Level 버튼 수정

명시적 Level Down 버튼이 조건에 맞지 않을 때 아무 반응이 없는 문제가 있습니다.

수정 요청:

* Level 버튼을 눌렀는데 이동이 불가능하면 반드시 안내 문구를 표시합니다.
* 아무 반응 없이 종료되면 안 됩니다.

### Level 2 Down 규칙

Level 2를 Level Down 하면 Level 3이 됩니다.

조건:

* 해당 Level 2에 하위 Level 3이 없어야 합니다.
* 같은 Level 1 안의 형제 Level 2만 대상이 됩니다.

위치:

* 바로 뒤에 같은 부모의 Level 2가 있으면 그 Level 2의 첫 번째 Level 3으로 이동합니다.
* 뒤에 Level 2가 없으면 바로 앞 같은 부모의 Level 2의 마지막 Level 3으로 이동합니다.
* 앞뒤에 받을 수 있는 Level 2가 없으면 Level Down 불가입니다.

불가 문구:

```txt
이 메뉴를 Level 3으로 변경할 수 없습니다.
같은 Level 1 안에 이동할 대상 Level 2가 없습니다.
```

자식이 있는 경우 문구:

```txt
"{메뉴명}"에는 하위메뉴가 있어 Level 3으로 변경할 수 없습니다.
먼저 하위메뉴를 이동하거나 삭제해 주세요.
```

---

## 9. Level 3 Up 규칙 확인

Level 3을 Level Up 하면 Level 2가 됩니다.

처리:

* 현재 부모 Level 2의 바로 위쪽에 Level 2로 올라갑니다.
* 해당 위치로 자동 스크롤합니다.
* 주변 Level 3을 자동으로 흡수하지 않습니다.
* 클릭한 해당 아이템 하나만 Level 변경됩니다.

---

## 10. 최종 구조 검증

모든 드래그, 버튼 이동, 삭제, 통합 후에는 tree 구조 검증을 수행합니다.

검증 항목:

* Level 1은 root에만 존재하는가
* Level 2는 Level 1의 children에만 존재하는가
* Level 3은 Level 2의 children에만 존재하는가
* Level 4 이상이 생기지 않았는가
* parent가 끊긴 메뉴가 없는가

검증 실패 시 해당 변경은 롤백합니다.
