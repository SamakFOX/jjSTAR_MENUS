# Visual Editor Mode reorder 후 validate 실패 문제 수정 요청

## 현재 문제

Visual Editor Mode에서 같은 레벨끼리 순서 변경을 시도하면 아래 로그가 발생하며 이동이 막힙니다.

```txt
Invalid visual tree blocked after drop:
{ dragLevel: 1, dropMode: 'reorder', targetLevel: 1 }

Invalid visual tree blocked after drop:
{ dragLevel: 2, dropMode: 'reorder', targetLevel: 2 }

Invalid visual tree blocked after drop:
{ dragLevel: 3, dropMode: 'reorder', targetLevel: 3 }
````

즉, Level 1 → Level 1, Level 2 → Level 2, Level 3 → Level 3의 reorder는 모두 허용되어야 하는데,
drop 이후 `validateVisualTree()`에서 잘못된 트리로 판단되어 차단되고 있습니다.

---

# 중요 전제

List Mode는 현재 정상 동작하고 있으므로 수정하지 않습니다.

이번 수정은 **Visual Editor Mode 전용 reorder 처리와 validate 처리만 보정**합니다.

---

# 원인 추정

현재 문제는 reorder 자체가 금지된 것이 아니라,
reorder 이후 만들어진 `updatedNodes` 구조가 validate 기준을 통과하지 못하는 것입니다.

가능한 원인은 아래 중 하나입니다.

```txt
1. reorder 후 dragItem의 parentId가 사라짐
2. reorder 후 dragItem의 parentId가 잘못된 레벨을 가리킴
3. Level 1 reorder 시 하위 Level 2/3이 함께 이동하지 않음
4. Level 2 reorder 시 하위 Level 3이 함께 이동하지 않음
5. flat list 구조인데 validateVisualTree가 tree 구조처럼 검사함
6. nodes 배열에 parent node보다 child node가 먼저 배치됨
7. reorder 후 children 관계와 parentId 관계가 서로 불일치함
8. validateVisualTree가 이동 중 임시 placeholder 또는 drag overlay까지 검사함
```

---

# 핵심 수정 방향

같은 레벨끼리 reorder는 정상 동작해야 합니다.

```txt
Level 1 → Level 1 reorder 허용
Level 2 → Level 2 reorder 허용
Level 3 → Level 3 reorder 허용
```

단, reorder 후 구조는 아래 규칙을 유지해야 합니다.

```txt
Level 1 = parentId 없음
Level 2 = parentId가 반드시 Level 1
Level 3 = parentId가 반드시 Level 2
```

---

# Level 1 reorder 처리 규칙

Level 1끼리 reorder할 때:

```txt
dragItem.level === 1
targetItem.level === 1
dropMode === 'reorder'
```

처리 기준:

```txt
dragItem은 Level 1 상태 유지
dragItem.parentId는 null 또는 undefined 유지
dragItem의 하위 Level 2, Level 3도 함께 그룹으로 이동
```

Level 1만 단독으로 이동하고 하위 Level 2/3이 제자리에 남으면
전체 구조가 깨질 수 있으므로, Level 1 reorder는 반드시 하위 메뉴까지 그룹 단위로 이동해야 합니다.

---

# Level 2 reorder 처리 규칙

Level 2끼리 reorder할 때:

```txt
dragItem.level === 2
targetItem.level === 2
dropMode === 'reorder'
```

처리 기준:

```txt
dragItem.level = 2
dragItem.parentId = targetItem이 속한 Level 1 id
dragItem의 하위 Level 3도 함께 그룹으로 이동
```

같은 Level 1 안에서 이동하는 경우:

```txt
parentId 유지
순서만 변경
```

다른 Level 1 아래의 Level 2 사이로 이동하는 경우:

```txt
parentId를 targetItem.parentId로 변경
하위 Level 3도 함께 이동
```

---

# Level 3 reorder 처리 규칙

Level 3끼리 reorder할 때:

```txt
dragItem.level === 3
targetItem.level === 3
dropMode === 'reorder'
```

처리 기준:

```txt
dragItem.level = 3
dragItem.parentId = targetItem이 속한 Level 2 id
targetItem의 before/after 위치로 삽입
```

같은 Level 2 안에서 이동하는 경우:

```txt
parentId 유지
순서만 변경
```

다른 Level 2 아래의 Level 3 사이로 이동하는 경우:

```txt
parentId를 targetItem.parentId로 변경
순서 변경
```

# Visual Mode 전용 parentId 계산 함수

reorder 시 targetItem.parentId만 무조건 믿지 말고,
현재 flat list 기준으로도 부모를 찾을 수 있게 보정해주세요.

```js
function getVisualParentIdForReorder(dragItem, targetItem, nodes) {
  // Level 1 reorder
  if (dragItem.level === 1 && targetItem.level === 1) {
    return null;
  }

  // Level 2 reorder
  if (dragItem.level === 2 && targetItem.level === 2) {
    // 1차: targetItem.parentId가 유효한 Level 1인지 확인
    if (targetItem.parentId) {
      const parent = nodes.find((node) => node.id === targetItem.parentId);
      if (parent && parent.level === 1) {
        return parent.id;
      }
    }

    // 2차: flat list에서 targetItem 위쪽의 가장 가까운 Level 1 찾기
    const targetIndex = nodes.findIndex((node) => node.id === targetItem.id);

    for (let i = targetIndex - 1; i >= 0; i--) {
      if (nodes[i].level === 1) {
        return nodes[i].id;
      }
    }

    return null;
  }

  // Level 3 reorder
  if (dragItem.level === 3 && targetItem.level === 3) {
    // 1차: targetItem.parentId가 유효한 Level 2인지 확인
    if (targetItem.parentId) {
      const parent = nodes.find((node) => node.id === targetItem.parentId);
      if (parent && parent.level === 2) {
        return parent.id;
      }
    }

    // 2차: flat list에서 targetItem 위쪽의 가장 가까운 Level 2 찾기
    const targetIndex = nodes.findIndex((node) => node.id === targetItem.id);

    for (let i = targetIndex - 1; i >= 0; i--) {
      if (nodes[i].level === 2) {
        return nodes[i].id;
      }
    }

    return null;
  }

  return null;
}
```

# reorder 시 그룹 이동 처리

Level 1과 Level 2는 자식이 있을 수 있으므로,
단독 item만 이동하면 구조가 깨질 수 있습니다.

따라서 reorder 전에 dragItem의 하위 항목까지 함께 추출해야 합니다.

```js
function getVisualNodeGroup(item, nodes) {
  if (item.level === 1) {
    const level2Children = nodes.filter(
      (node) => node.level === 2 && node.parentId === item.id
    );

    const level2Ids = level2Children.map((node) => node.id);

    const level3Children = nodes.filter(
      (node) => node.level === 3 && level2Ids.includes(node.parentId)
    );

    return [item, ...level2Children, ...level3Children];
  }

  if (item.level === 2) {
    const level3Children = nodes.filter(
      (node) => node.level === 3 && node.parentId === item.id
    );

    return [item, ...level3Children];
  }

  return [item];
}
```

# applyVisualReorder 수정 요청

```js
function applyVisualReorder(dragItem, targetItem, position, nodes) {
  // 같은 레벨끼리만 reorder 허용
  if (dragItem.level !== targetItem.level) {
    console.warn('Invalid reorder: level mismatch');
    return nodes;
  }

  const newParentId = getVisualParentIdForReorder(dragItem, targetItem, nodes);

  if (dragItem.level === 1) {
    // Level 1은 parentId 없어야 함
    if (newParentId !== null) {
      console.warn('Invalid Level 1 parent');
      return nodes;
    }
  }

  if (dragItem.level === 2 && !newParentId) {
    console.warn('Level 2 reorder blocked: valid Level 1 parent not found');
    return nodes;
  }

  if (dragItem.level === 3 && !newParentId) {
    console.warn('Level 3 reorder blocked: valid Level 2 parent not found');
    return nodes;
  }

  const movingGroup = getVisualNodeGroup(dragItem, nodes);
  const movingIds = new Set(movingGroup.map((node) => node.id));

  // 기존 위치에서 이동 그룹 제거
  const remainingNodes = nodes.filter((node) => !movingIds.has(node.id));

  // targetItem의 새 index 계산
  const targetIndex = remainingNodes.findIndex((node) => node.id === targetItem.id);

  if (targetIndex === -1) {
    console.warn('Target item not found after removing moving group');
    return nodes;
  }

  const updatedGroup = movingGroup.map((node) => {
    // 실제 드래그된 노드만 parentId 보정
    if (node.id === dragItem.id) {
      return {
        ...node,
        level: dragItem.level,
        parentId: dragItem.level === 1 ? null : newParentId,
      };
    }

    // 자식들은 기존 parentId 유지
    return node;
  });

  const insertIndex = position === 'after' ? targetIndex + 1 : targetIndex;

  const updatedNodes = [
    ...remainingNodes.slice(0, insertIndex),
    ...updatedGroup,
    ...remainingNodes.slice(insertIndex),
  ];

  if (!validateVisualTree(updatedNodes)) {
    console.warn('Invalid visual tree blocked after reorder:', {
      dragLevel: dragItem.level,
      targetLevel: targetItem.level,
      dropMode: 'reorder',
      newParentId,
      dragTitle: dragItem.title,
      targetTitle: targetItem.title,
    });

    return nodes;
  }

  return updatedNodes;
}
```

# validateVisualTree 점검 요청

현재 validateVisualTree가 너무 엄격하거나
현재 데이터 구조와 맞지 않게 검사하고 있을 수 있습니다.

특히 아래를 확인해주세요.

## 1. placeholder / overlay / 임시 노드 제외

드래그 중 임시 노드가 nodes에 포함되어 있다면 validate 대상에서 제외해야 합니다.

```js
const realNodes = nodes.filter((node) => !node.isPlaceholder && !node.isOverlay);
```

## 2. Level 1 parentId 허용 범위

Level 1은 parentId가 없어야 하지만, 프로젝트에 따라 `null`, `undefined`, `''` 중 하나일 수 있습니다.

아래처럼 허용해주세요.

```js
function isEmptyParentId(parentId) {
  return parentId === null || parentId === undefined || parentId === '';
}
```

## 3. Level 2 / Level 3 parent 검증

```js
function validateVisualTree(nodes) {
  const realNodes = nodes.filter((node) => !node.isPlaceholder && !node.isOverlay);

  for (const node of realNodes) {
    if (node.level < 1 || node.level > 3) {
      return false;
    }

    if (node.level === 1) {
      if (!isEmptyParentId(node.parentId)) {
        return false;
      }
    }

    if (node.level === 2) {
      const parent = realNodes.find((item) => item.id === node.parentId);
      if (!parent || parent.level !== 1) {
        return false;
      }
    }

    if (node.level === 3) {
      const parent = realNodes.find((item) => item.id === node.parentId);
      if (!parent || parent.level !== 2) {
        return false;
      }
    }
  }

  return true;
}
```

# 중요한 요구사항

현재 로그는 Level 1, Level 2, Level 3 모두 reorder에서 발생하고 있습니다.

따라서 문제는 특정 레벨 하나가 아니라,
Visual Mode의 reorder 적용 후 구조 생성 또는 validate 로직 전체에 있습니다.

반드시 아래 기능은 유지되어야 합니다.

```txt
Level 1 → Level 1 순서 변경 가능
Level 2 → Level 2 순서 변경 가능
Level 3 → Level 3 순서 변경 가능
```

그리고 아래 제한은 유지되어야 합니다.

```txt
Level 1은 하위로 들어갈 수 없음
Level 2는 Level 1 아래에만 위치 가능
Level 3은 Level 2 아래에만 위치 가능
Level 4 생성 금지
```

# 완료 기준

| 테스트                                     | 기대 결과    |
| --------------------------------------- | -------- |
| Level 1을 다른 Level 1 앞/뒤로 이동             | 성공       |
| Level 1 이동 시 하위 Level 2/3               | 함께 이동    |
| Level 2를 같은 부모 안의 Level 2 앞/뒤로 이동       | 성공       |
| Level 2 이동 시 하위 Level 3                 | 함께 이동    |
| Level 2를 다른 Level 1 아래의 Level 2 앞/뒤로 이동 | 성공       |
| Level 3을 같은 부모 안의 Level 3 앞/뒤로 이동       | 성공       |
| Level 3을 다른 Level 2 아래의 Level 3 앞/뒤로 이동 | 성공       |
| reorder 후 validateVisualTree            | 통과       |
| reorder 후 Level 4 생성                    | 없음       |
| List Mode                               | 기존 동작 유지 |

# 최종 요약

네, Level 3 reorder도 Level 2와 같은 계열의 문제입니다.

현재는 같은 레벨끼리 reorder를 시도한 뒤,
updatedNodes의 parentId 또는 그룹 이동 구조가 깨져서
validateVisualTree에서 막히는 상태로 보입니다.

따라서 Visual Editor Mode 전용 reorder 처리에서 아래를 보정해주세요.

```txt
Level 1 reorder:
- parentId 없음 유지
- 하위 Level 2/3 함께 이동

Level 2 reorder:
- parentId는 target Level 2가 속한 Level 1로 보정
- 하위 Level 3 함께 이동

Level 3 reorder:
- parentId는 target Level 3이 속한 Level 2로 보정
```

List Mode는 수정하지 말고, Visual Editor Mode 전용 reorder 로직만 수정해주세요.

```
