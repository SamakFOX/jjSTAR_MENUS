# 비주얼모드 Level 2 드래그앤드랍 판정/프리뷰 오류 수정 요청

## 현재 문제

이전에 해결되었던 비주얼모드 드래그앤드랍이 다시 불안정해졌습니다.

특히 Level 2 메뉴를 이동할 때 문제가 발생합니다.

현재 발생하는 문제는 다음과 같습니다.

1. Level 2를 다른 Level 1 아래로 이동하려고 하면 금지 표시가 자주 뜸
2. Level 2끼리 순서 변경을 하려고 해도 금지 표시가 뜸
3. 금지 표시가 실제 마우스 위치가 아닌 엉뚱한 곳에 표시됨
4. 드롭 프리뷰에서 보이던 위치와 실제 드롭 후 위치가 다름
5. 기존 아이템이 좌우로 밀려 정렬되는 프리뷰가 부정확함
6. 드롭 위치에 따라 예상보다 뒤로 가거나 앞으로 당겨지는 문제가 있음

이 문제 때문에 비주얼모드에서 Level 2 메뉴 이동/정렬 사용성이 많이 떨어지고 있습니다.

---

## 핵심 수정 목표

비주얼모드에서 Level 2 이동/정렬이 정확하게 동작해야 합니다.

Level 2는 아래 두 가지 이동이 모두 허용되어야 합니다.

```txt
1. 같은 Level 1 안에서 Level 2 순서 변경
2. 다른 Level 1 아래로 Level 2 이동
````

따라서 Level 2를 Level 1 영역 안으로 이동하거나,
Level 2 사이에 정렬하려는 동작은 금지 처리하면 안 됩니다.

---

## Level 규칙 재정리

아래 규칙은 반드시 유지해 주세요.

```txt
Level 1 → 최상위에서만 존재 가능
Level 2 → Level 1 아래에만 존재 가능
Level 3 → Level 2 아래에만 존재 가능
```

허용되는 이동:

```txt
Level 1 ↔ Level 1 순서 변경 가능
Level 2 ↔ 같은 Level 1 내부 순서 변경 가능
Level 2 → 다른 Level 1 아래로 이동 가능
Level 3 ↔ 같은 Level 2 내부 순서 변경 가능
Level 3 → 다른 Level 2 아래로 이동 가능
```

금지되는 이동:

```txt
Level 1 → Level 2 또는 Level 3 아래로 이동 금지
Level 2 → Level 2 아래로 하위 삽입 금지
Level 2 → Level 3 아래로 이동 금지
Level 3 → Level 1 바로 아래로 이동 금지
Level 3 → Level 3 아래로 이동 금지
```

중요한 점은 다음입니다.

```txt
Level 2를 다른 Level 1 아래로 옮기는 것은 정상 허용 동작입니다.
Level 2끼리 순서를 바꾸는 것도 정상 허용 동작입니다.
```

## 금지 표시 위치 오류 수정

현재 금지 표시가 실제 마우스 위치가 아닌 다른 곳에 나타나는 문제가 있습니다.

금지 표시는 반드시 현재 마우스가 위치한 드롭 후보 영역 기준으로만 보여야 합니다.

### 잘못된 동작

```txt
마우스는 A 영역 위에 있음
그런데 금지 표시는 B 영역에 표시됨

또는

Level 2 이동이 가능한 영역인데 금지 표시가 뜸
```

### 기대 동작

```txt
마우스가 유효한 드롭존 위에 있음
→ 삽입 위치 표시

마우스가 유효하지 않은 드롭존 위에 있음
→ 해당 위치에만 금지 표시

마우스가 드롭존 밖에 있음
→ 금지 표시 제거 또는 중립 상태
```

금지 표시는 전역 상태처럼 남아 있으면 안 됩니다.
드래그 중 매 프레임 또는 dragOver 이벤트마다 현재 hover 대상 기준으로 정확히 갱신되어야 합니다.

---

## 드롭 프리뷰와 실제 결과 불일치 수정

현재 가장 큰 문제는 사용자가 보는 프리뷰와 실제 드롭 결과가 다르다는 점입니다.

예를 들어 Level 2 메뉴를 어떤 위치에 드래그했을 때,
화면에서는 특정 위치에 들어갈 것처럼 보이지만 실제 드롭하면 뒤로 가거나 앞으로 당겨집니다.

### 기대 동작

드래그 중 프리뷰에서 보이는 위치와
드롭 후 실제 반영되는 위치가 반드시 같아야 합니다.

```txt
프리뷰 위치 = 실제 드롭 결과
```

이를 위해 드래그 중에 계산하는 `previewIndex`와
드롭 시 실제 적용하는 `targetIndex`가 같은 기준을 사용해야 합니다.

## Level 2 정렬 프리뷰 기준

Level 2를 이동할 때는 기존 Level 2 아이템들이 자연스럽게 좌우 또는 상하로 밀려야 합니다.

### 기대 UI

```txt
[Level 2 A] [Level 2 B] [Level 2 C]
```

여기에서 `Level 2 X`를 B와 C 사이로 이동하면:

```txt
[Level 2 A] [Level 2 B] [Level 2 X placeholder] [Level 2 C]
```

처럼 정확한 삽입 위치가 보여야 합니다.

만약 A와 B 사이에 넣는다면:

```txt
[Level 2 A] [Level 2 X placeholder] [Level 2 B] [Level 2 C]
```

처럼 보여야 합니다.

그리고 드롭 후 실제 순서도 프리뷰와 동일해야 합니다.

---

## 드롭 위치 계산 방식 개선 요청

드롭 위치는 단순히 카드 전체 영역에 들어왔는지로 판단하지 말고,
마우스 좌표가 카드의 어느 부분에 있는지를 기준으로 계산해 주세요.

### 가로 정렬인 경우

```txt
마우스 X 좌표 < 대상 카드 중앙 X
→ 대상 카드 앞에 삽입

마우스 X 좌표 >= 대상 카드 중앙 X
→ 대상 카드 뒤에 삽입
```

### 세로 정렬인 경우

```txt
마우스 Y 좌표 < 대상 카드 중앙 Y
→ 대상 카드 앞에 삽입

마우스 Y 좌표 >= 대상 카드 중앙 Y
→ 대상 카드 뒤에 삽입
```

현재 비주얼모드가 상단 카테고리/하위 메뉴 배치처럼 가로 정렬을 많이 사용한다면,
Level 1/Level 2 영역은 X 좌표 기반 계산을 우선 적용해 주세요.

단, 실제 레이아웃이 줄바꿈되는 영역이라면 X/Y를 함께 고려해야 합니다.

## 자기 자신 이동 시 index 보정

같은 부모 안에서 아이템을 이동할 때는 기존 위치에서 제거한 뒤 삽입해야 하므로 index 보정이 필요합니다.

예시:

```txt
기존 순서: A B C D
B를 D 뒤로 이동
```

처리 순서:

```txt
1. B 제거 → A C D
2. D 뒤에 B 삽입 → A C D B
```

이때 제거 전 index와 삽입 index를 그대로 쓰면
한 칸 밀리는 문제가 생길 수 있습니다.

따라서 같은 parent 안에서 이동하는 경우에는
sourceIndex와 targetIndex 관계를 고려해 보정해 주세요.

예시:

```js
if (sourceParentId === targetParentId && sourceIndex < targetIndex) {
  adjustedTargetIndex = targetIndex - 1;
}
```

단, 이 보정은 실제 구현 구조에 맞게 적용해 주세요.

중요한 것은 다음입니다.

```txt
드래그 프리뷰 계산과 실제 드롭 계산이 같은 index 보정 규칙을 사용해야 합니다.
```

## Level 2의 부모 변경 처리

Level 2를 다른 Level 1 아래로 이동하는 경우는 허용되어야 합니다.

예시:

```txt
취업지원 > 취업정보
```

에 있던 Level 2를

```txt
정보공유
```

아래로 옮기는 것은 가능해야 합니다.

이때 처리 기준:

```txt
dragItem.level === 2
targetParent.level === 1
→ 허용
```

금지하면 안 되는 경우:

```txt
Level 2를 Level 1 그룹 안의 빈 공간에 드롭
Level 2를 다른 Level 1의 Level 2 목록 사이에 드롭
Level 2를 다른 Level 1의 마지막 위치에 드롭
```

이 세 경우는 모두 정상 허용입니다.

## Level 2가 Level 2 아래로 들어가는 것은 금지

단, Level 2가 다른 Level 2의 하위 메뉴가 되는 것은 금지입니다.

즉, 아래는 금지입니다.

```txt
Level 2 A
  └ Level 2 B
```

하지만 아래는 허용입니다.

```txt
Level 1 A
  ├ Level 2 A
  └ Level 2 B
```

따라서 드롭존을 구분해야 합니다.

```txt
Level 2 카드 사이 삽입 영역 → 허용
Level 2 카드 내부 하위 삽입 영역 → 금지
```

현재 문제는 이 두 영역이 혼동되어
Level 2 정렬도 하위 삽입으로 오판되는 것 같습니다.

## 드롭존 구분 요청

비주얼모드에서 드롭존을 아래처럼 명확히 분리해 주세요.

### 1. 정렬 드롭존

같은 레벨의 앞/뒤/사이에 넣는 영역입니다.

```txt
Level 2 사이 영역
→ Level 2 순서 변경 또는 다른 Level 1으로 이동 허용
```

### 2. 하위 삽입 드롭존

특정 부모 안으로 넣는 영역입니다.

```txt
Level 1 내부 영역
→ Level 2 삽입 허용

Level 2 내부 영역
→ Level 3 삽입 허용
```

### 3. 금지 드롭존

레벨 규칙에 맞지 않는 영역입니다.

```txt
Level 2 내부 영역에 Level 2 삽입 시도
→ 금지

Level 1 내부 영역에 Level 3 바로 삽입 시도
→ 금지
```

## 드래그 중 시각 피드백

드래그 중에는 다음 피드백을 명확히 보여주세요.

### 허용 위치

```txt
삽입 라인 또는 placeholder 표시
기존 아이템들이 밀려난 상태로 표시
```

### 금지 위치

```txt
마우스 위치 근처의 해당 드롭존에만 금지 표시
다른 위치에 금지 표시가 남지 않음
```

### 드롭존 밖

```txt
표시 제거
또는 원래 위치 유지
```

## Swapy 사용 관련 주의

만약 Swapy를 적용했거나 검토 중이라면,
Swapy의 swap 결과를 그대로 메뉴 구조에 반영하지 말고 반드시 기존 Level 규칙 검증을 통과한 경우에만 반영해 주세요.

Swapy는 슬롯 기반 교체에는 적합하지만,
현재 메뉴 편집기는 Level 1 / Level 2 / Level 3 계층 구조가 있으므로
slot 간 교체 결과를 그대로 믿으면 안 됩니다.

반드시 다음 단계를 거쳐야 합니다.

```txt
1. drag item 확인
2. hover/drop target 확인
3. source parent 확인
4. target parent 확인
5. target index 계산
6. Level 규칙 검증
7. preview 상태 갱신
8. drop 시 commitMenuChange(nextMenuState) 호출
```

## 히스토리 연동 주의

드롭이 실제로 성공한 경우에만 히스토리에 1회 저장해 주세요.

```txt
허용된 이동 성공 → commitMenuChange(nextMenuState) 1회 호출
금지 위치 드롭 → 히스토리 변화 없음
드래그 취소 → 히스토리 변화 없음
프리뷰 변경 → 히스토리 변화 없음
```

비주얼모드 프리뷰가 바뀔 때마다 히스토리가 증가하면 안 됩니다.

## 핵심 요약

현재 비주얼모드에서 Level 2 이동/정렬이 잘못 금지 처리되고,
금지 표시 위치와 드롭 결과가 부정확합니다.

수정 핵심은 다음입니다.

```txt
1. Level 2 → 다른 Level 1 아래 이동은 허용
2. Level 2 ↔ Level 2 순서 변경은 허용
3. Level 2 → Level 2 하위 삽입만 금지
4. 금지 표시는 실제 마우스 위치의 드롭존에만 표시
5. 드래그 프리뷰와 실제 드롭 결과가 반드시 일치
6. 기존 아이템이 삽입 위치 기준으로 밀려 보이도록 처리
7. 성공한 드롭만 commitMenuChange로 히스토리 1회 저장
```
