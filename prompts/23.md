# 인증코드 방식 로그인 + 최종 제출 저장 구현 요청

## 목표

현재 프로젝트는 Vercel에 배포할 예정입니다.

별도 Express/AWS 서버는 사용하지 않고,  
**Next.js / Vercel API Route + Supabase** 조합으로 인증코드 기반 로그인을 구현해주세요.

회원가입, 이메일 로그인, OAuth는 사용하지 않습니다.

테스터에게 미리 발급한 인증코드를 입력하면 메뉴 편집 화면에 입장할 수 있고,  
최종 제출 시 현재 메뉴 구조를 Supabase에 저장하는 방식입니다.

---

# 현재 Supabase 준비 상태

Supabase 프로젝트는 이미 생성되어 있습니다.

아래 테이블도 이미 생성되어 있습니다.

---

## 1. auth_codes 테이블

인증코드 검증용 테이블입니다.

```txt
auth_codes
```

컬럼 구조:

| 컬럼명 | 타입 | 설명 |
|---|---|---|
| id | uuid | primary key |
| code | text | 인증코드 |
| label | text | 테스터 구분용 |
| is_active | bool | 사용 가능 여부 |
| max_submit_count | int8 | 최대 제출 횟수 |
| submit_count | int8 | 현재 제출 횟수 |
| expires_at | timestamptz | 만료일 |
| created_at | timestamptz | 생성일 |
| memo | text | 메모 |

현재 테스트용 인증코드도 1개 이상 입력되어 있습니다.

인증코드는 아래 형식입니다.

```txt
JJ+학번
```

예시:

```txt
JJ202662001
JJ202612004
JJ202031024
```

---

## 2. submissions 테이블

최종 제출 저장용 테이블입니다.

```txt
submissions
```

컬럼 구조:

| 컬럼명 | 타입 | 설명 |
|---|---|---|
| id | uuid | primary key |
| code | text | 인증코드 |
| user_name | text | 사용자 이름 |
| menu_data | jsonb | 최종 메뉴 구조 |
| change_log | jsonb | 변경 로그 |
| submitted_at | timestamptz | 제출 시간 |
| user_agent | text | 브라우저 정보 |

---

## 3. RLS 상태

아래 두 테이블 모두 RLS를 켰습니다.

```txt
auth_codes: RLS enabled
submissions: RLS enabled
```

별도 public policy는 만들지 않았습니다.

따라서 프론트에서 Supabase를 직접 조회하거나 저장하면 안 됩니다.

반드시 Next.js API Route에서 Supabase Secret Key를 사용해서 접근해야 합니다.

# 2. /api/verify-code 구현

인증코드를 검증하는 API를 구현해주세요.

예상 파일:

```txt
src/app/api/verify-code/route.js
```

## 요청

```json
{
  "code": "JJ202600001",
  "userName": "홍길동"
}
```

## 처리 로직

```txt
1. code와 userName이 비어 있는지 확인
2. auth_codes에서 code 조회
3. code가 없으면 실패
4. is_active가 false면 실패
5. expires_at이 현재 시간보다 이전이면 실패
6. submit_count가 max_submit_count 이상이면 실패
7. 성공이면 ok: true 반환
```

## 응답 예시

성공:

```json
{
  "ok": true,
  "message": "인증되었습니다.",
  "code": "JJ202612345",
  "label": "테스터 1"
}
```

실패:

```json
{
  "ok": false,
  "message": "유효하지 않은 인증코드입니다."
}
```

## 구현 예시

```js
import { NextResponse } from 'next/server';
import { supabaseAdmin } from '@/lib/supabaseAdmin';

export async function POST(request) {
  try {
    const body = await request.json();
    const code = String(body.code || '').trim();
    const userName = String(body.userName || '').trim();

    if (!code || !userName) {
      return NextResponse.json(
        { ok: false, message: '인증코드와 이름을 입력해주세요.' },
        { status: 400 }
      );
    }

    const { data, error } = await supabaseAdmin
      .from('auth_codes')
      .select('code, label, is_active, expires_at, submit_count, max_submit_count')
      .eq('code', code)
      .single();

    if (error || !data) {
      return NextResponse.json(
        { ok: false, message: '유효하지 않은 인증코드입니다.' },
        { status: 401 }
      );
    }

    if (!data.is_active) {
      return NextResponse.json(
        { ok: false, message: '비활성화된 인증코드입니다.' },
        { status: 403 }
      );
    }

    if (data.expires_at && new Date(data.expires_at) < new Date()) {
      return NextResponse.json(
        { ok: false, message: '만료된 인증코드입니다.' },
        { status: 403 }
      );
    }

    const submitCount = Number(data.submit_count || 0);
    const maxSubmitCount = Number(data.max_submit_count || 0);

    if (maxSubmitCount > 0 && submitCount >= maxSubmitCount) {
      return NextResponse.json(
        { ok: false, message: '제출 가능 횟수를 초과한 인증코드입니다.' },
        { status: 403 }
      );
    }

    return NextResponse.json({
      ok: true,
      message: '인증되었습니다.',
      code: data.code,
      label: data.label,
    });
  } catch (error) {
    console.error('[verify-code] error:', error);

    return NextResponse.json(
      { ok: false, message: '인증 처리 중 오류가 발생했습니다.' },
      { status: 500 }
    );
  }
}
```

# 5. changeLog state 추가

현재 `history`, `lastAction`은 되돌리기와 최근 작업 표시용입니다.

제출용 변경 로그로는 부족하므로 부모 컴포넌트에 별도의 `changeLog` state를 추가해주세요.

```js
const [changeLog, setChangeLog] = useState([]);
```

메뉴가 변경될 때마다 아래 형태로 로그를 추가합니다.

```js
{
  type: 'reorder' | 'move' | 'rename' | 'add' | 'delete' | 'reset' | 'submit',
  message: '학사관리 메뉴 순서가 변경되었습니다.',
  createdAt: new Date().toISOString()
}
```

가능하면 기존 `updateTree(newTree, actionDescription)`가 호출되는 지점에서  
`actionDescription`을 `changeLog`에도 누적해주세요.

예시:

```js
function addChangeLog(type, message) {
  setChangeLog(prev => [
    ...prev,
    {
      type,
      message,
      createdAt: new Date().toISOString(),
    }
  ]);
}
```

# 6. /api/submit 구현

최종 제출 API를 구현해주세요.

예상 파일:

```txt
src/app/api/submit/route.js
```

## 요청

```json
{
  "code": "JJ202600001",
  "userName": "홍길동",
  "menuData": [],
  "changeLog": []
}
```

## 처리 로직

```txt
1. code, userName, menuData 확인
2. auth_codes에서 code 재검증
3. is_active 확인
4. expires_at 확인
5. submit_count / max_submit_count 확인
6. submissions에 제출 데이터 저장
7. auth_codes.submit_count 1 증가
8. 성공 응답 반환
```

## 저장 데이터

```json
{
  "code": "JJ202600001",
  "user_name": "홍길동",
  "menu_data": [],
  "change_log": [],
  "submitted_at": "now",
  "user_agent": "browser user agent"
}
```

## 구현 예시

```js
import { NextResponse } from 'next/server';
import { supabaseAdmin } from '@/lib/supabaseAdmin';

export async function POST(request) {
  try {
    const body = await request.json();

    const code = String(body.code || '').trim();
    const userName = String(body.userName || '').trim();
    const menuData = body.menuData;
    const changeLog = Array.isArray(body.changeLog) ? body.changeLog : [];

    if (!code || !userName || !menuData) {
      return NextResponse.json(
        { ok: false, message: '제출 정보가 부족합니다.' },
        { status: 400 }
      );
    }

    const { data: authCode, error: authError } = await supabaseAdmin
      .from('auth_codes')
      .select('code, is_active, expires_at, submit_count, max_submit_count')
      .eq('code', code)
      .single();

    if (authError || !authCode) {
      return NextResponse.json(
        { ok: false, message: '유효하지 않은 인증코드입니다.' },
        { status: 401 }
      );
    }

    if (!authCode.is_active) {
      return NextResponse.json(
        { ok: false, message: '비활성화된 인증코드입니다.' },
        { status: 403 }
      );
    }

    if (authCode.expires_at && new Date(authCode.expires_at) < new Date()) {
      return NextResponse.json(
        { ok: false, message: '만료된 인증코드입니다.' },
        { status: 403 }
      );
    }

    const submitCount = Number(authCode.submit_count || 0);
    const maxSubmitCount = Number(authCode.max_submit_count || 0);

    if (maxSubmitCount > 0 && submitCount >= maxSubmitCount) {
      return NextResponse.json(
        { ok: false, message: '제출 가능 횟수를 초과했습니다.' },
        { status: 403 }
      );
    }

    const userAgent = request.headers.get('user-agent') || '';

    const { error: insertError } = await supabaseAdmin
      .from('submissions')
      .insert({
        code,
        user_name: userName,
        menu_data: menuData,
        change_log: changeLog,
        user_agent: userAgent,
      });

    if (insertError) {
      console.error('[submit] insert error:', insertError);

      return NextResponse.json(
        { ok: false, message: '제출 저장 중 오류가 발생했습니다.' },
        { status: 500 }
      );
    }

    const { error: updateError } = await supabaseAdmin
      .from('auth_codes')
      .update({
        submit_count: submitCount + 1,
      })
      .eq('code', code);

    if (updateError) {
      console.error('[submit] submit_count update error:', updateError);
    }

    return NextResponse.json({
      ok: true,
      message: '최종 제출되었습니다.',
    });
  } catch (error) {
    console.error('[submit] error:', error);

    return NextResponse.json(
      { ok: false, message: '제출 처리 중 오류가 발생했습니다.' },
      { status: 500 }
    );
  }
}
```

# 7. EditToolbar 최종 제출 연결

`EditToolbar`에는 이미 `onSubmit` prop이 있습니다.

최종 제출 버튼은 아래처럼 연결되어 있습니다.

```jsx
<button onClick={onSubmit}>
  <Save size={18} />
  <span>최종 제출</span>
</button>
```

부모 컴포넌트에서 `handleSubmit`을 만들고 `onSubmit`으로 전달해주세요.

예시:

```js
async function handleSubmit() {
  const code = sessionStorage.getItem('jjstar_auth_code');
  const userName = sessionStorage.getItem('jjstar_user_name');

  if (!code || !userName) {
    alert('로그인이 필요합니다.');
    return;
  }

  if (!confirm('현재 메뉴 배치를 최종 제출하시겠습니까?')) {
    return;
  }

  const res = await fetch('/api/submit', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      code,
      userName,
      menuData: menuTree,
      changeLog,
    }),
  });

  const result = await res.json();

  if (!result.ok) {
    alert(result.message || '제출에 실패했습니다.');
    return;
  }

  alert('최종 제출되었습니다.');
}
```

# 8. 제출 정책

같은 인증코드로 여러 번 제출할 수 있습니다.

관리자 입장에서는 같은 `code`의 제출 중  
`submitted_at`이 가장 최신인 데이터를 최종 제출본으로 간주합니다.

즉, 제출할 때 기존 row를 update하지 말고,  
`submissions`에 새 row를 insert해주세요.

```txt
같은 code로 여러 번 제출 가능
각 제출은 submissions에 새 row로 저장
최종본은 submitted_at 최신 데이터
```

# 9. 나중에 제출 결과 읽어오기 기준

이번 구현의 핵심은 저장까지입니다.

추후 관리자/결과 조회 화면을 만들 때는 아래 기준으로 조회할 예정입니다.

```txt
submissions에서 code별 submitted_at 최신 데이터 조회
menu_data를 다시 menuTree로 넣어서 화면에 표시
change_log도 함께 표시 가능
```

읽어온 데이터는 아래처럼 복원할 수 있어야 합니다.

```js
setMenuTree(submission.menu_data);
setChangeLog(submission.change_log || []);
```

따라서 `menu_data`에는 반드시 현재 `menuTree` 전체 구조를 그대로 저장해주세요.

```txt
인증코드 로그인
로그인 상태 유지
최종 제출 저장
submit_count 증가
```

임시저장/재개 기능은 2차 작업에서 구현할 예정입니다.

이번 작업에서 임시저장까지 무리하게 넣지 마세요.

# 최종 요약

이번 작업은 인증코드 기반 로그인과 최종 제출 저장 기능을 추가하는 작업입니다.

핵심 흐름:

```txt
인증코드 입력
→ /api/verify-code 호출
→ Supabase auth_codes 검증
→ 성공 시 sessionStorage 저장
→ 메뉴 편집 화면 입장
→ 최종 제출
→ /api/submit 호출
→ submissions에 현재 menuTree 저장
→ auth_codes.submit_count 증가
```

중요:

```txt
실제 Supabase Secret Key는 코드에 직접 넣지 않습니다.
SUPABASE_SECRET_KEY 환경변수를 API Route에서만 사용합니다.
프론트 컴포넌트에서 Supabase Secret Key를 사용하면 안 됩니다.
기존 메뉴 편집 기능은 깨지면 안 됩니다.
```
