# 가이드 단계별 버그 수정 요청

## 작업 목표

현재 가이드가 전반적으로 구현되어 있지만, 일부 단계에서 아래 문제가 발생합니다.

```txt
1. Step 2에서 설명 대상이 아닌 아래 아이템 영역이 활성화되어 버튼을 가림
2. Step 4의 아이콘별 설명 형식이 다시 깨짐
3. Step 7에서 검은 tooltip만 꺼져야 하는데 가이드 설명창 자체가 꺼짐
4. Step 11/12 최종 제출 화면에서 실제 제출 버튼이 눌려 최종 제출될 위험이 있음
````

이번 수정은 **가이드 표시/동작 버그 수정만** 대상으로 합니다.

---

## 가장 중요한 조건

기존 기능은 건드리지 말아 주세요.

```txt
로그인/인증
메뉴 편집 기능
리스트모드 드래그앤드랍
비주얼모드 드래그앤드랍
임시저장/자동저장
미리보기
최종 제출 저장 로직
변경 로그 저장
툴팁(desc) 데이터 구조
undo/redo
초기화
관리자 화면
Supabase 저장 구조
```

가이드 중에는 실제 데이터가 바뀌면 안 됩니다.

```txt
가이드 표시만으로 menuState 변경 금지
가이드 표시만으로 commitMenuChange 호출 금지
가이드 표시만으로 undo/redo 증가 금지
가이드 표시만으로 change_log 추가 금지
가이드 표시만으로 임시저장/자동저장 실행 금지
가이드 중 최종 제출 버튼 클릭으로 실제 제출 금지
```

# 1. Step 2 하이라이트 대상 오류 수정

## 현재 문제

Step 2는 `편집 화면 기본 UI` 설명 단계인데,
상단 기본 UI가 아니라 아래 메뉴 아이템 영역이 활성화되어 있습니다.

현재 증상:

```txt
Step 2 / 12
제목: 편집 화면 기본 UI
설명 대상: 상단 편집 UI여야 함
실제 하이라이트: 아래 메뉴 아이템 / 편집 버튼 영역
결과: 아래 버튼이 가려져 클릭이 어려움
```

---

## 수정 요청

Step 2에서는 아래 영역만 하이라이트되어야 합니다.

```txt
편집 완료
리스트 편집
화면에서 편집
초기화
```

즉, 대상은 상단 좌측 편집 기본 UI 영역입니다.

아래 메뉴 아이템, 드래그 핸들, 아이템별 편집 버튼 영역은 Step 2에서 하이라이트되면 안 됩니다.

---

## Step 2 target 지정

Step 2의 target을 명확히 아래처럼 지정해 주세요.

```txt
data-guide-id="edit-basic-toolbar"
```

예시:

```jsx
<div data-guide-id="edit-basic-toolbar">
  <button>편집 완료</button>
  <button>리스트 편집</button>
  <button>화면에서 편집</button>
  <button>초기화</button>
</div>
```

Step 2에서는 반드시 이 target만 사용해 주세요.


즉, 가이드 설명창 안에 실제 버튼 아이콘과 동일하거나 유사한 아이콘을 함께 보여주세요.

# 4. Step 7 hover tooltip 처리 수정

## 현재 문제

Step 7 `메뉴 설명 확인` 단계에서
검은 tooltip이 가이드 설명창에 가려지는 문제를 수정하려고 했는데,
현재는 tooltip이 뜰 때 **가이드 설명창 자체가 꺼져버립니다.**

원했던 것은 이것이 아닙니다.

---

## 원하는 동작

Step 7에서는 가이드 설명창은 유지되어야 합니다.

다만, 검은 desc tooltip이 설명창에 가려질 경우
**검은 tooltip만 숨기거나 위치를 조정**해야 합니다.

즉:

```txt
가이드 설명창 유지
검은 desc tooltip은 가이드와 겹치면 숨김 또는 위치 조정
```

---

## 수정 요청

Step 7에서 아래처럼 처리하지 말아 주세요.

```js
if (isDemoTooltipVisible) {
  hideGuidePopover = true;
}
```

이렇게 하면 가이드 설명창이 사라져 사용자가 무엇을 해야 하는지 모릅니다.

---

## 올바른 처리

### 방식 A 권장: 가이드 중에는 검은 desc tooltip만 숨김

Step 7에서는 실제 검은 tooltip 대신,
가이드 내부 또는 별도 가이드 전용 tooltip을 사용해도 됩니다.

```txt
가이드 설명창은 유지
기존 검은 desc tooltip은 숨김
가이드용 예시 tooltip은 target 근처에 표시
```

예시:

```js
const shouldSuppressRealDescTooltip =
  guideActive &&
  currentGuideStep?.id === "menu-tooltip";
```

기존 desc tooltip 컴포넌트에서:

```jsx
{!shouldSuppressRealDescTooltip && isTooltipVisible && (
  <DescTooltip />
)}
```

---

### 방식 B: 검은 desc tooltip 위치 이동

검은 tooltip을 유지하고 싶다면
가이드 설명창과 겹치지 않는 위치로 이동해 주세요.

```txt
가이드 설명창: 위쪽
검은 desc tooltip: 메뉴 아래쪽
둘이 겹치면 검은 desc tooltip을 다른 방향으로 이동
```

하지만 초기 구현은 방식 A가 더 안정적입니다.

---

## Step 7 기대 결과

```txt
가이드 설명창은 계속 보임
메뉴명 hover 예시가 보임
검은 tooltip이 설명창을 가리지 않음
사용자가 "메뉴 위에 올리면 설명을 볼 수 있다"는 점을 이해함
```

# 5. Step 11 / Step 12 최종 제출 화면 실제 제출 방지

## 현재 문제

Step 11에서 의견을 입력하고 Step 12로 넘어가면,
최종 제출 화면의 실제 `제출` 버튼이 눌릴 수 있는 상태입니다.

이 상태에서 사용자가 제출 버튼을 누르면 실제 최종 제출이 되어버릴 수 있습니다.

가이드 중에는 절대 실제 제출이 발생하면 안 됩니다.

---

## 수정 요청

가이드가 활성화된 상태에서는 최종 제출 화면의 실제 `제출` 버튼과 `취소` 버튼이
기존 submit/cancel 동작을 실행하지 않도록 막아 주세요.

---

## 적용 조건

```txt
guideActive === true
AND currentGuideStep is submit-related step
```

이때는 실제 submit/cancel 동작을 막습니다.

---

## 원하는 동작

가이드 중 최종 제출 화면에서 사용자가 `제출` 또는 `취소` 버튼을 누르면:

```txt
1. 실제 제출 또는 취소 실행하지 않음
2. "가이드 완료" 안내 팝업 또는 토스트를 2초간 표시
3. 2초 후 가이드 종료
4. 처음 로그인한 기본 화면으로 복귀
```

---

## 안내 문구 예시

```txt
가이드가 완료되었습니다.
이제 실제 메뉴 편집을 시작해 보세요.
```

또는:

```txt
가이드 완료!
처음 화면으로 돌아갑니다.
```

---

## 구현 예시

```js
function handleSubmitButtonClick(e) {
  if (guideActive && isSubmitGuideStep) {
    e.preventDefault();
    e.stopPropagation();

    showGuideCompleteToast();

    setTimeout(() => {
      finishGuideAndReturnHome();
    }, 2000);

    return;
  }

  handleRealSubmit();
}
```

취소 버튼도 동일하게:

```js
function handleCancelButtonClick(e) {
  if (guideActive && isSubmitGuideStep) {
    e.preventDefault();
    e.stopPropagation();

    showGuideCompleteToast();

    setTimeout(() => {
      finishGuideAndReturnHome();
    }, 2000);

    return;
  }

  handleRealCancel();
}
```

---

## submit button disabled 여부

가이드 중에 버튼을 완전히 disabled 처리하면 사용자가 클릭해 볼 수 없으므로,
disabled보다는 클릭 이벤트를 가이드 완료 처리로 가로채는 방식을 권장합니다.

```txt
버튼은 클릭 가능해 보임
하지만 실제 제출은 실행하지 않음
클릭하면 가이드 완료 처리
```

# 6. 가이드 완료 후 복귀 처리

## 현재 문제

가이드 완료 후 최종 제출 화면에 남아 있으면 사용자가 실제 작업을 시작하기 어렵습니다.

## 수정 요청

가이드 완료 시 반드시 처음 로그인했을 때 상태로 돌아가게 해 주세요.

```txt
가이드 완료
→ guideActive false
→ guideStep 초기화
→ submit 화면 종료
→ edit mode 종료
→ 로그인 직후 기본 화면 표시
```

---

## 복귀 함수 예시

```js
function finishGuideAndReturnHome() {
  setGuideActive(false);
  setGuideStep(0);

  setSubmitStep("edit"); // 또는 실제 구조에 맞게 submit 화면 종료
  setIsEditMode(false);
  setEditMode("list"); // 필요 시 초기값
  setShowPreview(false);

  setGuideCompleteToastVisible(false);
}
```

실제 상태 변수명은 프로젝트 구조에 맞게 조정해 주세요.
