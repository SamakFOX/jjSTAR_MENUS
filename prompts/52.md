# 가이드 툴팁 표시 및 조건 충족 UI 개선 요청

## 작업 목표

현재 가이드에서 두 가지 UX 문제가 있습니다.

1. 메뉴 hover 툴팁 단계에서 검은 툴팁이 가이드 설명창에 가려집니다.
2. 조건 충족 시 표시되는 “좋아요...” 안내 문구가 눈에 잘 띄지 않고, 완료 상태처럼 보이지 않습니다.

따라서 아래 두 가지를 수정해 주세요.

```txt
1. 툴팁 시연 중에는 가이드 설명창이 툴팁을 가리지 않도록 처리
2. 조건 충족 안내는 파란 배경 + 체크 아이콘 형태로 변경
````

---

## 가장 중요한 조건

기존 기능은 건드리지 말아 주세요.

```txt
메뉴 편집 기능
리스트모드 드래그앤드랍
비주얼모드 드래그앤드랍
임시저장/자동저장
최종 제출
변경 로그
툴팁(desc) 데이터 구조
가이드 단계 흐름
```

이번 수정은 **가이드 표시 UI만 개선**하는 작업입니다.

---

# 1. 툴팁 시연 중 가이드 설명창 가림 문제 수정

## 현재 문제

메뉴 hover 툴팁 가이드 단계에서 검은 툴팁이 표시되는데,
가이드 설명창이 그 툴팁을 가려서 실제 툴팁 내용을 보기 어렵습니다.

현재 상태:

```txt
메뉴명 hover
→ 검은 desc 툴팁 표시
→ 가이드 설명창이 툴팁 위를 덮음
→ 툴팁 일부가 가려짐
```

---

## 수정 요청

메뉴 hover 툴팁이 나오는 단계에서는
검은 툴팁과 가이드 설명창이 겹치지 않도록 처리해 주세요.

---

## 우선 적용 방식

툴팁이 표시되는 동안에는 가이드 설명창을 잠시 숨기거나,
툴팁과 겹치지 않는 위치로 이동해 주세요.

권장 방식은 아래입니다.

```txt
툴팁 시연 전: 가이드 설명창 표시
포인터가 메뉴명으로 이동
툴팁 표시 시점: 가이드 설명창 숨김 또는 위쪽으로 이동
툴팁 확인 후: 가이드 설명창 다시 표시
```

---

## 방식 A: 툴팁 표시 중 설명창 숨김

가장 단순하고 확실한 방식입니다.

```txt
desc tooltip visible === true
→ guide popover temporarily hidden
```

예시:

```js
const shouldHideGuidePopover =
  currentGuideStep?.id === "menu-tooltip" &&
  isDemoTooltipVisible;
```

```jsx
{guideActive && !shouldHideGuidePopover && (
  <GuidePopover />
)}
```

이 경우 검은 툴팁이 가려지지 않습니다.

---

## 방식 B: 툴팁 표시 중 설명창 위치 이동

설명창을 완전히 숨기고 싶지 않다면,
툴팁이 나올 때 설명창을 대상 요소 위쪽이나 멀리 떨어진 위치로 이동해 주세요.

```txt
툴팁은 메뉴명 아래
가이드 설명창은 메뉴명 위
```

단, 화면이 좁으면 다시 겹칠 수 있으므로
초기 구현은 **방식 A: 툴팁 표시 중 설명창 숨김**을 권장합니다.

---

## 적용 대상

이 처리는 모든 가이드 단계에 적용할 필요는 없습니다.

아래 단계에만 적용해 주세요.

```txt
메뉴 설명 확인 / hover 툴팁 가이드 단계
```

예상 step id 예시:

```txt
menu-tooltip
tooltip-guide
desc-tooltip
```

실제 코드의 step id에 맞춰 적용해 주세요.

---

## 기대 동작

```txt
1. 가이드 설명창에서 “메뉴 위에 마우스를 올리면 설명을 볼 수 있어요” 안내
2. 포인터 애니메이션이 메뉴명으로 이동
3. 메뉴명이 hover 상태가 됨
4. 검은 desc 툴팁 표시
5. 이때 가이드 설명창은 잠시 사라지거나 툴팁을 가리지 않는 위치로 이동
6. 툴팁 내용이 명확히 보임
7. 툴팁 확인 완료 후 다음 단계 진행 가능
```

# 2. 조건 충족 안내 UI 변경

## 현재 문제

조건 충족 시 아래처럼 문구가 표시됩니다.

```txt
좋아요. 각 아이콘 위에 올리면 기능 이름도 확인할 수 있어요.
```

또는:

```txt
좋아요. 어려운 메뉴는 이렇게 설명을 확인하면 됩니다.
```

이 방식은 “조건이 완료되었다”는 느낌이 약합니다.

---

## 수정 요청

조건 충족 시에는 단순 문구 대신
**파란 배경 + 체크 아이콘**이 있는 완료 표시로 바꿔 주세요.

---

## 원하는 UI 방향

조건 충족 전:

```txt
먼저 강조된 영역에 마우스를 올려보세요.
```

조건 충족 후:

```txt
✓ 확인 완료
```

또는:

```txt
✓ 기능을 확인했어요.
```

---

## 스타일 예시

```txt
배경: 파란색
글자: 흰색
아이콘: 체크
위치: div 앞 또는 오른쪽 끝
```

예시 형태:

```txt
[✓] 기능을 확인했어요.
```

또는 도장처럼:

```txt
기능을 확인했어요.                         [✓]
```

---

## 추천 UI

가이드 설명창 안에 아래처럼 표시해 주세요.

```txt
┌──────────────────────────────┐
│ ✓ 확인 완료                  │
└──────────────────────────────┘
```

또는:

```txt
┌──────────────────────────────┐
│ 기능을 확인했어요.        ✓  │
└──────────────────────────────┘
```

---

.guide-complete-message {

```

---

## JSX 예시

```jsx
{stepCompleted ? (
  <div className="guide-complete-message">
    <span>기능을 확인했어요.</span>
    <span className="check-icon">✓</span>
  </div>
) : (
  <div className="guide-action-hint">
    강조된 영역에 마우스를 올려보세요.
  </div>
)}
```

# 3. 단계별 완료 문구 예시

## 핸들 단계

조건 충족 전:

```txt
왼쪽 핸들을 눌러보세요.
```

조건 충족 후:

```txt
✓ 핸들 조작을 확인했어요.
```

---

## 아이템별 편집 버튼 단계

조건 충족 전:

```txt
각 아이콘 위에 마우스를 올려보세요.
```

조건 충족 후:

```txt
✓ 편집 버튼 기능을 확인했어요.
```

---

## 메뉴 툴팁 단계

조건 충족 전:

```txt
메뉴명 위에 마우스를 올려 설명을 확인해 보세요.
```

조건 충족 후:

```txt
✓ 메뉴 설명을 확인했어요.
```

---

## 비주얼모드 단계

조건 충족 전:

```txt
비주얼모드의 핸들을 확인해 보세요.
```

조건 충족 후:

```txt
✓ 비주얼모드 조작 방식을 확인했어요.
```

---

# 4. 버튼 활성화 방식

조건 충족이 필요한 단계에서는 기존처럼 조건 충족 전에는 `다음` 버튼을 비활성화해도 됩니다.

조건이 충족되면:

```txt
완료 표시가 파란색으로 변경
체크 아이콘 표시
다음 버튼 활성화
```

---

## 동작 예시

```txt
조건 미충족
→ 안내 박스: 회색/연한 파랑
→ 다음 버튼 비활성화

조건 충족
→ 안내 박스: 파란 배경
→ 체크 아이콘 표시
→ 다음 버튼 활성화
```

---

# 5. 기존 “좋아요” 문구 제거

아래와 같은 문구는 제거하거나 완료 표시로 대체해 주세요.

```txt
좋아요. 각 아이콘 위에 올리면 기능 이름도 확인할 수 있어요.
좋아요. 어려운 메뉴는 이렇게 설명을 확인하면 됩니다.
좋아요. 이 핸들을 잡고 움직이면 됩니다.
```

대신 아래처럼 표시해 주세요.

```txt
✓ 편집 버튼 기능을 확인했어요.
✓ 메뉴 설명을 확인했어요.
✓ 핸들 조작을 확인했어요.
```

# 6. 최종 기대 결과

| 항목       | 기존             | 수정 후              |
| -------- | -------------- | ----------------- |
| 툴팁 단계    | 검은 툴팁이 설명창에 가림 | 툴팁 표시 중 설명창 숨김/이동 |
| 조건 충족 표시 | “좋아요...” 문구    | 파란 완료 박스 + 체크 아이콘 |
| 다음 버튼    | 조건 충족 후 활성화    | 기존 유지             |
| 사용자 인식   | 설명을 읽고 넘기는 느낌  | 직접 확인 완료된 느낌      |
| 기존 기능    | 유지             | 유지                |

# 7. 핵심 요약

이번 수정은 두 가지입니다.

```txt
1. 메뉴 hover 툴팁 가이드 단계에서 검은 desc 툴팁이 가이드 설명창에 가리지 않게 하기
2. 조건 충족 시 “좋아요” 문구 대신 파란 배경 + 체크 아이콘 완료 표시로 바꾸기
```

특히 툴팁 단계에서는:

```txt
desc tooltip visible → guide popover hide 또는 위치 이동
```

조건 충족 단계에서는:

```txt
미충족: 안내 문구
충족: 파란 완료 박스 + 체크 아이콘
```

으로 처리해 주세요.
