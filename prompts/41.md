# 최종 제출 확인 화면 UI 수정 및 intention 컬럼 저장 연동 요청

## 현재 상태

Supabase `submissions` 테이블에 아래 컬럼을 추가했습니다.

```txt
intention text
````

현재 화면에서는 최종 제출 확인 화면에 메뉴 구성 의도를 입력하는 textarea가 있습니다.

이제 이 textarea에 입력한 값을 최종 제출 시 `submissions.intention` 컬럼에 함께 저장되도록 수정해 주세요.

---

## 작업 목표

이번 작업은 최종 제출 확인 화면과 제출 저장 payload만 수정합니다.

수정 목표는 다음과 같습니다.

```txt
1. 상단 PC 화면 / 모바일 화면 탭 제거
2. 상단 돌아가기 버튼 제거
3. 기존 우측 하단 제출 버튼 제거
4. 하단에 취소 / 제출 버튼 추가
5. 제출 버튼은 파란 배경 + 흰 글씨로 강조
6. textarea에 입력한 메뉴 구성 의도를 submissions.intention 컬럼에 저장
7. 제출 성공 시 제출 완료 화면으로 이동
```

---

다른 기능은 현재 잘 동작하고 있으므로 건드리지 말아 주세요.

```txt
로그인/인증 기능
리스트모드

```

이번 수정 범위는 오직 다음입니다.

```txt
최종 제출 확인 화면 UI
최종 제출 payload에 intention 추가
```

# 1. PC 화면 / 모바일 화면 탭 제거

현재 최종 제출 확인 화면 상단에 아래 탭이 있습니다.

```txt
PC 화면
모바일 화면
```

이 화면에서는 필요하지 않으므로 제거해 주세요.

---

# 2. 상단 돌아가기 버튼 제거

현재 우측 상단에 `돌아가기` 버튼이 있습니다.

하단에 `취소` 버튼을 둘 예정이므로 상단 돌아가기 버튼은 제거해 주세요.

```txt
상단 돌아가기 버튼 제거
```

---

# 3. 기존 우측 하단 제출 버튼 제거

현재 카드 오른쪽 아래에 독립적인 `제출` 버튼이 있습니다.

하단에 `취소 / 제출` 버튼 영역을 새로 둘 예정이므로,
기존 우측 하단 제출 버튼은 제거해 주세요.

---

# 4. 질문 2 라디오 제거

현재 아래 질문이 있습니다.

```txt
2. 이 메뉴 배치로 최종 제출하시겠습니까?
[예] [취소]
```

하단에 `취소 / 제출` 버튼을 만들 예정이므로,
이 라디오 질문은 제거해도 됩니다.

수정 후 화면은 아래처럼 단순해지면 됩니다.

```txt
이 상태로 제출하시겠습니까?

1. 메뉴 구성 의도가 어떻게 되나요?
[textarea]

[취소] [제출]
```

---

# 5. 하단 취소 / 제출 버튼 추가

기존 PC/모바일 탭처럼 넓은 2분할 버튼 디자인을 하단에 배치해 주세요.

## 배치

```txt
[ 취소 ] [ 제출 ]
```

또는 카드 하단 전체 너비를 사용해 2열로 배치해 주세요.

```txt
┌────────────────────┐ ┌────────────────────┐
│        취소        │ │        제출        │
└────────────────────┘ └────────────────────┘
```

---

## 취소 버튼 스타일

취소 버튼은 밝은 배경으로 처리해 주세요.

```txt
배경: 흰색 또는 연한 회색
글자: 진한 남색/회색
테두리: 연한 회색
```

예시:

```jsx
className="bg-white text-slate-700 border border-slate-300"
```

---

## 제출 버튼 스타일

제출 버튼은 파란 배경에 흰 글씨로 강조해 주세요.

```txt
배경: 파란색
글자: 흰색
```

예시:

```jsx
className="bg-[#005BAC] text-white border border-[#005BAC]"
```

또는 현재 프로젝트에서 사용하는 메인 파란색이 있다면 그 색상을 사용해 주세요.

---

# 6. 취소 동작

하단 `취소` 버튼을 누르면 실제 저장 없이 편집 화면으로 돌아가게 해 주세요.

```txt
취소 클릭
→ 제출하지 않음
→ 편집 화면으로 복귀
```

이때 현재까지 편집한 메뉴 상태는 유지되어야 합니다.

---

# 7. 제출 동작

하단 `제출` 버튼을 누르면 실제 최종 제출이 진행되어야 합니다.

## 제출 전 검증

textarea 값이 비어 있으면 제출하지 않고 안내 문구를 보여주세요.

```txt
메뉴 구성 의도를 입력해주세요.
```

공백만 입력한 경우도 비어 있는 것으로 처리해 주세요.

```js
if (!intention.trim()) {
  setSubmitError("메뉴 구성 의도를 입력해주세요.");
  return;
}
```

# 8. submissions.intention 컬럼 저장

Supabase `submissions` 테이블에 아래 컬럼을 이미 추가했습니다.

```txt
intention text
```

최종 제출 시 textarea에 입력한 값을 이 컬럼에 저장해 주세요.

## 저장 대상

```txt
submissions.intention
```

## 최종 제출 payload 예시

기존 저장 payload가 있다면 그대로 유지하고, 아래 필드만 추가해 주세요.

```js
{
  ...existingPayload,
  intention: intention.trim()
}
```

예시:

```js
const payload = {
  auth_code: authCode,
  menu_data: currentMenuData,
  change_log: changeLog,
  intention: intention.trim(),
  submitted_at: new Date().toISOString(),
};
```

현재 프로젝트가 camelCase를 사용 중이라도 Supabase 컬럼명이 `intention`이므로
insert/update payload에는 반드시 아래 키를 포함해 주세요.

```js
intention: intention.trim()
```

# 9. 기존 저장 로직 유지

기존에 `submissions` 테이블에 저장하던 값들은 그대로 유지해 주세요.

예상 기존 저장 항목 예시:

```txt
인증코드
메뉴 구조
변경 로그
제출 시각
기타 기존 필드
```

이번 수정에서는 기존 저장 구조를 바꾸지 말고,
`intention` 필드만 추가해 주세요.

나쁜 예:

```js
// 기존 payload를 새로 만들면서 기존 필드를 누락
const payload = {
  intention: intention.trim()
};
```

좋은 예:

```js
const payload = {
  ...existingPayload,
  intention: intention.trim()
};
```

# 10. 제출 함수 예시

현재 최종 제출 함수가 있다면 textarea 값을 인자로 넘기거나,
해당 함수 내부 payload에 포함해 주세요.

예시:

```js
async function handleSubmitConfirm() {
  const trimmedIntention = intention.trim();

  if (!trimmedIntention) {
    setSubmitError("메뉴 구성 의도를 입력해주세요.");
    return;
  }

  setIsSubmitting(true);

  try {
    await handleFinalSubmit({
      intention: trimmedIntention,
    });

    setSubmitStep("submitted");
  } catch (error) {
    setSubmitError("제출 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요.");
  } finally {
    setIsSubmitting(false);
  }
}
```

기존 `handleFinalSubmit()` 내부에서는 아래처럼 payload에 추가해 주세요.

```js
async function handleFinalSubmit({ intention }) {
  const payload = {
    ...existingPayload,
    intention,
  };

  await supabase
    .from("submissions")
    .insert(payload);
}
```

만약 기존 코드가 `upsert` 또는 `update`를 사용 중이라면,
그 payload에도 동일하게 `intention`을 포함해 주세요.

# 11. 제출 중 중복 클릭 방지

제출 버튼 클릭 후 서버 저장 중에는 중복 제출을 막아 주세요.

```jsx
<button disabled={isSubmitting}>
  {isSubmitting ? "제출 중..." : "제출"}
</button>
```

취소 버튼도 제출 중에는 비활성화해도 됩니다.

```jsx
<button disabled={isSubmitting}>
  취소
</button>
```

# 12. 제출 성공 후 완료 화면

제출 성공 후에는 제출 완료 화면으로 이동해 주세요.

## 완료 화면 문구

```txt
참여해주셔서 감사합니다
```

보조 문구:

```txt
제출하신 메뉴 배치와 의견이 정상적으로 저장되었습니다.
```

## 완료 화면에서 미리보기 가능

완료 화면에서도 기존 미리보기 기능을 사용할 수 있으면 좋습니다.

```txt
미리보기 보기
```

기존 미리보기 기능을 재사용해 주세요.

Supabase `submissions` 테이블에 `intention text` 컬럼을 추가했습니다.

```txt
submissions.intention
```

하단에 `취소 / 제출` 버튼을 배치해 주세요.

```txt
취소 → 저장하지 않고 편집 화면 복귀
제출 → intention과 함께 최종 제출 저장
