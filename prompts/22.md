# Visual Editor Mode 상단 Level 1 탭 자동 전환 감도 개선 요청

## 현재 상태

현재 Visual Editor Mode의 gap/drop slot 방식은 정상적으로 동작하고 있습니다.

드래그 중 아이템 사이 slot 기준으로 drop되는 방식이 잘 구현되었고,  
preview 위치와 실제 drop 위치도 잘 일치합니다.

따라서 기존 drag/drop slot 로직은 최대한 건드리지 말아주세요.

---

# 남은 개선 요청

Level 2 또는 Level 3 메뉴를 다른 Level 1 영역으로 이동하려고 할 때,  
상단 Level 1 레이블/탭 쪽으로 드래그하면 해당 Level 1 탭으로 바로 전환되어야 합니다.

현재는 상단 Level 1 탭 근처로 드래그해도 즉시 전환되지 않고,  
탭 근처에서 계속 움직이다 보면 어쩌다가 전환됩니다.

즉, 상단 탭 자동 전환 감지가 불안정합니다.

---

# 중요 전제

아래 기능은 현재 잘 동작하므로 수정하지 말아주세요.

```txt
- gap/drop slot 방식
- preview 위치와 실제 drop 위치 일치 로직
- Level 2 reorder
- Level 3 reorder
- Level 2 다른 Level 1 아래로 이동
- Level 3 다른 Level 2 아래로 이동
- Level 4 생성 방지
````

이번 수정은 **상단 Level 1 탭 자동 전환 감도 개선만** 목표로 합니다.

---

# 원하는 동작

## Level 2 드래그 중

Level 2 메뉴를 드래그해서 상단 Level 1 탭 위로 가져가면,
그 Level 1 탭이 즉시 활성화되어야 합니다.

예시:

```txt
현재 활성 탭: SUPER STAR
드래그 중인 항목: 나의정보 Level 2
상단 탭 '정보공유' 위로 드래그
→ 정보공유 탭으로 자동 전환
→ 정보공유 내부 Level 2 drop slot 표시
→ 원하는 slot에 drop 가능
```

---

## Level 3 드래그 중

Level 3 메뉴를 다른 Level 1 안의 Level 2 아래로 옮기려면,
먼저 상단 Level 1 탭으로 이동해야 합니다.

따라서 Level 3 드래그 중에도 상단 Level 1 탭 위로 가져가면
해당 Level 1 탭으로 자동 전환되어야 합니다.

예시:

```txt
현재 활성 탭: SUPER STAR
드래그 중인 항목: My 정보 Level 3
상단 탭 '정보공유' 위로 드래그
→ 정보공유 탭으로 자동 전환
→ 그 안의 Level 2 목록이 표시됨
→ 원하는 Level 2 내부 Level 3 slot에 drop 가능
```

# 핵심 문제 추정

현재 상단 탭 전환이 어쩌다 되는 이유는 아래 중 하나일 가능성이 있습니다.

```txt
1. 탭 hover 감지 영역이 너무 좁음
2. 드래그 overlay가 상단 탭 위를 덮고 있어서 pointer event가 탭에 전달되지 않음
3. 드래그 중인 preview element가 탭 hover를 막고 있음
4. onMouseEnter / onPointerEnter가 drag 상태에서 안정적으로 발생하지 않음
5. dragOver 이벤트가 탭 영역에서 preventDefault 되지 않아 drop/hover 판정이 불안정함
6. 탭 전환에 debounce 또는 delay가 너무 길게 걸려 있음
7. 상단 탭의 실제 DOM rect와 사용자가 보는 위치가 다름
```

---

# 수정 방향

상단 Level 1 탭 전환은 일반 hover 이벤트만 믿지 말고,
드래그 중 pointer 좌표가 탭 rect 안에 들어왔는지 직접 검사해주세요.

즉, 드래그 중에는 아래 로직을 추가해주세요.

```txt
현재 pointer 위치 확인
→ 상단 Level 1 탭들의 boundingClientRect 검사
→ pointer가 특정 탭 rect 안에 있으면
→ 해당 Level 1 탭으로 전환
```

---

# 자동 탭 전환 함수 예시

```js
function getHoveringTopTab(pointerX, pointerY, topTabElements) {
  for (const tab of topTabElements) {
    const rect = tab.element.getBoundingClientRect();

    const isInside =
      pointerX >= rect.left &&
      pointerX <= rect.right &&
      pointerY >= rect.top &&
      pointerY <= rect.bottom;

    if (isInside) {
      return tab;
    }
  }

  return null;
}
```

---

# 드래그 중 탭 전환 처리 예시

```js
function handleDragMove(event) {
  const pointerX = event.clientX;
  const pointerY = event.clientY;

  const hoveringTab = getHoveringTopTab(
    pointerX,
    pointerY,
    topTabElements
  );

  if (hoveringTab && hoveringTab.id !== activeLevel1Id) {
    switchActiveLevel1Tab(hoveringTab.id);
  }

  // 기존 drop slot 계산은 그대로 유지
  updateDropSlotIntent(event);
}
```

# 탭 감지 영역 확장 요청

상단 탭은 실제 텍스트 영역만 감지하지 말고,
탭 전체 영역을 감지해야 합니다.

가능하면 위아래로 약간 여유 영역을 주세요.

```js
function isPointerInsideExpandedRect(pointerX, pointerY, rect, padding = 8) {
  return (
    pointerX >= rect.left &&
    pointerX <= rect.right &&
    pointerY >= rect.top - padding &&
    pointerY <= rect.bottom + padding
  );
}
```

권장:

```txt
상단 탭 hover 감지 영역 padding: 8px ~ 12px
```

# 드래그 overlay pointer-events 확인

드래그 중 따라다니는 preview나 overlay가
상단 탭 hover를 막지 않도록 해주세요.

```css
.drag-overlay,
.drag-preview,
.visual-drag-ghost {
  pointer-events: none;
}
```

중요:

```txt
드래그 중인 복제 요소가 상단 탭 위에 있어도
그 아래의 탭 hover 감지를 방해하면 안 됩니다.
```

---

# z-index 확인

상단 Level 1 탭 영역은 드래그 중에도 감지 가능해야 합니다.

```css
.top-level-tabs {
  position: relative;
  z-index: 100;
}

.drag-overlay {
  z-index: 999;
  pointer-events: none;
}
```

`drag-overlay`의 z-index가 높아도 `pointer-events: none`이면
탭 감지를 막지 않습니다.

---

# 탭 전환 debounce 주의

탭 전환을 너무 자주 하면 화면이 튈 수 있으므로
짧은 delay는 괜찮습니다.

하지만 현재처럼 “계속 움직이다 보면 어쩌다 전환”되는 수준이면
delay가 너무 길거나 hover 감지가 불안정한 것입니다.

권장:

```txt
즉시 전환 또는 100ms 이하 delay
```

예시:

```js
const TAB_SWITCH_DELAY = 80;
```

---

# 같은 탭 반복 전환 방지

같은 탭 위에 계속 있을 때는 반복 실행하지 않도록 해주세요.

```js
if (hoveringTab.id !== activeLevel1Id) {
  switchActiveLevel1Tab(hoveringTab.id);
}
```

필요하면 마지막 전환 시간을 저장해서 과도한 전환을 막아주세요.

```js
const lastTabSwitchTimeRef = useRef(0);

function canSwitchTabNow() {
  return Date.now() - lastTabSwitchTimeRef.current > 100;
}
```

---

# 드래그 중 탭 전환 후 drop slot 갱신

상단 탭이 전환되면,
새로 열린 Level 1 내부의 drop slot들이 즉시 다시 계산되어야 합니다.

```txt
탭 전환
→ activeLevel1Id 변경
→ 해당 Level 1의 Level 2/Level 3 목록 렌더링
→ drop slot rect 재측정
→ 현재 pointer 위치 기준으로 drop intent 재계산
```

즉, 탭만 바뀌고 drop slot이 이전 탭 기준으로 남으면 안 됩니다.

