# 1. Step 5 문제: 앞에 떠 있는 예시 아이템이 마우스 애니메이션을 가림

## 현재 문제

Step 5에서 핸들 드래그 가이드를 보여줄 때
마우스 애니메이션이 있어도 앞에 떠 있는 아이템/드래그 프리뷰 때문에 보이지 않습니다.

현재 화면상 문제:

```txt
하이라이트된 아이템 앞쪽에 예시 아이템 또는 ghost item이 떠 있음
그 아이템이 마우스 애니메이션을 덮고 있음
그래서 사용자가 "핸들을 아래로 드래그한다"는 애니메이션을 볼 수 없음
```

즉, 단순 z-index 문제가 아니라
**마우스 애니메이션 앞에 아이템 프리뷰가 떠 있어서 가리는 구조**입니다.

---

## 수정 요청

Step 5에서는 핸들 드래그 예시를 보여줄 때
앞에 떠 있는 예시 아이템/ghost item/drag preview가 마우스 애니메이션을 가리지 않도록 처리해 주세요.

---

## 원하는 방식

Step 5에서는 아래 중 하나로 처리해 주세요.

### 방식 A: 예시 아이템/ghost preview 제거

가이드 중에는 실제로 아이템을 들어 올린 것처럼 보이는 프리뷰를 만들지 말고,
**핸들만 강조 + 마우스 애니메이션만 표시**해 주세요.

```txt
아이템 전체 프리뷰 표시 안 함
핸들만 강조
마우스 포인터가 핸들을 잡고 아래로 움직이는 애니메이션 표시
```

---

### 방식 B: ghost item을 뒤로 보내기

예시 아이템/ghost preview가 꼭 필요하다면
마우스 애니메이션보다 뒤에 있어야 합니다.

```txt
ghost item / drag preview: z-index 9015
마우스 애니메이션: z-index 9045
가이드 설명창: z-index 9050
```

---

### 방식 C: ghost item 투명도 낮추기 + pointer 위로 올리기

```txt
ghost item opacity: 0.25 이하
마우스 애니메이션은 ghost item 위에 표시
```

---

## Step 5에서 반드시 보여야 하는 것

Step 5에서 사용자가 봐야 하는 핵심은 이것입니다.

```txt
핸들을 잡는다
아래로 끌어내린다
위치가 바뀔 수 있다는 것을 이해한다
```

따라서 아래 애니메이션이 반드시 보여야 합니다.

```txt
1. 마우스 포인터가 핸들 중앙으로 이동
2. 핸들을 누르는 모션
3. 핸들을 잡은 채 아래 방향으로 짧게 이동
4. 다시 원래 위치 또는 반복
```

---

## Step 5 금지사항

```txt
마우스 애니메이션 위에 ghost item이 올라오면 안 됨
하이라이트된 아이템이 마우스 애니메이션을 덮으면 안 됨
가이드 설명창이 마우스 애니메이션 경로를 가리면 안 됨
실제 menuState를 변경하면 안 됨
실제 drag event를 발생시키면 안 됨
```

---

## Step 5 z-index 강제

아래 순서를 반드시 지켜 주세요.

```txt
dim overlay: 9000
기존 화면 UI: 9005
하이라이트 박스: 9010
ghost item / drag preview: 9015 이하
마우스 포인터 애니메이션: 9045
가이드 설명창: 9050
가이드 버튼: 9060
```

특히 중요:

```txt
마우스 포인터 애니메이션은 ghost item보다 반드시 위
```

# 2. Step 7 문제: 설명창 위치가 그대로이고 하이라이트와 겹침

## 현재 문제

Step 7 메뉴 설명 확인 단계가 여전히 하이라이트 영역과 겹칩니다.

현재 문제:

```txt
가이드 설명창이 하이라이트 영역과 겹침
메뉴 설명 tooltip 위치도 애매함
hover 설명을 확인하는 흐름이 잘 보이지 않음
```

---

## 수정 요청

Step 7 설명창 위치를 Step 5처럼 **오른쪽 배치**로 고정해 주세요.

```txt
Step 7 placement: right
```

단, 오른쪽 공간이 부족할 때만 fallback 합니다.

```txt
1순위: 오른쪽
2순위: 왼쪽
3순위: 위
4순위: 아래
```

---

## Step 7에서 tooltip 표시 방식

Step 7에서는 tooltip이 처음부터 계속 떠 있으면 안 됩니다.

아래 흐름으로 보여주세요.

```txt
1. 처음에는 tooltip 없음
2. 마우스 포인터 애니메이션이 메뉴명 위로 이동
3. 포인터가 메뉴명 위에 도착한 순간 tooltip 표시
4. tooltip이 1~2초 보임
5. 단계가 끝나거나 포인터가 벗어나면 tooltip 숨김
```

---

## Step 7 금지사항

```txt
가이드 설명창이 하이라이트 영역을 덮으면 안 됨
가이드 설명창이 tooltip을 덮으면 안 됨
가이드 설명창 자체가 사라지면 안 됨
tooltip이 처음부터 계속 떠 있으면 안 됨
```

# 3. Step 9 문제: 등록관리 예시는 좋지만 마우스 애니메이션이 없음

## 현재 문제

Step 9에서 `등록관리` 중분류 메뉴를 대상으로 잡은 것은 좋습니다.

하지만 핸들을 움직이라는 마우스 애니메이션이 보이지 않습니다.

현재 문제:

```txt
등록관리 하이라이트는 좋음
하지만 핸들을 잡고 이동하는 마우스 애니메이션이 없음
그래서 사용자가 비주얼모드에서도 핸들을 잡아야 한다는 점을 알기 어려움
```

---

## 수정 요청

Step 9에도 Step 5와 같은 방식의 마우스 애니메이션을 넣어 주세요.

대상:

```txt
등록관리 메뉴의 점 모양 핸들
```

동작:

```txt
1. 등록관리 메뉴의 핸들 강조
2. 마우스 포인터가 핸들 중앙으로 이동
3. 핸들을 누르는 모션
4. 좌우 또는 대각선으로 짧게 이동
5. 실제 menuState는 변경하지 않음
```

---

## Step 9에서 확인할 것

아래를 반드시 확인해 주세요.

```txt
Step 9에서 pointer animation 옵션이 켜져 있는지
등록관리 핸들 element를 제대로 찾는지
포인터 좌표가 등록관리 핸들 중앙 기준인지
포인터 z-index가 하이라이트와 ghost item보다 높은지
포인터가 설명창 뒤에 가려지지 않는지
```

권장 data-guide-id:

```jsx
data-guide-id="visual-register-handle"
```

또는:

```txt
targetTitle: "등록관리"
targetLevel: 2
targetPart: "handle"
```

# 4. Step 5 / Step 9 공통 포인터 애니메이션 함수화

Step 5와 Step 9는 같은 “핸들을 잡는 방법”을 설명하므로
공통 포인터 애니메이션 함수를 쓰면 좋습니다.

```js
showHandleDragGuide({
  handleElement,
  direction: "vertical" // Step 5
});
```

```js
showHandleDragGuide({
  handleElement,
  direction: "horizontal" // Step 9
});
```

단, 실제 drag 이벤트는 발생시키지 마세요.

```txt
CSS transform 또는 가이드 전용 pointer layer만 사용
실제 drag/drop 로직 호출 금지
```
