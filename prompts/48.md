# 사용자 가이드(튜토리얼) 개선 요청

## 작업 목표

현재 가이드 기능은 기본 방향은 괜찮지만, 실제 사용성 측면에서 아래 문제가 있습니다.

### 현재 문제
1. 가이드 팝업/오버레이가 일부 UI를 너무 가립니다.
2. 드래그 가이드 애니메이션이 **핸들을 탭/드래그하는 느낌**이 아니라,  
   마치 **아이템 전체를 끌고 이동하는 느낌**으로 보여서 오해를 줄 수 있습니다.
3. 첫 진입 화면, 편집 진입, 편집 버튼, 아이템별 편집 버튼, 비주얼모드, 최종 제출 등  
   단계별 설명이 아직 충분하지 않습니다.
4. 가이드가 필요한 위치는 여러 군데인데, 현재는 일부만 설명되고 있습니다.

---

## 가장 중요한 조건

기존에 잘 동작하는 기능은 건드리지 말아 주세요.

```txt
메뉴 이동/정렬 로직
리스트모드 드래그앤드랍 로직
비주얼모드 드래그앤드랍 로직
임시저장 / 자동저장
최종 제출 저장
변경 로그 저장
툴팁 표시 기능의 기본 구조
미리보기
제출 확인 페이지
관리자 화면
````

이번 작업은 **가이드 UI/UX 개선**이 목적입니다.
기존 기능을 리팩토링하거나 핵심 동작을 바꾸지 말고,
**가이드 표시 방식과 가이드 흐름만 개선**해 주세요.

---

# 1. 가이드 전체 방향

## 가이드 기본 원칙

가이드는 아래 조건을 만족해야 합니다.

```txt
1. 실제 데이터를 변경하지 않아야 함
2. 실제 히스토리(undo/redo)에 영향을 주면 안 됨
3. 실제 임시저장/자동저장을 발생시키면 안 됨
4. UI를 완전히 가리지 않아야 함
5. 사용자가 지금 무엇을 해야 하는지 한눈에 이해할 수 있어야 함
```

---

## 가이드 표시 방식

가이드는 전체 모달처럼 화면 중앙을 크게 막기보다는,
**대상 UI를 강조하고 옆/위/아래에 설명 말풍선을 붙이는 방식**이 더 좋습니다.

추천 구조:

```txt
[대상 요소 강조]
+ [설명 말풍선]
+ [다음 / 이전 / 건너뛰기 / 다시 보지 않기]
```

---

## 가이드 팝업 가림 문제 해결

현재 가이드 팝업이 실제 안내 대상 영역 위를 덮어 일부가 안 보이거나,
클릭이 어려워질 수 있습니다.

따라서 아래처럼 수정해 주세요.

### 요구사항

```txt
가이드 팝업은 설명 대상 요소를 직접 가리지 않도록 배치
대상 요소와 겹치더라도 클릭을 막지 않도록 조정
팝업이 대상 위를 덮지 않도록 자동 위치 조정
화면이 좁을 때는 위/아래/좌/우 중 남는 위치로 붙이기
```

### 추천 방식

* 설명 박스는 `floating` 말풍선처럼 배치
* 대상 요소에는 `highlight ring`만 적용
* 설명 박스는 대상 외곽에 붙여서 노출
* 필요한 경우 `arrow`(꼬리)로 어떤 요소를 설명하는지 연결

# 2. 첫 화면 가이드

## 대상 화면

* 로그인 후 첫 화면
* 아직 편집 시작 전 기본 화면

---

## 2-1. 첫 화면: "편집하기 눌러서 편집할 수 있어요"

### 대상

* 상단의 `메뉴 편집하기` 버튼
  (또는 편집 시작 버튼)

### 목적

사용자가 첫 진입 시
**"아, 여기서 편집을 시작하는구나"**를 바로 이해할 수 있도록 안내합니다.

### 표시 문구 예시

```txt
메뉴를 직접 바꿔보고 싶다면 먼저 "메뉴 편집하기" 버튼을 눌러 주세요.
```

또는

```txt
여기서 메뉴 편집을 시작할 수 있어요.
```

### 동작

* `메뉴 편집하기` 버튼 강조
* 버튼 근처에 말풍선 표시
* `다음` 버튼으로 다음 가이드 단계 이동

---

## 2-2. 첫 화면: 각 버튼 기능 설명

### 대상

첫 화면 상단 버튼들:

* 임시저장
* 미리보기
* 최종 제출
* (필요 시 로그아웃 제외 가능)

### 목적

사용자가 **편집 전에도 무엇을 할 수 있는지** 기본 기능을 이해하도록 합니다.

### 설명할 버튼

```txt
임시저장: 작업 중인 메뉴를 저장합니다.
미리보기: 현재 메뉴 구성을 실제 화면처럼 확인할 수 있습니다.
최종 제출: 현재 메뉴 구성을 최종 제출합니다.
```

### 방식

* 각 버튼을 순서대로 강조
* 한 번에 한 버튼씩 설명
* 또는 한 영역 전체를 강조하고 오른쪽 설명 카드로 요약 가능

# 3. 편집화면 가이드

## 대상 화면

* 사용자가 편집 모드에 들어간 이후

---

## 3-1. 드래그 핸들 가이드 개선

### 현재 문제

현재 애니메이션이 **아이템 전체를 잡고 이동하는 것처럼** 보입니다.
하지만 실제로는 **왼쪽 핸들을 잡고 드래그해야** 하므로,
가이드도 정확히 핸들 위치를 기준으로 보여줘야 합니다.

### 수정 요청

드래그 예시 애니메이션은
**아이템 전체 이동 애니메이션**이 아니라
**핸들을 탭/클릭/드래그하는 애니메이션**으로 바꿔 주세요.

### 원하는 표현

```txt
1. 핸들 위치를 강조
2. 손가락/포인터 아이콘이 핸들을 탭
3. 핸들을 누른 상태로 아래(또는 위)로 드래그하는 짧은 애니메이션
4. 이동 가능한 위치에 placeholder 또는 강조 표시
5. 실제 데이터는 바뀌지 않음
```

### 중요

```txt
실제 menuState 변경 금지
실제 commitMenuChange 호출 금지
실제 드래그 처리 금지
가이드용 시각 효과만 표시
```

### 안내 문구 예시

```txt
왼쪽 이동 핸들을 잡고 위아래로 드래그하면 메뉴 순서를 바꿀 수 있어요.
```

또는

```txt
이 점 모양 핸들을 눌러 원하는 위치로 끌어보세요.
```

---

## 3-2. 가이드 팝업이 대상 UI를 가리지 않도록 개선

### 현재 문제

가이드 팝업이 실제 가이드 대상보다 앞에 떠서 일부 요소가 안 보이거나,
클릭이 어려워질 가능성이 있습니다.

### 수정 요청

```txt
설명 팝업은 대상 요소를 가리지 않도록 위치 조정
대상 강조 영역은 항상 시야에 보여야 함
필요 시 설명 팝업은 우측/좌측/상단/하단 자동 배치
좁은 화면에서는 팝업 크기 축소 또는 단계 분리
```

### 구현 방향

* 대상 요소는 `highlight`
* 팝업은 `floating help card`
* 타겟과 겹치지 않도록 placement 계산
* 대상 UI의 클릭을 막지 않도록 처리

---

## 3-3. 아이템별 편집기 버튼 기능 설명

### 대상

각 아이템 우측의 편집 버튼 영역
예: 좌우 이동, 수정, 삭제 등

### 목적

사용자가 각 버튼의 의미를 이해할 수 있도록 합니다.

### 설명할 버튼 예시

```txt
왼쪽/오른쪽 이동 버튼: 메뉴를 다른 위치 또는 다른 그룹으로 이동할 때 사용
수정 버튼: 메뉴명을 수정할 수 있음
삭제 버튼: 해당 메뉴를 삭제할 수 있음
```

### 방식

* 버튼 영역 강조
* 각 버튼을 순서대로 설명
* 아이콘 위 hover 툴팁이 이미 있다면,
  가이드에서는 "이 아이콘은 이런 기능입니다" 수준으로 간단히 설명

---

## 3-4. 메뉴명이 어렵거나 헷갈릴 때 hover 툴팁이 뜬다는 안내

### 대상

메뉴명 hover 설명 기능

### 목적

사용자가 **메뉴가 무슨 기능인지 잘 모르겠을 때 도움을 받을 수 있음**을 알려줍니다.

### 안내 문구 예시

```txt
메뉴 이름이 헷갈릴 때는 마우스를 올려 설명을 볼 수 있어요.
```

또는

```txt
어떤 메뉴인지 어려우면 메뉴 위에 마우스를 올려보세요. 설명 툴팁이 표시됩니다.
```

### 방식

* 특정 메뉴 하나를 예시로 강조
* hover 설명 말풍선 예시를 같이 보여줌
* 실제 툴팁 기능은 기존 구조 재사용

---

# 4. 비주얼모드 가이드

## 대상 화면

* 비주얼모드 편집 화면

### 목적

비주얼모드가 리스트모드와 다르게 보이므로,
사용자가 비주얼모드에서 무엇을 어떻게 옮기는지 이해할 수 있도록 안내합니다.

### 설명해야 할 내용

```txt
대분류는 상단 탭 순서만 바꿀 수 있음
중분류는 현재 대분류 안에서 순서를 바꾸거나 다른 대분류로 이동 가능
소분류는 같은 대분류 안의 다른 중분류 또는 다른 대분류의 중분류 아래로 이동 가능
핸들을 잡고 원하는 위치에 드롭하면 됨
```

### 안내 문구 예시

```txt
비주얼모드에서는 실제 화면처럼 보면서 메뉴를 옮길 수 있어요.
핸들을 잡고 원하는 위치로 이동해 보세요.
```

### 중요

비주얼모드 가이드는 아래를 보여줘야 합니다.

```txt
1. 어느 부분이 드래그 핸들인지
2. 어디로 이동 가능한지
3. 대분류 / 중분류 / 소분류의 이동 범위가 어떻게 다른지
```

### 애니메이션 방향

* 비주얼모드도 아이템 전체 이동처럼 보이지 않게
* **핸들 중심 탭 + 드래그** 느낌으로 표현
* 드롭 가능한 위치에 강조 표시

# 5. 최종 제출 가이드

## 대상 화면

* 최종 제출 확인 화면

### 목적

사용자가 이 화면의 의미를 정확히 이해하도록 안내합니다.

### 설명해야 할 내용

```txt
현재 구성한 메뉴와 변경 로그가 최종 제출됩니다.
입력한 메뉴 구성 의도도 함께 저장됩니다.
제출 후에는 완료 화면으로 이동합니다.
취소를 누르면 편집 화면으로 돌아갑니다.
```

### 안내 문구 예시

```txt
여기서 메뉴 구성 의도를 작성한 뒤 제출하면 최종 제출이 완료됩니다.
```

또는

```txt
최종 제출 시 현재 메뉴 배치와 변경 내용이 함께 저장됩니다.
```

### 버튼 설명

```txt
취소: 제출하지 않고 이전 화면으로 돌아갑니다.
제출: 현재 메뉴 배치를 최종 제출합니다.
```

### C. 비주얼모드 가이드

### D. 최종 제출 가이드

1. 구성 의도 입력 설명
2. 제출/취소 버튼 설명

# 8. 가이드 구현 시 주의사항

## 실제 데이터 변경 금지

가이드 때문에 아래가 발생하면 안 됩니다.

```txt
undo/redo 증가
change_log 추가
임시저장 발생
자동저장 발생
menuState 변경
draft 변경
최종 제출 데이터 변경
```

## 기존 기능 유지

가이드는 **기존 기능을 설명하는 레이어**여야 합니다.
기능 자체를 바꾸면 안 됩니다.

# 10. 핵심 요약

이번 작업은 **가이드 기능 보완**입니다.

핵심은 아래입니다.

```txt
