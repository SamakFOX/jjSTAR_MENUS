# 메뉴 편집기 추가 수정 요청: Level 2 → Level 1 이동 오류 및 장바구니 드롭존 분리

## 1. 현재 문제 요약

현재 메뉴 편집기에서 다음 문제가 발생하고 있습니다.

```txt
1. Level 2 메뉴를 다른 Level 1 하위로 이동하려고 하면 금지 경고가 뜸
2. 왼쪽 장바구니 드롭존이 제대로 인식되지 않음
3. 현재 구현은 장바구니 아이콘이 메뉴 카드 내부에 붙어 있어서 드롭존과 정렬 영역이 섞이는 것으로 보임
4. 모두 접기 기능이 하나뿐이라 Level 1 기준/Level 2 기준으로 보기 제어가 어려움
````

---

## 2. Level 2 → Level 1 하위 이동 오류 수정

현재 Level 2 메뉴를 다른 Level 1의 장바구니 드롭존에 넣으려고 하면 아래 경고가 뜹니다.

```txt
Level 2 메뉴는 Level 1 하위에만 배치할 수 있습니다.
```

하지만 이 상황은 금지가 아니라 **허용**되어야 합니다.

---

## 3. 정상 규칙

Level 2 메뉴는 반드시 Level 1의 children으로만 존재해야 합니다.

따라서 아래 이동은 허용되어야 합니다.

```txt
Level 2 → Level 1의 drop-into 영역
```

예시:

```txt
star 학생지원 L1
  JJSTAR 이용 매뉴얼 L2

학사정보 L1
  학사관리 L2
```

`JJSTAR 이용 매뉴얼 L2`를 `학사정보 L1`의 drop-into 영역에 넣으면:

```txt
star 학생지원 L1

학사정보 L1
  JJSTAR 이용 매뉴얼 L2
  학사관리 L2
```

이렇게 되어야 합니다.

---

## 4. Level 2 관련 드롭 규칙

장바구니/drop-into 영역 기준으로 다음 조건을 적용해 주세요.

| 드래그 대상  | 드롭 대상             | 처리                            |
| ------- | ----------------- | ----------------------------- |
| Level 2 | Level 1 drop-into | 허용, 해당 Level 1의 children으로 이동 |
| Level 2 | Level 2 drop-into | 통합 팝업                         |
| Level 2 | Level 3 drop-into | 금지                            |
| Level 2 | root              | 금지                            |
| Level 2 | 일반 행 영역           | 같은 Level 내 순서 정렬              |

현재처럼 `Level 2 → Level 1 drop-into`를 차단하면 안 됩니다.

---

## 5. 조건문 수정 요청

현재 코드에서 아래 케이스를 금지로 처리하고 있는 것으로 보입니다.

```txt
dragged.level === 2 && target.level === 1
```

이 조건은 금지가 아니라 허용이어야 합니다.

수정 기준:

```txt
dragged.level === 2 && target.level === 1
→ 허용

dragged.level === 2 && target.level === 2
→ 통합 팝업

dragged.level === 2 && target.level === 3
→ 금지
```

---

## 6. Level 2 → Level 1 이동 시 확인 팝업

Level 2를 다른 Level 1 하위로 이동할 때, 해당 Level 2에 하위 Level 3이 있다면 함께 이동 여부를 확인합니다.

문구:

```txt
"{Level 2 이름}"의 하위메뉴도 함께 이동하시겠습니까?
```

버튼:

```txt
[함께 이동] [취소]
```

처리:

```txt
함께 이동 → Level 2와 하위 Level 3 전체를 대상 Level 1 children으로 이동
취소 → 아무 작업도 하지 않음
```

---

## 7. Level 2 → Level 2 통합 처리 유지

Level 2는 다른 Level 2의 자식이 될 수 없습니다.

따라서 Level 2를 Level 2의 drop-into 영역에 넣으면 이동이 아니라 통합 확인 팝업을 띄웁니다.

문구:

```txt
"{이동할 Level 2 이름}"은 Level 2 메뉴이므로 "{대상 Level 2 이름}"의 하위 메뉴가 될 수 없습니다.

대신 "{이동할 Level 2 이름}" 메뉴를 제거하고,
그 하위메뉴를 "{대상 Level 2 이름}"의 하위메뉴로 통합할 수 있습니다.

진행하시겠습니까?
```

버튼:

```txt
[통합하기] [취소]
```

처리:

```txt
통합하기 → 이동할 Level 2 자체는 제거하고, 그 하위 Level 3만 대상 Level 2의 children으로 이동
취소 → 아무 작업도 하지 않음
```

# 8. 장바구니 드롭존 UI 구조 변경 요청

## 8.1 현재 문제

현재 구현은 장바구니 아이콘이 메뉴 카드 내부 또는 카드와 같은 행 안에 들어가 있습니다.

이로 인해 다음 문제가 발생하는 것으로 보입니다.

```txt
1. 장바구니 영역과 순서 정렬 영역이 겹침
2. 장바구니에 드롭해도 일반 정렬로 인식됨
3. 드래그 중 메뉴 카드가 움직이면 장바구니 영역도 같이 움직여 drop-into 판정이 불안정함
4. Level 2 → Level 1 담기 기능이 제대로 작동하지 않음
```

---

## 8.2 수정 방향

장바구니 드롭존을 메뉴 카드 내부가 아니라 **왼쪽 바깥 별도 컬럼**으로 분리해 주세요.

즉, 아래처럼 구현합니다.

```txt
[장바구니 drop-into 영역]   [메뉴 카드 sort 영역]
[장바구니 drop-into 영역]      [메뉴 카드 sort 영역]
[장바구니 drop-into 영역]         [메뉴 카드 sort 영역]
```

시안 기준:

```txt
왼쪽 고정 컬럼: 장바구니 아이콘
오른쪽 영역: 실제 메뉴 카드
```

---

## 8.3 중요한 구조

장바구니 아이콘은 메뉴 카드와 같은 transform/sortable 대상 안에 넣지 않습니다.

권장 구조:

```jsx
<div className="menu-row-shell">
  <DropIntoZone id={`drop-into:${item.id}`} />
  <SortableMenuCard id={`sort:${item.id}`} />
</div>
```

장바구니 영역과 메뉴 카드 영역은 역할을 명확히 분리합니다.

```txt
drop-into:{menuId}
→ 해당 메뉴 안으로 넣기 / 통합 전용

sort:{menuId}
→ 순서 정렬 전용
```

---

## 8.4 장바구니 드롭존 위치

장바구니는 메뉴 카드 왼쪽 바깥에 별도 영역으로 둡니다.

현재 구현처럼 메뉴 카드 안쪽 왼편에 포함하지 않습니다.

### 목표 형태

```txt
[🛒]  [ Level 1 카드 ---------------------------------- ]
[🛒]      [ Level 2 카드 ------------------------------ ]
[🛒]          [ Level 3 카드 -------------------------- ]
```

또는 시안처럼:

```txt
[🛒]  [카드]
[🛒]      [카드]
[🛒]          [카드]
```

---

## 8.5 드롭존 크기

장바구니 아이콘은 시각적으로는 작아도 실제 드롭 영역은 충분히 넓어야 합니다.

권장:

```txt
시각 아이콘 크기: 24px 내외
실제 드롭 영역: 44px ~ 52px
```

드롭존 배경:

```txt
모서리가 둥근 네모 배경
연한 회색 또는 흰색 배경
드래그 오버 시 노란색 또는 파란색 강조
```

---

## 8.6 드래그 중 표시

드래그 중일 때만 장바구니 드롭존을 더 명확히 강조합니다.

```txt
평소:
- 흐린 장바구니 아이콘
- 연한 테두리

드래그 중:
- drop 가능한 대상만 선명하게 표시
- drop 불가능한 대상은 흐리게 표시
- drop-over 시 하이라이트와 문구 표시
```

예시 문구:

```txt
"학사정보" 하위로 이동
"수강관리"에 하위메뉴 통합
Level 3은 Level 1 하위에 직접 배치할 수 없습니다.
```

# 9. sort 영역과 drop-into 영역의 역할 분리

## 9.1 drop-into 영역

장바구니 아이콘 영역입니다.

여기에 드롭했을 때만 다음 처리를 합니다.

```txt
Level 2 → Level 1: 하위로 이동
Level 2 → Level 2: 통합 팝업
Level 3 → Level 2: 하위로 이동
```

## 9.2 sort 영역

메뉴 카드 영역입니다.

여기에 드롭하면 무조건 순서 정렬만 처리합니다.

```txt
Level 1 카드 영역 → Level 1 순서정렬 모드에서만 정렬
Level 2 카드 영역 → 같은 Level 1 안의 Level 2 순서 정렬
Level 3 카드 영역 → 같은 Level 2 안의 Level 3 순서 정렬
```

## 9.3 금지

메뉴 카드 영역에 드롭했을 때 하위 이동이나 통합이 발생하면 안 됩니다.

```txt
카드 위에 드롭 = 순서 정렬
장바구니에 드롭 = 하위 이동 / 통합
```

---

# 10. Level 2 / Level 3이 Level 1로 올라가는 문제 차단

드래그드롭 시 Level 2나 Level 3이 root 또는 Level 1 위치로 올라가면 안 됩니다.

구조 제한:

```txt
Level 1 = root에만 존재
Level 2 = Level 1의 children에만 존재
Level 3 = Level 2의 children에만 존재
```

금지해야 할 결과:

```txt
Level 2가 root로 올라감
Level 3이 root로 올라감
Level 3이 Level 1의 직접 자식이 됨
Level 2가 Level 2의 직접 자식이 됨
Level 2가 Level 3의 자식이 됨
```

단, Level 2를 Level 2의 drop-into에 넣는 경우는 금지가 아니라 통합 팝업으로 처리합니다.

# 11. 레벨 보정 방식 금지

드롭 후 잘못된 level을 강제로 보정하지 마세요.

금지 예시:

```js
level: Math.max(1, Math.min(MAX_LEVEL, nextLevel))
```

이 방식은 Level 2나 Level 3이 잘못 올라갔는데도 눈에 보이는 level만 보정해서 구조를 더 꼬이게 만들 수 있습니다.

대신 이동 전에 구조 검증을 해야 합니다.

```txt
Level 2 이동 가능 부모: Level 1
Level 3 이동 가능 부모: Level 2
```

조건에 맞지 않으면 이동을 취소합니다.

# 12. 모두 접기 기능 분리 요청

현재 `모두 접기` 하나만 있어 작업 상황에 따라 보기 제어가 불편합니다.

다음처럼 버튼을 분리해 주세요.

## 권장 버튼

```txt
[대분류만 보기] [중분류까지 보기] [전체 펼치기]
```

또는:

```txt
[Level 1만 보기] [Level 2까지 보기] [전체 펼치기]
```

---

## 12.1 대분류만 보기

Level 1만 표시합니다.

```txt
Level 1 표시
Level 2 숨김
Level 3 숨김
```

---

## 12.2 중분류까지 보기

Level 1과 Level 2만 표시합니다.

```txt
Level 1 표시
Level 2 표시
Level 3 숨김
```

---

## 12.3 전체 펼치기

모든 레벨을 표시합니다.

```txt
Level 1 표시
Level 2 표시
Level 3 표시
```

---

# 13. 현재 수정 우선순위

이번 수정에서 우선순위는 다음과 같습니다.

```txt
1. 장바구니 drop-into 영역을 메뉴 카드 바깥의 왼쪽 별도 컬럼으로 분리
2. drop-into와 sort 영역을 DnD ID부터 분리
3. Level 2 → Level 1 drop-into 이동 허용
4. Level 2 → Level 2 drop-into 통합 팝업 유지
5. Level 3 → Level 2 drop-into 이동 허용
6. Level 2 / Level 3이 root나 잘못된 레벨로 올라가는 것 차단
7. 모두 접기 기능을 대분류만 보기 / 중분류까지 보기 / 전체 펼치기로 분리
```

# 14. 핵심 요약

```txt
장바구니 아이콘은 메뉴 카드 내부가 아니라 왼쪽 별도 컬럼에 둔다.

장바구니 영역 = drop-into 전용
메뉴 카드 영역 = sort 전용

Level 2 → Level 1 장바구니는 허용
Level 2 → Level 2 장바구니는 통합 팝업
Level 3 → Level 2 장바구니는 허용

Level 2나 Level 3이 Level 1/root로 올라가는 것은 금지한다.

모두 접기 기능은 Level 1만 보기 / Level 2까지 보기 / 전체 펼치기로 분리한다.
```
