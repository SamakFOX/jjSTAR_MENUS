# 리스트 편집 화면 레이아웃 고정 방식 수정 요청

## 현재 문제

이전 수정으로 리스트 편집 화면의 상단 도구 영역을 sticky/fixed 처리했더니 레이아웃이 깨지는 문제가 발생했습니다.

특히 아래와 같은 현상이 있습니다.

- 상단 편집 바가 본문 위에 겹침
- 리스트 영역 일부가 가려짐
- 우측 안내 영역과 좌측 리스트 영역의 높이/위치가 어긋남
- 스크롤 시 상단 영역이 화면을 덮어서 메뉴 편집이 불편함
- 전체 상단 영역을 고정하니 레이아웃이 과하게 복잡해짐

따라서 기존처럼 상단 전체를 고정하는 방식은 사용하지 말아 주세요.

---

## 수정 방향

상단 전체를 고정하지 말고, **되돌리기/다시실행이 포함된 작업 도구 박스만 우측 안내 영역 위쪽에 고정 배치**해 주세요.

즉, 기존 상단 바는 일반 레이아웃 흐름에 두고,  
아래 기능들이 들어있는 박스만 별도 위치로 옮겨 주세요.

대상 기능:

```txt
[Level 1 추가]
[Level 1 순서변경]
[되돌리기]
[다시실행]
[Level 1만 보기]
[Level 2까지 보기]
[전체 펼치기]
최근 20회의 편집 이력을 되돌리거나 다시 실행할 수 있습니다.
````

---

## 원하는 배치 구조

현재 우측에는 안내 영역이 있습니다.

예시:

```txt
[안내]
1. 편집 모드에서 메뉴 순서 변경...
2. 화면에서 편집 모드는...
3. 메뉴 배치가 완료되면...
```

이 안내 영역 바로 위에 되돌리기/다시실행 도구 박스를 배치해 주세요.

### 변경 전 느낌

```txt
상단 전체 편집 바
────────────────────

좌측: 도구 박스 + 메뉴 리스트        우측: 안내
```

### 변경 후 원하는 구조

```txt
상단 편집 바
────────────────────

좌측: 메뉴 리스트                    우측: 도구 박스
                                      우측: 안내
                                      우측: 제출 안내
```

즉, 되돌리기 도구 박스는 좌측 리스트 위에 있지 않고
**오른쪽 사이드 패널의 최상단 영역**에 들어가면 됩니다.

---

## 우측 영역 고정 요청

우측 안내 영역은 스크롤해도 움직이지 않게 해 주세요.

현재는 좌측 메뉴 리스트가 길어지면 전체 페이지가 함께 스크롤되면서 우측 안내 영역도 같이 움직이는 구조처럼 보입니다.

수정 후에는 다음처럼 동작하면 좋겠습니다.

### 기대 동작

* 좌측 메뉴 리스트만 스크롤됨
* 우측 도구 박스 / 안내 박스 / 제출 안내 박스는 화면에 고정
* 상단 전체 편집 바는 기존 위치 유지
* 되돌리기/다시실행 버튼은 항상 우측에서 접근 가능
* 리스트 항목과 상단 바가 서로 겹치지 않음

---

## 레이아웃 구조 제안

전체 편집 화면을 다음과 같은 2컬럼 구조로 정리해 주세요.

```jsx
<div className="edit-page">
  {/* 상단 편집 바 */}
  <div className="edit-top-bar">
    편집 완료 / 리스트 편집 / 화면에서 편집 / 초기화 / 미리보기 / 최종 제출
  </div>

  {/* 본문 2컬럼 */}
  <div className="edit-layout">
    {/* 좌측: 메뉴 리스트 */}
    <section className="menu-list-panel">
      메뉴 리스트
    </section>

    {/* 우측: 고정 사이드 패널 */}
    <aside className="edit-side-panel">
      <div className="history-tool-box">
        Level 1 추가 / Level 1 순서변경 / 되돌리기 / 다시실행 / 보기 옵션
      </div>

      <div className="guide-box">
        안내
      </div>

      <div className="submit-guide-box">
        제출 안내
      </div>
    </aside>
  </div>
</div>
```

## CSS 방향 제안

`position: fixed`를 남발하지 말고, 가능하면 전체 화면 높이 안에서 좌측만 스크롤되는 구조로 잡아 주세요.

예시:

```css
.edit-page {
  min-height: 100vh;
  background: #f5f6fb;
}

.edit-top-bar {
  position: relative;
  z-index: 10;
  background: #ffffff;
}

.edit-layout {
  display: grid;
  grid-template-columns: minmax(0, 1fr) 348px;
  gap: 28px;
  max-width: 1120px;
  margin: 0 auto;
  padding: 24px 0;
  align-items: start;
}

.menu-list-panel {
  min-width: 0;
}

.edit-side-panel {
  position: sticky;
  top: 96px;
  display: flex;
  flex-direction: column;
  gap: 24px;
  align-self: start;
}
```

단, 현재 프로젝트의 헤더 높이에 따라 `top: 96px` 값은 조정해 주세요.

---

## 좌측 리스트만 스크롤하는 방식이 필요할 경우

페이지 전체 스크롤 대신 좌측 리스트만 스크롤시키고 싶다면 아래 구조도 가능합니다.

```css
.edit-layout {
  display: grid;
  grid-template-columns: minmax(0, 1fr) 348px;
  gap: 28px;
  max-width: 1120px;
  margin: 0 auto;
  height: calc(100vh - 160px);
  overflow: hidden;
}

.menu-list-panel {
  overflow-y: auto;
  min-height: 0;
  padding-right: 8px;
}

.edit-side-panel {
  overflow: visible;
}
```

이 방식에서는:

* 좌측 메뉴 리스트만 내부 스크롤
* 우측 패널은 고정된 것처럼 유지
* 상단 바와 겹치지 않음

## 히스토리 도구 박스 구성

우측 사이드 패널 최상단에 아래 박스를 배치해 주세요.

```txt
[Level 1 추가] [Level 1 순서변경]

[되돌리기 (n)] [다시실행 (n)]

[Level 1만 보기] [Level 2까지 보기] [전체 펼치기]

최근 20회의 편집 이력을 되돌리거나 다시 실행할 수 있습니다.
```

버튼이 너무 많아 한 줄에 모두 들어가지 않으면 2~3줄로 나누어도 됩니다.

예시:

```jsx
<div className="history-tool-box">
  <div className="tool-row">
    <button>＋ Level 1 추가</button>
    <button>Level 1 순서변경</button>
  </div>

  <div className="tool-row">
    <button>↶ 되돌리기 ({undoCount})</button>
    <button>↷ 다시실행 ({redoCount})</button>
  </div>

  <div className="tool-row compact">
    <button>Level 1만 보기</button>
    <button>Level 2까지 보기</button>
    <button>전체 펼치기</button>
  </div>

  <p>최근 20회의 편집 이력을 되돌리거나 다시 실행할 수 있습니다.</p>
</div>
```

---

## 중요: 고정하지 말아야 할 영역

아래 영역은 더 이상 sticky/fixed로 고정하지 말아 주세요.

```txt
[편집 완료]
[리스트 편집]
[화면에서 편집]
[초기화]
[미리보기]
[최종 제출]
```

이 상단 편집 바는 기존처럼 자연스럽게 화면 상단에 배치하되, 리스트 스크롤과 겹치지 않게만 유지하면 됩니다.

---

## 우측 안내 영역 동작

우측 패널 안의 구성은 다음 순서로 보여 주세요.

```txt
1. 히스토리/도구 박스
2. 안내 박스
3. 제출 안내 박스
```

우측 패널은 스크롤해도 위치가 유지되어야 합니다.

가능하면 `position: sticky`를 사용하고,
`position: fixed`는 레이아웃이 더 깨질 수 있으므로 피해주세요.

## 최종 기대 결과

| 항목              | 기대 결과                    |
| --------------- | ------------------------ |
| 상단 편집 바         | 기존 위치 유지, 고정하지 않음        |
| 되돌리기/다시실행 도구 박스 | 우측 안내 박스 위에 배치           |
| 우측 안내 영역        | 스크롤해도 움직이지 않음            |
| 좌측 메뉴 리스트       | 길어지면 좌측 영역에서만 스크롤        |
| 버튼 겹침           | 없어야 함                    |
| 메뉴 항목 가림        | 없어야 함                    |
| 전체 레이아웃         | 좌측 편집 / 우측 도구+안내 구조로 안정화 |

---

## 핵심 요약

이전처럼 상단 전체를 sticky 처리하지 말고,
되돌리기/다시실행이 포함된 도구 박스만 우측 사이드 패널의 안내 영역 위에 배치해 주세요.

우측 패널은 스크롤 시 움직이지 않게 유지하고,
좌측 메뉴 리스트만 스크롤되는 구조로 변경해 주세요.

```
