# Visual Editor Mode 전용 드래그/드롭 제한 보정 요청

## 중요 전제

현재 **List Mode는 정상 동작하고 있으므로 절대 수정하지 마세요.**

이번 수정 범위는 **Visual Editor Mode 전용**입니다.

List Mode의 기존 drag/drop, reorder, 상태 업데이트 로직은 변경하지 않고,  
Visual Editor Mode에서만 레벨별 허용 동작을 정확히 보정해주세요.

---

# 현재 문제

이전 수정 이후 Visual Editor Mode에서 잘못된 하위 삽입은 어느 정도 막혔지만,  
정상적으로 가능해야 하는 이동까지 막히고 있습니다.

특히 아래 기능이 필요합니다.

```txt
Level 2:
- 같은 Level 2끼리 순서 변경 가능해야 함
- 다른 Level 1 아래로 이동 가능해야 함

Level 3:
- 같은 부모 Level 2 안에서 순서 변경 가능해야 함
- 다른 Level 2 부모 아래로 이동 가능해야 함
````

현재는 Level 2 드래그 시 순서 변경까지 막히는 문제가 있습니다.

---

# 핵심 원칙

Visual Mode에서 막아야 하는 것은
**잘못된 하위 삽입**이지, 정상적인 순서 변경이나 부모 변경이 아닙니다.

따라서 아래처럼 구분해주세요.

```txt
reorder = 같은 레벨끼리 순서 변경
move-to-parent = 허용된 부모 아래로 이동
move-as-invalid-child = 금지
```

---

# Level 2 동작 규칙

## Level 2에서 허용해야 하는 동작

Level 2 메뉴를 드래그했을 때는 아래 동작이 가능해야 합니다.

| 동작                               | 허용 여부 |
| -------------------------------- | ----- |
| 같은 Level 1 안에서 Level 2끼리 순서 변경   | 허용    |
| 다른 Level 1 아래로 이동                | 허용    |
| 다른 Level 1 아래의 Level 2 목록 사이에 삽입 | 허용    |
| Level 1의 마지막 Level 2로 이동         | 허용    |
| Level 3 아래로 들어가기                 | 금지    |
| 다른 Level 2의 하위로 들어가기             | 금지    |
| Level 4 생성                       | 절대 금지 |

---

## Level 2에서 금지해야 하는 동작

```txt
Level 2 → Level 2의 자식으로 들어가기 금지
Level 2 → Level 3의 자식으로 들어가기 금지
Level 2 → Level 3과 하위 삽입 상호작용 금지
Level 2 → Level 4 생성 금지
```

단, 아래는 금지하면 안 됩니다.

```txt
Level 2 → Level 2 사이로 순서 변경
Level 2 → 다른 Level 1 아래로 이동
Level 2 → 다른 Level 1 아래의 Level 2 목록 안으로 이동
```

## Level 3에서 허용해야 하는 동작

Level 3 메뉴를 드래그했을 때는 아래 동작이 가능해야 합니다.

| 동작                                | 허용 여부 |
| --------------------------------- | ----- |
| 같은 Level 2 부모 안에서 Level 3끼리 순서 변경 | 허용    |
| 다른 Level 2 부모 아래로 이동              | 허용    |
| 다른 Level 2 아래의 Level 3 목록 사이에 삽입  | 허용    |
| Level 2의 마지막 Level 3으로 이동         | 허용    |
| Level 1 바로 아래로 이동                 | 금지    |
| Level 3의 하위로 들어가기                 | 금지    |
| Level 4 생성                        | 절대 금지 |

---

## Level 3에서 금지해야 하는 동작

```txt
Level 3 → Level 1 바로 아래로 이동 금지
Level 3 → Level 3의 자식으로 들어가기 금지
Level 3 → Level 4 생성 금지
```

단, 아래는 금지하면 안 됩니다.

```txt
Level 3 → 같은 부모 Level 2 안에서 순서 변경
Level 3 → 다른 Level 2 부모 아래로 이동
Level 3 → 다른 Level 2 아래의 Level 3 목록 안으로 이동
```

# Drop Mode 구분

Visual Mode에서는 drop 동작을 최소한 아래처럼 구분해주세요.

```ts
type VisualDropMode =
  | 'reorder'
  | 'move-to-parent'
  | 'move-as-child';
```

## reorder

같은 레벨 항목 사이에서 순서를 바꾸는 동작입니다.

```txt
Level 1 ↔ Level 1
Level 2 ↔ Level 2
Level 3 ↔ Level 3
```

## move-to-parent

허용된 부모 아래로 이동하는 동작입니다.

```txt
Level 2 → Level 1 아래로 이동
Level 3 → Level 2 아래로 이동
```

## move-as-child

특정 항목의 하위로 직접 넣는 동작입니다.

이 동작은 반드시 레벨 규칙을 검사해야 합니다.

```txt
Level 2는 Level 1의 하위로만 가능
Level 3은 Level 2의 하위로만 가능
Level 1은 하위 이동 불가
```

# Visual Mode 전용 허용 규칙

List Mode에 영향이 없도록 Visual Mode 전용 함수로 처리해주세요.

```js
function canVisualMoveToParent(dragItem, newParentItem) {
  // Level 1은 부모를 가질 수 없음
  if (dragItem.level === 1) return false;

  // Level 2는 Level 1 아래로만 이동 가능
  if (dragItem.level === 2) {
    return newParentItem.level === 1;
  }

  // Level 3은 Level 2 아래로만 이동 가능
  if (dragItem.level === 3) {
    return newParentItem.level === 2;
  }

  return false;
}
```

# Visual Mode 전용 순서 변경 규칙

중요:
Level 2와 Level 3의 순서 변경은 반드시 가능해야 합니다.

```js
function canVisualReorder(dragItem, targetItem) {
  // 순서 변경은 같은 레벨끼리만 가능
  if (dragItem.level !== targetItem.level) return false;

  // Level 1은 Level 1끼리 순서 변경 가능
  if (dragItem.level === 1) {
    return true;
  }

  // Level 2는 Level 2끼리 순서 변경 가능
  // 같은 부모 안의 정렬은 당연히 가능해야 하고,
  // 다른 Level 1 아래의 Level 2 사이로 이동하는 경우도 가능해야 함
  if (dragItem.level === 2) {
    return targetItem.level === 2;
  }

  // Level 3은 Level 3끼리 순서 변경 가능
  // 같은 부모 안의 정렬은 가능해야 하고,
  // 다른 Level 2 아래의 Level 3 사이로 이동하는 경우도 가능해야 함
  if (dragItem.level === 3) {
    return targetItem.level === 3;
  }

  return false;
}
```

# Reorder 시 parentId 처리 규칙

Visual Mode에서 같은 레벨끼리 reorder할 때,
targetItem의 parentId를 기준으로 새 parentId를 결정해주세요.

## Level 2 reorder

Level 2를 다른 Level 2 위치로 옮기는 경우:

```txt
dragItem.level === 2
targetItem.level === 2
```

처리:

```txt
dragItem.parentId = targetItem.parentId
targetItem 주변 위치로 순서 삽입
```

즉, Level 2끼리 reorder할 때는
같은 부모 안이면 단순 순서 변경이고,
다른 부모 아래의 Level 2 위/아래로 놓으면 부모도 같이 변경됩니다.

---

## Level 3 reorder

Level 3을 다른 Level 3 위치로 옮기는 경우:

```txt
dragItem.level === 3
targetItem.level === 3
```

처리:

```txt
dragItem.parentId = targetItem.parentId
targetItem 주변 위치로 순서 삽입
```

즉, Level 3끼리 reorder할 때는
같은 부모 안이면 단순 순서 변경이고,
다른 부모 아래의 Level 3 위/아래로 놓으면 부모 Level 2도 같이 변경됩니다.

# Level 2에서 Level 3은 drop target 제외

Level 2를 드래그 중일 때 Level 3은 여전히 drop target이 아니어야 합니다.

```js
function shouldIgnoreVisualTarget(dragItem, targetItem) {
  // Level 2는 Level 3과 상호작용하지 않음
  if (dragItem.level === 2 && targetItem.level === 3) {
    return true;
  }

  // Level 3은 Level 1과 직접 상호작용하지 않음
  if (dragItem.level === 3 && targetItem.level === 1) {
    return true;
  }

  return false;
}
```

# Visual Drop 처리 예시

```js
function handleVisualDrop(dragItem, targetItem, dropMode, position) {
  // Visual Editor Mode 전용
  // List Mode에서는 사용하지 않음

  if (shouldIgnoreVisualTarget(dragItem, targetItem)) {
    console.warn('Ignored invalid visual target:', {
      dragLevel: dragItem.level,
      targetLevel: targetItem.level,
      dropMode,
    });
    return;
  }

  if (dropMode === 'reorder') {
    if (!canVisualReorder(dragItem, targetItem)) {
      console.warn('Invalid visual reorder blocked:', {
        dragLevel: dragItem.level,
        targetLevel: targetItem.level,
      });
      return;
    }

    applyVisualReorder(dragItem, targetItem, position);
    return;
  }

  if (dropMode === 'move-to-parent') {
    if (!canVisualMoveToParent(dragItem, targetItem)) {
      console.warn('Invalid visual parent move blocked:', {
        dragLevel: dragItem.level,
        targetLevel: targetItem.level,
      });
      return;
    }

    applyVisualMoveToParent(dragItem, targetItem);
    return;
  }

  if (dropMode === 'move-as-child') {
    if (!canVisualMoveToParent(dragItem, targetItem)) {
      console.warn('Invalid visual child move blocked:', {
        dragLevel: dragItem.level,
        targetLevel: targetItem.level,
      });
      return;
    }

    applyVisualMoveToParent(dragItem, targetItem);
    return;
  }
}
```

# applyVisualReorder 처리 기준

```js
function applyVisualReorder(dragItem, targetItem, position) {
  // position: 'before' | 'after'

  // 같은 레벨끼리만 들어옴
  const newParentId = targetItem.parentId || null;

  const updatedItem = {
    ...dragItem,
    parentId: newParentId,
    level: targetItem.level,
  };

  // updatedItem을 targetItem의 before/after 위치에 삽입
  // 이때 기존 위치에서는 dragItem을 제거
  // children이 있는 Level 2를 옮길 경우, 하위 Level 3들은 같이 따라가야 함
  // children이 있는 Level 1을 옮길 경우, 하위 Level 2/3들도 같이 따라가야 함
}
```

# 최종 요약

Visual Editor Mode에서 막아야 하는 것은
정상적인 reorder가 아니라 **잘못된 하위 삽입**입니다.

따라서 아래 기능은 반드시 유지되어야 합니다.

```txt
Level 2:
- Level 2끼리 순서 변경 가능
- 다른 Level 1 아래로 이동 가능

Level 3:
- Level 3끼리 순서 변경 가능
- 다른 Level 2 아래로 이동 가능
```

그리고 아래 동작만 차단해주세요.

```txt
Level 2 → Level 3 아래로 이동 금지
Level 2 → Level 2의 자식으로 이동 금지
Level 3 → Level 1 바로 아래로 이동 금지
Level 3 → Level 3의 자식으로 이동 금지
Level 4 생성 금지
```

이번 수정은 반드시 **Visual Editor Mode 전용**으로 처리하고,
List Mode는 수정하지 마세요.
