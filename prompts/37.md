# 비주얼모드 Level 1 탭 활성화/전환 기준 수정 요청

## 현재 문제

비주얼모드에서 Level 2 메뉴를 드래그하여 다른 Level 1 탭으로 이동할 때,  
실제 마우스 포인터가 Level 1 레이블 위에 정확히 올라가지 않았는데도 탭이 활성화되고 전환되는 문제가 있습니다.

현재는 마우스 위치가 아니라 드래그 중인 카드 또는 DragOverlay의 충돌 영역을 기준으로 Level 1 탭 전환이 발생하는 것처럼 보입니다.

그 결과 사용자는 레이블 위에 정확히 올린 것이 아닌데도 탭이 전환되어 조작감이 어색합니다.

---

## 수정 목표

Level 1 탭 활성화와 전환은 반드시 **실제 마우스 포인터 위치**를 기준으로 판단해 주세요.

즉, 드래그 중인 카드의 크기, DragOverlay의 위치, collision detection 결과만으로 탭을 전환하면 안 됩니다.

---

## 핵심 원칙

```txt
Level 1 탭 전환 기준 = 실제 마우스 포인터가 Level 1 레이블의 실제 DOM 영역 안에 있는지
````

아래 기준은 사용하지 말아 주세요.

```txt
드래그 중인 카드 박스가 탭과 겹치는지
DragOverlay가 탭과 충돌하는지
dnd-kit의 over.id가 Level 1인지
탭 주변 넓은 영역에 들어왔는지
```

---

## 기대 동작

Level 2를 드래그 중일 때:

```txt
마우스 포인터가 Level 1 레이블 위에 정확히 있음
→ 해당 Level 1 탭 활성화 후보

마우스 포인터가 Level 1 레이블 밖에 있음
→ 탭 활성화/전환 없음

마우스 포인터가 탭과 탭 사이 빈 공간에 있음
→ 탭 활성화/전환 없음

마우스 포인터가 탭 아래 본문 영역에 있음
→ 탭 활성화/전환 없음

드래그 카드가 탭 위에 겹쳐 보이지만 실제 포인터가 탭 밖에 있음
→ 탭 활성화/전환 없음
```

---

## 구현 방향

Level 1 탭 DOM 요소들의 실제 위치를 기준으로 hit test를 해주세요.

예시:

```js
function isPointerInsideElement(pointer, element) {
  if (!pointer || !element) return false;

  const rect = element.getBoundingClientRect();

  return (
    pointer.x >= rect.left &&
    pointer.x <= rect.right &&
    pointer.y >= rect.top &&
    pointer.y <= rect.bottom
  );
}
```

조금 더 안정적으로 하려면 가장자리 오작동 방지를 위해 inset을 줄 수 있습니다.

```js
const SAFE_INSET_X = 4;
const SAFE_INSET_Y = 3;

function isPointerInsideTab(pointer, tabElement) {
  if (!pointer || !tabElement) return false;

  const rect = tabElement.getBoundingClientRect();

  return (
    pointer.x >= rect.left + SAFE_INSET_X &&
    pointer.x <= rect.right - SAFE_INSET_X &&
    pointer.y >= rect.top + SAFE_INSET_Y &&
    pointer.y <= rect.bottom - SAFE_INSET_Y
  );
}
```

이렇게 하면 탭 가장자리를 살짝 스치거나 근처를 지나는 것만으로 전환되는 문제를 줄일 수 있습니다.

## pointer 좌표 저장

dnd-kit의 `over` 값만으로 판단하지 말고, 실제 pointer 좌표를 저장해서 사용해 주세요.

예시 방향:

```js
const pointerRef = useRef({ x: 0, y: 0 });

function handlePointerMove(event) {
  pointerRef.current = {
    x: event.clientX,
    y: event.clientY,
  };
}
```

또는 dnd-kit 이벤트에서 pointer 좌표를 안정적으로 얻을 수 있는 방식으로 처리해 주세요.

중요한 점은 다음입니다.

```txt
Level 1 탭 전환 판단에는 pointerRef.current 값을 사용
DragOverlay rect나 active node rect는 사용하지 않음
```

## Level 1 탭 ref 관리

Level 1 탭마다 ref 또는 DOM element를 추적할 수 있게 해주세요.

예시:

```js
const level1TabRefs = useRef(new Map());

function setLevel1TabRef(level1Id, node) {
  if (node) {
    level1TabRefs.current.set(level1Id, node);
  } else {
    level1TabRefs.current.delete(level1Id);
  }
}
```

렌더링 예시:

```jsx
<div
  ref={(node) => setLevel1TabRef(level1.id, node)}
  data-level1-id={level1.id}
>
  {level1.title}
</div>
```

## Level 1 hover 후보 계산

드래그 중 pointer가 실제로 어떤 Level 1 탭 안에 있는지 직접 찾아주세요.

```js
function getLevel1TabUnderPointer(pointer) {
  for (const [level1Id, element] of level1TabRefs.current.entries()) {
    if (isPointerInsideTab(pointer, element)) {
      return level1Id;
    }
  }

  return null;
}
```

## Level 1 전환 조건

Level 2 드래그 중 Level 1 전환은 아래 조건을 모두 만족할 때만 실행해 주세요.

```txt
1. 현재 드래그 중인 아이템이 Level 2일 것
2. dragMode가 LEVEL_2_SORT_OR_MOVE일 것
3. 실제 마우스 포인터가 Level 1 탭 DOM 영역 내부에 있을 것
4. 현재 activeLevel1Id와 다른 Level 1일 것
5. 일정 시간 같은 탭 위에 머물렀을 것
```

## hover delay 적용

마우스가 탭을 잠깐 스치는 것만으로 전환되지 않도록 짧은 delay를 적용해 주세요.

권장값:

```txt
180ms ~ 250ms
```

예시:

```js
const TAB_SWITCH_DELAY = 200;
const pendingLevel1SwitchRef = useRef(null);
const level1SwitchTimerRef = useRef(null);

function scheduleLevel1Switch(targetLevel1Id) {
  if (!targetLevel1Id) {
    cancelLevel1Switch();
    return;
  }

  if (targetLevel1Id === activeLevel1Id) {
    cancelLevel1Switch();
    return;
  }

  if (pendingLevel1SwitchRef.current === targetLevel1Id) {
    return;
  }

  cancelLevel1Switch();

  pendingLevel1SwitchRef.current = targetLevel1Id;

  level1SwitchTimerRef.current = setTimeout(() => {
    setActiveLevel1Id(targetLevel1Id);
    pendingLevel1SwitchRef.current = null;
    level1SwitchTimerRef.current = null;
  }, TAB_SWITCH_DELAY);
}

function cancelLevel1Switch() {
  if (level1SwitchTimerRef.current) {
    clearTimeout(level1SwitchTimerRef.current);
  }

  level1SwitchTimerRef.current = null;
  pendingLevel1SwitchRef.current = null;
}
```

## drag move 처리 예시

```js
function handleVisualDragMove(event) {
  if (!visualDragState) return;

  const { dragMode } = visualDragState;

  if (dragMode !== "LEVEL_2_SORT_OR_MOVE") {
    return;
  }

  const pointer = pointerRef.current;
  const targetLevel1Id = getLevel1TabUnderPointer(pointer);

  if (!targetLevel1Id) {
    cancelLevel1Switch();
    return;
  }

  scheduleLevel1Switch(targetLevel1Id);
}
```

중요:

```txt
over.id만으로 Level 1 전환하지 말 것
pointer가 실제 tab rect 안에 들어왔는지 반드시 확인할 것
```

## 전환 후 처리

Level 1이 실제로 전환되면 Level 2 preview parent도 해당 Level 1로 갱신해 주세요.

```js
setActiveLevel1Id(targetLevel1Id);

setVisualDragState((prev) => ({
  ...prev,
  previewParentId: targetLevel1Id,
  previewIndex: getEndIndexOfLevel2(targetLevel1Id),
}));
```

단, 이 변경은 드래그 중 미리보기 상태 변경일 뿐이므로 히스토리에 저장하면 안 됩니다.

---

## 히스토리 주의

Level 1 탭 전환은 보기 상태 또는 preview 상태 변경입니다.

따라서 아래 동작으로는 히스토리가 증가하면 안 됩니다.

```txt
Level 1 탭 hover
Level 1 탭 활성화
Level 1 탭 전환
previewParentId 변경
previewIndex 변경
```

히스토리는 실제 드롭 성공 시에만 증가해야 합니다.

```txt
드롭 성공 → commitMenuChange(nextMenuState) 1회 호출
```

## 핵심 요약

현재 Level 1 탭 전환이 실제 마우스 위치보다 넓은 충돌 영역을 기준으로 발생하는 문제가 있습니다.

수정 핵심은 다음입니다.

```txt
1. Level 1 탭 전환은 실제 pointer 좌표 기준으로만 판단
2. DragOverlay 또는 카드 rect 기준으로 전환하지 않기
3. Level 1 탭 DOM의 getBoundingClientRect 내부에 포인터가 있을 때만 후보로 인정
4. 180~250ms hover delay 적용
5. 탭 밖으로 나가면 전환 타이머 취소
6. 현재 활성 탭으로는 재전환하지 않기
7. 탭 전환만으로 히스토리 증가시키지 않기
```
