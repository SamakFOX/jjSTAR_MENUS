# 신규 사용자 / 초기화 사용자용 예시 조작 가이드 구현 요청

## 작업 목표

임시저장 데이터가 없는 사용자 또는 초기화를 실행한 사용자에게  
메뉴 편집 방법을 쉽게 이해할 수 있도록 예시 조작 가이드를 보여주고 싶습니다.

현재 드래그 핸들이 작거나 기능을 처음 접하는 사용자는  
어디를 잡고 어떻게 이동해야 하는지 바로 이해하기 어려울 수 있습니다.

따라서 최초 진입 또는 초기화 직후에만  
간단한 드래그 예시 안내를 보여주는 기능을 추가해 주세요.

---

## 가장 중요한 조건

이 기능은 안내용입니다.

실제 메뉴 데이터, 히스토리, 임시저장, 최종 제출 데이터에 영향을 주면 안 됩니다.

```txt
가이드 애니메이션은 실제 메뉴 상태를 변경하지 않음
가이드 표시만으로 undo/redo 히스토리 증가 금지
가이드 표시만으로 임시저장 dirty 처리 금지
가이드 표시만으로 자동저장 실행 금지
가이드 표시만으로 change_log 추가 금지
````

---

## 표시 조건

가이드는 아래 조건 중 하나일 때 표시해 주세요.

```txt
1. 로그인 후 해당 auth_code의 drafts 데이터가 없음
2. 사용자가 초기화 버튼을 눌러 메뉴가 초기 상태로 돌아감
```

즉:

```txt
draft 있음 → 가이드 자동 표시 안 함
draft 없음 → 가이드 표시
초기화 실행 → 가이드 다시 표시 가능
```

---

## 가이드 표시 방식

추천 방식은 화면 위에 얇은 안내 오버레이 또는 말풍선 형태입니다.

예시 문구:

```txt
메뉴 왼쪽의 이동 핸들을 잡고 드래그하면 순서를 바꿀 수 있습니다.
```

또는:

```txt
예시: 이 핸들을 잡고 원하는 위치로 끌어보세요.
```

---

## 예시 조작 애니메이션

실제 메뉴를 이동하지 말고, 시각적으로만 아래 동작을 보여주세요.

```txt
1. 특정 메뉴의 드래그 핸들 강조
2. 손가락/마우스 포인터 아이콘 표시
3. 메뉴 카드가 살짝 이동하는 듯한 애니메이션
4. 이동 가능한 위치에 placeholder처럼 안내 표시
5. 원래 상태로 복귀
```

중요:

```txt
실제 menuState 변경 금지
실제 commitMenuChange 호출 금지
실제 drag 이벤트 발생 금지
```

---

## 적용 화면

우선순위는 아래와 같습니다.

```txt
1순위: 리스트모드
2순위: 비주얼모드
```

처음에는 리스트모드에서만 가이드를 보여줘도 괜찮습니다.

비주얼모드는 구조가 복잡하므로,
리스트모드 가이드가 안정화된 뒤 확장해도 됩니다.

---

## 리스트모드 가이드 예시

리스트모드에서 첫 번째 Level 2 또는 Level 3 메뉴를 대상으로
드래그 핸들을 강조해 주세요.

예시:

```txt
[핸들 강조] star 관리
```

가이드 말풍선:

```txt
이 부분을 잡고 위아래로 드래그해 메뉴 순서를 바꿀 수 있습니다.
```

버튼:

```txt
시작하기
다시 보지 않기
```

---

## 비주얼모드 가이드 예시

비주얼모드에서는 상단 또는 메뉴 그룹 근처에 아래처럼 안내합니다.

```txt
메뉴 제목 왼쪽의 핸들을 잡고 원하는 위치로 이동하세요.
```

Level 2 예시:

```txt
중분류 메뉴는 같은 대분류 안에서 순서를 바꾸거나 다른 대분류로 이동할 수 있습니다.
```

Level 3 예시:

```txt
소분류 메뉴는 다른 중분류 아래로 이동할 수 있습니다.
```

## 가이드 상태 관리

가이드 표시 여부는 auth_code별로 관리하는 것이 좋습니다.

단, 처음 구현에서는 클라이언트 상태만으로도 충분합니다.

### 간단 구현

```js
const [showGuide, setShowGuide] = useState(false);
```

로그인 후 draft 확인:

```js
if (!draft) {
  setShowGuide(true);
}
```

초기화 후:

```js
setShowGuide(true);
```

---

## 다시 보지 않기

사용자가 가이드를 원하지 않을 수 있으므로
`다시 보지 않기` 버튼을 제공해 주세요.

```txt
다시 보지 않기
```

이 경우 해당 auth_code 기준으로 가이드가 자동 표시되지 않게 하면 좋습니다.

초기 구현에서는 localStorage를 사용해도 됩니다.

```js
localStorage.setItem(`hideGuide:${authCode}`, "true");
```

로그인 시:

```js
const hidden = localStorage.getItem(`hideGuide:${authCode}`) === "true";

if (!draft && !hidden) {
  setShowGuide(true);
}
```

---

## Supabase 저장 여부

가이드 표시 여부까지 Supabase에 저장할 필요는 없습니다.

초기에는 localStorage로 충분합니다.

이유:

```txt
가이드는 사용성 안내일 뿐 핵심 데이터가 아님
기기마다 다시 보여도 큰 문제 없음
Supabase 테이블을 늘릴 필요 없음
```

단, 나중에 관리자 통계가 필요하면 별도 필드나 테이블을 고려할 수 있습니다.

---

## 임시저장 데이터와의 관계

임시저장 데이터가 있으면 사용자가 이미 작업을 시작한 것으로 보고
가이드를 자동 표시하지 않습니다.

```txt
draft 있음 → 가이드 표시 안 함
draft 없음 → 가이드 표시
```

단, 사용자가 직접 `도움말 보기` 버튼을 누르면 다시 볼 수 있게 해도 됩니다.

---

## 초기화와의 관계

초기화를 누르면 메뉴가 처음 상태로 돌아가므로,
사용자에게 다시 안내를 보여줘도 됩니다.

```txt
초기화 실행
→ menuState 초기화
→ undo/redo 초기화
→ draft 삭제 또는 초기 상태 저장
→ showGuide true
```

단, `다시 보지 않기`를 선택한 사용자는 초기화 후에도 자동 표시하지 않아도 됩니다.

---

## UI 위치 제안

가이드는 화면 전체를 막는 강한 모달보다
작은 안내 오버레이가 좋습니다.

추천 형태:

```txt
반투명 오버레이 + 핸들 주변 강조
또는 우측 상단 작은 도움말 카드
```

예시 문구:

```txt
처음이신가요?
왼쪽 점 모양 핸들을 잡고 드래그하면 메뉴 위치를 바꿀 수 있습니다.
```

버튼:

```txt
알겠어요
다시 보지 않기
```

---

## 도움말 다시 보기 버튼

우측 편집 도구나 안내 박스에 아래 버튼을 추가해도 좋습니다.

```txt
사용 방법 보기
```

이 버튼을 누르면 가이드를 다시 볼 수 있습니다.

기존 기능에 영향이 없도록 단순히 `setShowGuide(true)`만 실행하면 됩니다.

---

## 구현 시 주의사항

아래 함수들은 가이드 표시 중에 호출되면 안 됩니다.

```txt
commitMenuChange
saveDraft
handleFinalSubmit
setUndoStack
setRedoStack
addChangeLog
```

가이드 애니메이션은 CSS transform이나 별도 dummy overlay로 처리해 주세요.

---

## 추천 구현 방식

### 좋은 방식

```txt
실제 메뉴 카드 위에 guide-highlight class만 적용
가짜 포인터 아이콘을 CSS animation으로 이동
메뉴 카드는 transform animation만 시각적으로 표시
실제 데이터는 변경하지 않음
```

### 피해야 할 방식

```txt
실제 drag 이벤트를 강제로 발생
실제 menuState를 잠깐 변경했다가 복구
commitMenuChange 호출 후 undo
임시저장 데이터에 예시 조작 반영
```

## 예시 컴포넌트

```jsx
function GuideOverlay({ targetRef, onClose, onDontShowAgain }) {
  return (
    <div className="guide-overlay">
      <div className="guide-card">
        <strong>메뉴 이동 방법</strong>
        <p>왼쪽 이동 핸들을 잡고 드래그하면 메뉴 순서를 바꿀 수 있습니다.</p>

        <div className="guide-actions">
          <button onClick={onClose}>알겠어요</button>
          <button onClick={onDontShowAgain}>다시 보지 않기</button>
        </div>
      </div>

      <div className="guide-pointer-animation" />
    </div>
  );
}
```

## 스타일 방향

```css
.guide-highlight {
  outline: 2px solid #005BAC;
  box-shadow: 0 0 0 6px rgba(0, 91, 172, 0.12);
  border-radius: 12px;
}

.guide-pointer-animation {
  animation: guide-drag-demo 1.8s ease-in-out infinite;
}

@keyframes guide-drag-demo {
  0% {
    transform: translate(0, 0);
    opacity: 0;
  }
  20% {
    opacity: 1;
  }
  60% {
    transform: translate(80px, 0);
    opacity: 1;
  }
  100% {
    transform: translate(80px, 0);
    opacity: 0;
  }
}
```

실제 이동 방향은 리스트모드에서는 위아래 방향으로,
비주얼모드에서는 좌우 또는 카드 사이 방향으로 맞춰 주세요.

## 핵심 요약

임시저장 데이터가 없는 사용자 또는 초기화한 사용자에게
드래그 사용법을 알려주는 가이드 오버레이를 표시해 주세요.

가이드는 실제 메뉴 데이터를 변경하지 않는 시각적 안내여야 합니다.

```txt
draft 없음 → 가이드 표시
초기화 후 → 가이드 표시 가능
draft 있음 → 자동 표시 안 함
다시 보지 않기 → localStorage에 저장
```

Supabase 테이블을 새로 추가할 필요는 없고,
초기 구현은 localStorage + 화면 상태로 처리하면 됩니다.

```
