# 비주얼모드 드롭 판정 영역 확대 요청

## 현재 문제

비주얼모드에서 메뉴를 드래그할 때, 아이템이 들어갈 위치 표시가 너무 늦게 나타납니다.

현재는 메뉴 카드의 가운데 좁은 영역에 정확히 닿아야만 기존 아이템이 밀리고, 해당 위치에 들어간다는 표시가 나타납니다.

사용자 입장에서는 카드 근처나 카드 사이에 충분히 가져간 것 같은데도 반응하지 않고, 정확히 좁은 중심 영역에 맞춰야 해서 조작이 불편합니다.

---

## 원인 추정

현재 드롭 판정이 아래 중 하나에 의존하고 있는 것으로 보입니다.

```txt
카드 중심점 기준
드래그 아이템 중심점 기준
텍스트/핸들 영역만 droppable로 등록
아이템 내부 좁은 영역만 ref 등록
over.id가 특정 아이템 위에 있을 때만 preview 계산
````

이 경우 카드 전체나 카드 사이 공간이 드롭 후보로 인식되지 않아, 사용자가 느끼는 반응 영역이 너무 좁아집니다.

---

## 수정 목표

비주얼모드에서 드롭 판정 영역을 넓혀 주세요.

메뉴 카드의 가운데 부분만 반응하는 것이 아니라, 아래 영역들이 모두 드롭 후보 계산에 포함되어야 합니다.

```txt
메뉴 카드 전체 영역
메뉴 카드 사이 공간
해당 Level의 정렬 트랙 전체 영역
부모 그룹 내부의 빈 공간
```

---

## 핵심 원칙

드래그 중인 레벨에 따라 해당 레벨의 정렬 트랙 전체가 활성화되어야 합니다.

```txt
Level 1 드래그 중 → Level 1 탭/레이블 트랙 전체
Level 2 드래그 중 → 현재 Level 1 내부의 Level 2 트랙 전체
Level 3 드래그 중 → 현재 Level 2 내부의 Level 3 트랙 전체
```

카드 텍스트나 핸들에 정확히 닿아야만 반응하는 방식은 피해주세요.

---

## Droppable ref 적용 위치 확인

`useDroppable` 또는 `useSortable`의 `setNodeRef`가 카드 내부 일부 요소에만 붙어 있다면, 카드 전체 wrapper로 옮겨 주세요.

### 나쁜 예시

```jsx
<div className="visual-menu-card">
  <span ref={setNodeRef}>{menu.title}</span>
</div>
```

이 경우 텍스트 부분만 드롭 영역이 될 수 있습니다.

### 좋은 예시

```jsx
<div
  ref={setNodeRef}
  className="visual-menu-card"
>
  <span>{menu.title}</span>
</div>
```

카드 전체가 드롭/정렬 계산 대상이 되어야 합니다.

---

## 정렬 트랙 전체를 Droppable로 등록

개별 카드뿐만 아니라, Level별 목록 컨테이너도 droppable 영역으로 등록해 주세요.

예시:

```jsx
<div
  ref={setLevel2TrackRef}
  data-droppable-type="LEVEL_2_TRACK"
  className="level-2-track"
>
  {level2Items.map(...)}
</div>
```

이렇게 하면 카드 사이 빈 공간이나 목록 내부 빈 공간에서도 드롭 위치를 계산할 수 있습니다.

---

## 드롭 위치 계산 개선

`over.id`가 특정 카드 위에 있을 때만 preview를 계산하지 말고,
현재 마우스 포인터가 해당 레벨 트랙 안에 있다면 가장 가까운 삽입 위치를 계산해 주세요.

예시 흐름:

```txt
1. pointer가 Level 2 트랙 내부에 있는지 확인
2. 트랙 내부의 Level 2 카드 rect 목록을 가져옴
3. pointer 위치 기준으로 가장 가까운 삽입 index 계산
4. previewIndex 갱신
5. placeholder 표시
```

---

## 삽입 index 계산 기준

가로 배치라면 X 좌표를 기준으로 계산해 주세요.

```js
function getInsertIndexByX(pointerX, itemRects) {
  for (let i = 0; i < itemRects.length; i++) {
    const rect = itemRects[i];
    const centerX = rect.left + rect.width / 2;

    if (pointerX < centerX) {
      return i;
    }
  }

  return itemRects.length;
}
```

세로 배치라면 Y 좌표 기준으로 계산해 주세요.

```js
function getInsertIndexByY(pointerY, itemRects) {
  for (let i = 0; i < itemRects.length; i++) {
    const rect = itemRects[i];
    const centerY = rect.top + rect.height / 2;

    if (pointerY < centerY) {
      return i;
    }
  }

  return itemRects.length;
}
```

줄바꿈되는 그리드 형태라면 X/Y를 함께 고려해 주세요.

## 카드 사이 공간에서도 반응하게 하기

현재는 카드 중앙에 가까워야만 반응합니다.

수정 후에는 아래처럼 카드 사이 공간에 포인터가 있어도 해당 위치에 placeholder가 나타나야 합니다.

```txt
[카드 A]   [포인터]   [카드 B]
          ↑
      여기에 placeholder 표시
```

즉, 포인터가 정확히 카드 위에 있지 않아도, 해당 Level 트랙 내부라면 삽입 위치를 계산해야 합니다.

## dnd-kit collisionDetection 조정

`@dnd-kit`을 사용 중이라면 collision detection이 너무 좁게 잡혀 있을 수 있습니다.

현재 `closestCenter`만 사용 중이라면 `pointerWithin`, `rectIntersection`, 또는 커스텀 collision detection을 검토해 주세요.

권장 방향:

```txt
1. 실제 pointer가 활성 Level 트랙 안에 있는지 먼저 확인
2. 트랙 안에 있으면 해당 트랙의 아이템 rect 기준으로 insertIndex 직접 계산
3. dnd-kit의 over.id만으로 위치를 결정하지 않기
```

즉, dnd-kit collision 결과는 참고만 하고,
최종 previewIndex는 pointer 좌표와 실제 item rect 목록으로 직접 계산하는 것이 좋습니다.

## 빈 트랙 처리

해당 부모 안에 아이템이 없는 경우에도 드롭 가능해야 합니다.

예시:

```txt
Level 2를 빈 Level 1 그룹으로 이동
Level 3을 빈 Level 2 그룹으로 이동
```

이 경우 pointer가 해당 트랙 내부에 있으면:

```txt
previewIndex = 0
```

으로 처리해 주세요.

## 시각 피드백

드래그 중에는 사용자가 어느 영역이 현재 드롭 가능한지 알 수 있어야 합니다.

권장 표시:

```txt
현재 활성 Level 트랙에 은은한 배경 강조
삽입 위치에 placeholder 또는 라인 표시
기존 아이템이 placeholder 기준으로 밀림
```

단, 금지표시는 되도록 사용하지 말고, 허용된 트랙만 강조해 주세요.

---

## 드롭 결과 일치

가장 중요한 원칙은 아래입니다.

```txt
드래그 중 보이는 placeholder 위치 = 실제 드롭 후 위치
```

따라서 previewIndex를 계산한 로직과 실제 drop 시 targetIndex 계산 로직이 달라지면 안 됩니다.

드롭 시에는 드래그 중 마지막으로 계산된 `previewParentId`, `previewIndex`를 그대로 사용해 주세요.

---

## 히스토리 주의

드래그 중 previewIndex가 바뀌는 것은 히스토리에 저장하면 안 됩니다.

```txt
드래그 중 위치 미리보기 변경 → 히스토리 변화 없음
드롭 성공 → commitMenuChange(nextMenuState) 1회 호출
드롭 취소 → 히스토리 변화 없음
```

트랙 내 item rect 목록을 기준으로 `previewIndex`를 직접 계산해 주세요.@
