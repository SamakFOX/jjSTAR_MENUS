# 비주얼모드 Level 1 탭 전환 민감도 수정 요청

## 현재 상태

비주얼모드 Drag & Drop 로직과 반응성은 많이 좋아졌습니다.

현재 확인된 개선점:

```txt
드래그 반응성 좋음
위치 계산 정확도 좋아짐
드롭 후 실제 위치도 잘 들어감
Level 2 이동 로직도 전반적으로 안정적임
````

다만 한 가지 수정이 필요합니다.

---

## 현재 문제

Level 2 메뉴를 드래그하는 중에 다른 Level 1 탭으로 전환되는 타이밍이 너무 민감합니다.

마우스가 Level 1 탭에 충분히 가까이 가지 않았는데도,
다른 Level 1 탭으로 자동 전환되는 문제가 있습니다.

예시:

```txt
Level 2 메뉴를 드래그 중
→ 마우스가 Level 1 탭 근처에 있지만 실제 탭 위는 아님
→ 그런데 해당 Level 1 탭으로 전환됨
```

이 때문에 사용자가 의도하지 않았는데 상단 탭이 바뀌고,
작업 흐름이 불안정하게 느껴집니다.

---

## 수정 목표

Level 1 탭 전환은 **마우스가 실제 Level 1 탭 영역 안에 명확하게 들어왔을 때만** 발생해야 합니다.

현재처럼 탭 주변의 넓은 영역이나 근처 hover만으로 전환되면 안 됩니다.

---

## 기대 동작

Level 2 드래그 중 다른 Level 1로 전환되는 조건은 아래처럼 제한해 주세요.

```txt
마우스 포인터가 Level 1 탭의 실제 bounding box 내부에 들어왔을 때만 전환
```

즉, 아래 조건을 만족해야 합니다.

```js
pointerX >= tabRect.left
pointerX <= tabRect.right
pointerY >= tabRect.top
pointerY <= tabRect.bottom
```

이 조건을 만족하지 않으면 Level 1 전환을 하지 말아 주세요.

---

## 전환 민감도 조정

탭 전환 영역을 넓게 잡지 말고, 실제 탭 크기보다 작거나 같은 수준으로 잡아 주세요.

가능하면 아래처럼 안전 영역을 줄여도 됩니다.

```js
const SAFE_INSET_X = 8;
const SAFE_INSET_Y = 4;

const isInsideTab =
  pointerX >= tabRect.left + SAFE_INSET_X &&
  pointerX <= tabRect.right - SAFE_INSET_X &&
  pointerY >= tabRect.top + SAFE_INSET_Y &&
  pointerY <= tabRect.bottom - SAFE_INSET_Y;
```

이렇게 하면 탭 가장자리만 스쳐도 전환되는 문제를 줄일 수 있습니다.

---

## hover delay 추가 요청

가능하다면 Level 1 탭 전환에 짧은 delay를 추가해 주세요.

마우스가 탭 위에 잠깐 스치는 정도로는 전환하지 말고,
해당 탭 위에 일정 시간 머물렀을 때만 전환되면 좋겠습니다.

권장 값:

```txt
150ms ~ 250ms
```

예시:

```js
const TAB_SWITCH_DELAY = 200;
```

동작 방식:

```txt
Level 2 드래그 중 다른 Level 1 탭 위에 진입
→ 즉시 전환하지 않음
→ 200ms 동안 같은 탭 위에 머무르면 전환
→ 중간에 탭 밖으로 나가면 전환 취소
→ 다른 탭으로 이동하면 기존 타이머 취소 후 새 타이머 시작
```

---

## 같은 탭으로는 재전환하지 않기

현재 활성화된 Level 1 탭 위에 있을 때는 전환 로직을 실행하지 않아도 됩니다.

```js
if (targetLevel1Id === activeLevel1Id) {
  return;
}
```

이 처리를 통해 불필요한 상태 업데이트와 깜빡임을 줄여 주세요.

---

## Level 1 전환 조건 정리

Level 2 드래그 중 Level 1 탭 전환은 아래 조건을 모두 만족할 때만 실행해 주세요.

```txt
1. 현재 드래그 중인 아이템이 Level 2일 것
2. dragMode가 LEVEL_2_SORT_OR_MOVE일 것
3. 마우스 포인터가 Level 1 탭의 실제 영역 내부에 있을 것
4. 현재 활성 Level 1과 다른 탭일 것
5. 같은 탭 위에 150~250ms 정도 머물렀을 것
```

위 조건 중 하나라도 만족하지 않으면 탭을 전환하지 말아 주세요.

## 잘못된 동작 예시

아래 상황에서는 Level 1 탭 전환이 발생하면 안 됩니다.

```txt
마우스가 탭 아래 본문 영역에 있음
마우스가 탭과 탭 사이 빈 공간에 있음
마우스가 탭 주변 여백에 있음
마우스가 Level 2 카드 위에 있음
마우스가 드래그 중인 카드의 overlay 기준으로 탭과 겹친 것처럼 보이지만 실제 포인터는 탭 밖에 있음
마우스가 탭 가장자리를 아주 잠깐 스침
```

특히 중요한 점:

```txt
드래그 중인 카드의 overlay 위치가 아니라 실제 마우스 포인터 위치를 기준으로 판단
```

해주세요.

---

## DragOverlay 기준으로 판단하지 않기

현재 문제는 드래그 중인 카드 또는 overlay의 영역이 Level 1 탭 근처에 닿으면 전환되는 방식 때문일 수 있습니다.

Level 1 탭 전환은 카드 박스 충돌 기준이 아니라
**실제 pointer 좌표 기준**으로 판단해 주세요.

나쁜 기준:

```txt
드래그 중인 카드 영역이 Level 1 탭과 충돌하면 전환
```

좋은 기준:

```txt
실제 마우스 포인터가 Level 1 탭 내부에 들어오면 전환
```

---

## 구현 예시

```js
const TAB_SWITCH_DELAY = 200;
const SAFE_INSET_X = 8;
const SAFE_INSET_Y = 4;

function isPointerInsideTab(pointer, tabElement) {
  if (!pointer || !tabElement) return false;

  const rect = tabElement.getBoundingClientRect();

  return (
    pointer.x >= rect.left + SAFE_INSET_X &&
    pointer.x <= rect.right - SAFE_INSET_X &&
    pointer.y >= rect.top + SAFE_INSET_Y &&
    pointer.y <= rect.bottom - SAFE_INSET_Y
  );
}
```

전환 처리 예시:

```js
function scheduleLevel1Switch(targetLevel1Id) {
  if (targetLevel1Id === activeLevel1Id) return;

  clearTimeout(level1SwitchTimerRef.current);

  level1SwitchTimerRef.current = setTimeout(() => {
    setActiveLevel1Id(targetLevel1Id);
  }, TAB_SWITCH_DELAY);
}
```

탭 밖으로 나가거나 다른 대상이 되면 타이머를 취소합니다.

```js
function cancelLevel1Switch() {
  clearTimeout(level1SwitchTimerRef.current);
  level1SwitchTimerRef.current = null;
}
```

## dnd-kit 사용 시 참고

`@dnd-kit`에서 `over` 대상만으로 Level 1 전환을 판단하면,
충돌 감지 방식에 따라 마우스가 실제 탭 안에 없어도 전환될 수 있습니다.

따라서 `over.id`만 믿지 말고,
pointer 좌표와 실제 탭 DOM rect를 함께 확인해 주세요.

필요하다면 drag move 이벤트에서 pointer 좌표를 저장해 사용해 주세요.

예시:

```js
function handleDragMove(event) {
  setPointerPosition({
    x: event.activatorEvent?.clientX ?? event.delta?.x,
    y: event.activatorEvent?.clientY ?? event.delta?.y,
  });
}
```

실제 구현에서는 dnd-kit 이벤트 구조에 맞게 현재 pointer 좌표를 안정적으로 가져와 주세요.

## Level 3 전환에도 동일 원칙 적용

Level 3을 드래그 중 다른 Level 2로 전환하는 로직이 있다면,
동일하게 너무 민감하게 전환되지 않도록 해 주세요.

단, 이번에 특히 문제가 되는 것은 Level 2 드래그 중 Level 1 탭 전환입니다.

우선순위:

```txt
1순위: Level 2 드래그 중 Level 1 탭 전환 민감도 수정
2순위: Level 3 드래그 중 Level 2 전환 민감도도 동일 기준 적용
```

---

## 히스토리 영향 주의

Level 1 탭 전환은 드래그 중 보기 상태 변경일 뿐입니다.

따라서 탭 전환만으로는 히스토리가 증가하면 안 됩니다.

```txt
Level 1 탭 전환 → 히스토리 변화 없음
드롭 성공 → commitMenuChange 1회 호출
드롭 취소 → 히스토리 변화 없음
```

## 핵심 요약

현재 비주얼모드 로직과 위치 반영은 좋아졌지만,
Level 2 드래그 중 Level 1 탭 전환이 너무 민감합니다.

수정 핵심은 다음입니다.

```txt
1. Level 1 전환은 실제 마우스 포인터가 탭 내부에 있을 때만 실행
2. DragOverlay나 카드 충돌 기준으로 전환하지 않기
3. 탭 전환에 150~250ms delay 적용
4. 탭 주변 여백이나 본문 영역에서는 전환하지 않기
5. 현재 활성 탭으로는 불필요한 재전환 금지
6. 탭 전환만으로 히스토리 증가 금지
```
