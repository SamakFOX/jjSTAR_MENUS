# 가이드 Step 5 / Step 7 / Step 9 추가 수정 요청

## 작업 목표

현재 가이드에서 아래 3개 단계에 문제가 있습니다.

```txt
1. Step 5 리스트모드 핸들 설명이 UI를 가리고, 마우스 애니메이션이 뒤에 깔려 보이지 않음
2. Step 7 메뉴 설명 확인 단계에서 hover 툴팁이 나오지 않음
3. Step 9 비주얼모드 예시는 레벨1이 아니라 "등록관리" 중분류 메뉴로 보여주는 것이 더 적절함
````

이번 수정은 **가이드 표시 방식만 수정**하는 작업입니다.

기존 메뉴 편집 기능, 드래그앤드랍, 저장, 제출, 로그 기능은 건드리지 말아 주세요.

# 1. Step 5 리스트모드 핸들 가이드 수정

## 현재 문제

Step 5에서 리스트모드 핸들을 설명할 때,
가이드 설명창이 강조 대상과 너무 겹치거나 아래쪽 메뉴 영역을 가립니다.

또한 마우스/포인터 애니메이션이 하이라이트 뒤에서 실행되어 잘 보이지 않습니다.

현재 증상:

```txt
핸들 강조는 보임
하지만 설명창이 메뉴 영역과 겹침
포인터 애니메이션이 하이라이트 뒤에 깔림
사용자가 어디를 잡아야 하는지 명확하지 않음
```

---

## 수정 요청

Step 5에서는 설명창과 포인터 애니메이션이 반드시 잘 보여야 합니다.

### 설명창 위치

Step 5의 설명창은 강조 대상과 겹치지 않도록 배치해 주세요.

권장 위치:

```txt
대상 아이템 위쪽 또는 왼쪽 위
```

단, 아래 조건을 반드시 지켜 주세요.

```txt
설명창이 드래그 핸들을 가리지 않음
설명창이 아이템 전체를 덮지 않음
설명창이 마우스 애니메이션 경로를 가리지 않음
```

---

## 포인터 애니메이션 z-index 수정

마우스 포인터 애니메이션은 하이라이트보다 위에 보여야 합니다.

권장 z-index:

```txt
하이라이트: 9010
포인터/마우스 애니메이션: 9020
가이드 설명창: 9030
```

현재 포인터가 뒤에 깔려 보이므로 아래처럼 수정해 주세요.

```css
.guide-pointer,
.guide-cursor,
.guide-hand-animation {
  z-index: 9020;
  position: fixed;
  pointer-events: none;
}
```

하이라이트 영역보다 낮은 z-index를 쓰고 있다면 반드시 수정해 주세요.

---

## Step 5 애니메이션 방식

Step 5는 아이템 전체가 아니라 **핸들 위치**를 기준으로 안내해야 합니다.

원하는 방식:

```txt
1. 핸들 버튼만 pulse 강조
2. 포인터가 핸들 중앙으로 이동
3. 포인터가 핸들을 누르는 모션
4. 위/아래로 아주 짧게 드래그하는 모션
5. 실제 menuState는 변경하지 않음
```

---

## Step 5 기대 결과

```txt
핸들 위치가 명확히 보임
포인터 애니메이션이 하이라이트 위에서 보임
설명창이 핸들과 애니메이션을 가리지 않음
사용자가 "이 점 모양 핸들을 잡는구나"를 바로 이해함
```

# 2. Step 7 메뉴 설명 확인 hover 툴팁 수정

## 현재 문제

Step 7 `메뉴 설명 확인` 단계에서
메뉴명에 hover된 것처럼 보이긴 하지만 실제 설명 툴팁이 나오지 않습니다.

원래 의도는 다음과 같습니다.

```txt
메뉴명 위에 마우스를 올림
→ 메뉴 설명 툴팁이 표시됨
→ 사용자가 어려운 메뉴의 설명을 확인할 수 있음을 이해
```

현재는 hover 툴팁이 안 나와서 가이드 의미가 약합니다.

---

## 수정 요청

Step 7에서는 반드시 메뉴 설명 툴팁이 보이도록 해 주세요.

### 원하는 동작

```txt
1. 포인터 아이콘이 메뉴명 위로 이동
2. 해당 메뉴명이 hover 상태처럼 강조
3. 메뉴 설명 툴팁이 표시됨
4. 가이드 설명창은 유지
5. 툴팁과 설명창이 서로 가리지 않도록 위치 조정
```

---

## 중요 수정

이전 요청에서 말한 것은 **가이드 설명창을 끄라는 뜻이 아닙니다.**

원하는 동작은 아래입니다.

```txt
가이드 설명창은 유지
메뉴 설명 툴팁도 표시
둘이 겹치면 툴팁 위치를 조정
```

또는 구현상 겹침 문제가 심하면:

```txt
기존 검은 툴팁 대신 가이드 전용 tooltip을 표시
```

해도 됩니다.

---

## Step 7 툴팁 구현 방식

### 방식 A: 실제 desc tooltip 표시

가능하면 기존 desc tooltip 기능을 그대로 사용해 주세요.

```txt
guideActive && currentStep === "menu-tooltip"
→ target menu에 guide-hover 상태 부여
→ desc tooltip 강제 표시
```

예시:

```jsx
const forceShowDescTooltip =
  guideActive && currentGuideStep?.id === "menu-tooltip";
```

```jsx
<DescTooltip
  visible={isHovered || forceShowDescTooltip}
  desc={item.desc}
/>
```

---

### 방식 B: 가이드 전용 tooltip 표시

실제 hover 이벤트 구현이 어렵다면,
가이드 전용 tooltip을 target 메뉴명 아래에 표시해도 됩니다.

```txt
가이드 전용 tooltip은 실제 데이터를 변경하지 않음
desc 내용 또는 예시 설명을 표시
기존 desc tooltip 로직은 건드리지 않음
```

예시 문구:

```txt
나의 학사정보를 확인하고, 학사 관련 업무를 처리하는 메뉴들이 모여 있어요.
```

---

## Step 7 위치 조건

툴팁이 보이도록 설명창 위치를 조정해 주세요.

```txt
설명창은 툴팁을 가리지 않는 위치
툴팁은 메뉴명 근처
포인터는 하이라이트보다 위
```

권장:

```txt
가이드 설명창: 대상 위쪽 또는 오른쪽
메뉴 tooltip: 대상 아래쪽
```

---

## Step 7 기대 결과

```txt
메뉴명 hover 상태가 보임
설명 툴팁이 실제로 표시됨
가이드 설명창도 유지됨
둘이 서로 가리지 않음
```

# 3. Step 9 비주얼모드 예시 대상 변경

## 현재 문제

Step 9 비주얼모드 편집 가이드가 상단 레벨1 메뉴를 대상으로 잡혀 있습니다.

그런데 레벨1 영역은 파란 배경이고, 하이라이트/포인터도 파란 계열이라 가시성이 떨어집니다.

또한 레벨1은 비주얼모드에서 직접적인 예시 대상으로 보기 어렵습니다.

---

## 수정 요청

Step 9 비주얼모드 편집 예시는 **레벨1이 아니라 중분류 메뉴인 `등록관리`를 대상으로** 해 주세요.

대상:

```txt
등록관리
```

레벨:

```txt
L2 / 중분류
```

---

## 이유

```txt
등록관리는 흰 배경 위의 중분류 메뉴라 하이라이트가 잘 보임
사용자가 비주얼모드에서 메뉴를 어떻게 움직이는지 더 쉽게 이해함
레벨1보다 실제 이동/정렬 예시로 적합함
```

---

## Step 9 target 지정

Step 9에서는 아래처럼 `등록관리` 메뉴에 data-guide-id를 붙이거나,
해당 title/code 기준으로 target을 찾아 주세요.

권장:

```jsx
<div data-guide-id="visual-mode-sample-l2">
  등록관리
</div>
```

또는:

```txt
targetTitle: "등록관리"
targetLevel: 2
```

---

## Step 9 설명 문구 예시

```txt
비주얼모드에서도 점 모양 핸들을 잡고 원하는 위치에 놓으면 됩니다.
중분류 메뉴는 같은 대분류 안에서 순서를 바꾸거나, 다른 대분류로 이동할 수 있어요.
```

또는:

```txt
예시로 "등록관리" 같은 중분류 메뉴를 잡고 이동할 수 있습니다.
```

---

## Step 9 애니메이션

Step 9에서도 아이템 전체가 아니라 **등록관리 메뉴의 핸들 위치**를 기준으로 포인터 애니메이션을 보여주세요.

```txt
1. 등록관리 메뉴의 핸들 강조
2. 포인터가 등록관리 핸들로 이동
3. 짧게 드래그하는 시각 효과
4. 실제 menuState는 변경하지 않음
```

## Step 9 기대 결과

```txt
비주얼모드 가이드 대상이 등록관리 중분류 메뉴로 표시됨
하이라이트와 포인터가 잘 보임
레벨1 파란 배경 위에서 잘 안 보이는 문제 해결
중분류 이동 방식이 더 명확하게 전달됨
```

# 4. 공통 z-index 재확인

Step 5, Step 7, Step 9 모두 포인터가 하이라이트 뒤에 깔리면 안 됩니다.

아래 기준을 다시 확인해 주세요.

```txt
dim overlay: 9000
하이라이트: 9010
포인터/마우스 애니메이션: 9020
가이드 tooltip: 9025
가이드 설명창: 9030
가이드 버튼: 9040
```

특히 포인터/마우스 애니메이션은 반드시 `position: fixed` 또는 적절한 overlay layer에 있어야 합니다.
