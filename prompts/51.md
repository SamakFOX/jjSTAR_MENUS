# 변경 로그에 되돌리기/다시실행 추가 및 최종제출 화면 가이드 재표시 오류 수정 요청

## 작업 목표

현재 두 가지 수정이 필요합니다.

```txt
1. 되돌리기 / 다시실행 작업도 변경 로그에 남기기
2. 최종제출 화면에서 가이드가 다시 뜨는 문제 수정
````

기존에 정상 동작하는 기능은 건드리지 말고, 아래 두 부분만 수정해 주세요.

---

# 1. 되돌리기 / 다시실행도 변경 로그에 남기기

## 현재 문제

메뉴 이동, 이름 변경, 삭제 등은 변경 로그에 남지만,
`되돌리기`와 `다시실행`은 로그에 남지 않습니다.

하지만 관리자 화면에서 사용자가 어떤 작업 흐름을 거쳤는지 확인하려면
되돌리기/다시실행도 로그에 남는 것이 좋습니다.

---

## 수정 요청

사용자가 `되돌리기` 버튼을 눌렀을 때와 `다시실행` 버튼을 눌렀을 때도 `change_log`에 기록해 주세요.

---

## 로그 메시지 예시

### 되돌리기

```json
{
  "type": "undo",
  "message": "사용자가 마지막 작업을 되돌렸습니다.",
  "createdAt": "2026-05-28T04:00:00.000Z"
}
```

조금 더 자세히 가능하다면:

```json
{
  "type": "undo",
  "message": "사용자가 \"수강편람\" 메뉴 이동 작업을 되돌렸습니다.",
  "targetLogType": "move",
  "targetMessage": "\"수강편람\" 메뉴가 \"수강관리\" 하위에서 \"교육과정\" 하위로 이동했습니다.",
  "createdAt": "2026-05-28T04:00:00.000Z"
}
```

---

### 다시실행

```json
{
  "type": "redo",
  "message": "사용자가 되돌린 작업을 다시 실행했습니다.",
  "createdAt": "2026-05-28T04:01:00.000Z"
}
```

조금 더 자세히 가능하다면:

```json
{
  "type": "redo",
  "message": "사용자가 \"수강편람\" 메뉴 이동 작업을 다시 실행했습니다.",
  "targetLogType": "move",
  "targetMessage": "\"수강편람\" 메뉴가 \"수강관리\" 하위에서 \"교육과정\" 하위로 이동했습니다.",
  "createdAt": "2026-05-28T04:01:00.000Z"
}
```

---

## 구현 방향

현재 undo/redo 로직을 크게 바꾸지 말고,
버튼 실행 성공 시점에 로그만 추가해 주세요.

예시:

```js
function createUndoLog(targetLog) {
  return {
    type: "undo",
    message: targetLog?.message
      ? `사용자가 "${targetLog.message}" 작업을 되돌렸습니다.`
      : "사용자가 마지막 작업을 되돌렸습니다.",
    targetLogType: targetLog?.type || null,
    targetMessage: targetLog?.message || null,
    createdAt: new Date().toISOString(),
  };
}
```

```js
function createRedoLog(targetLog) {
  return {
    type: "redo",
    message: targetLog?.message
      ? `사용자가 "${targetLog.message}" 작업을 다시 실행했습니다.`
      : "사용자가 되돌린 작업을 다시 실행했습니다.",
    targetLogType: targetLog?.type || null,
    targetMessage: targetLog?.message || null,
    createdAt: new Date().toISOString(),
  };
}
```

---

## 주의사항

되돌리기/다시실행 로그를 남긴다고 해서 undo/redo 스택이 꼬이면 안 됩니다.

즉, 아래는 유지해야 합니다.

```txt
되돌리기 클릭 → menuState는 이전 상태로 복구
다시실행 클릭 → menuState는 다시 실행 상태로 복구
change_log에는 undo/redo 기록 추가
undo/redo 카운트는 기존 방식 유지
```

단, undo/redo 로그 추가 때문에 다시 undo stack이 증가하거나 redo stack이 초기화되면 안 됩니다.

```txt
undo/redo 로그 추가는 편집 히스토리 변경으로 취급하지 않기
```

즉, `commitMenuChange()`를 통해 로그를 추가하지 말고,
별도의 `appendChangeLogOnly()` 같은 함수로 로그만 추가하는 것이 안전합니다.

---

## 권장 함수

```js
function appendChangeLogOnly(logItem) {
  setChangeLog((prev) => [
    ...prev,
    {
      ...logItem,
      createdAt: logItem.createdAt || new Date().toISOString(),
    },
  ]);

  setIsDirty(true);
}
```

되돌리기/다시실행 시:

```js
appendChangeLogOnly(createUndoLog(lastActionLog));
```

```js
appendChangeLogOnly(createRedoLog(redoneActionLog));
```

# 2. 최종제출 화면에서 가이드가 다시 뜨는 문제 수정

## 현재 문제

가이드는 한 번 본 사람에게 다시 뜨지 않도록 구현되어 있습니다.

그런데 유독 `최종제출 화면`으로 넘어갈 때는 아래 사용자에게도 가이드가 다시 표시됩니다.

```txt
임시저장이 있는 사용자
최종제출이 있는 사용자
닫기 버튼을 한 번 누른 사용자
다시 보지 않기를 누른 사용자
```

다른 화면에서는 가이드가 안 뜨는데, 최종제출 화면에서만 다시 뜹니다.

---

## 원인 추정

최종제출 화면 가이드가 전체 가이드 상태와 별도로 관리되고 있을 가능성이 큽니다.

예상 문제:

```txt
submit 화면 진입 시 showSubmitGuide를 무조건 true로 설정
hideGuide:${authCode} 값을 확인하지 않음
guideDismissed 상태를 확인하지 않음
draft/submission 존재 여부를 확인하지 않음
전체 GUIDE_STEPS와 submit guide가 분리되어 있음
```

예를 들어 아래 같은 코드가 있으면 문제입니다.

```js
useEffect(() => {
  if (submitStep === "submitForm") {
    setShowSubmitGuide(true);
  }
}, [submitStep]);
```

이러면 사용자가 가이드를 이미 껐어도 최종제출 화면에 들어갈 때마다 다시 뜹니다.

---

## 수정 요청

최종제출 화면 가이드도 전체 가이드 표시 규칙을 동일하게 따라야 합니다.

```txt
hideGuide:${authCode} === true 이면 표시 안 함
사용자가 X/닫기/다시 보지 않기를 누른 경우 표시 안 함
draft가 있거나 이미 작업 이력이 있으면 자동 표시 안 함
최종제출 화면에 진입했다는 이유만으로 자동 표시하지 않음
```

---

## 기대 동작

| 상황                      | 최종제출 화면 가이드 |
| ----------------------- | ----------- |
| 처음 접속한 신규 사용자, 가이드 진행 중 | 표시 가능       |
| 가이드를 X로 닫은 사용자          | 표시 안 함      |
| 다시 보지 않기 누른 사용자         | 표시 안 함      |
| 임시저장 데이터 있는 사용자         | 자동 표시 안 함   |
| 최종제출 데이터 있는 사용자         | 자동 표시 안 함   |
| 사용 방법 보기로 가이드 재시작한 경우   | 표시 가능       |
| 전체 튜토리얼 흐름 중 최종제출 단계 도달 | 표시 가능       |

---

## 핵심 원칙

최종제출 화면 가이드는 독립적으로 자동 실행하지 말고,
전체 튜토리얼 상태 안에서만 실행되게 해 주세요.

즉:

```txt
guideActive === true
AND currentGuideStep이 submit 관련 step
```

일 때만 최종제출 화면 가이드를 보여주세요.

나쁜 방식:

```js
if (submitStep === "submitForm") {
  setShowSubmitGuide(true);
}
```

좋은 방식:

```js
const shouldShowSubmitGuide =
  guideActive &&
  currentGuideStep?.screen === "submit";
```

---

## 닫기 처리 통합

가이드의 X 버튼, 건너뛰기, 다시 보지 않기 버튼은 모든 화면에서 동일하게 동작해야 합니다.

```js
function closeGuide() {
  setGuideActive(false);
  setGuideStep(0);
}
```

```js
function dontShowGuideAgain() {
  localStorage.setItem(`hideGuide:${authCode}`, "true");
  setGuideActive(false);
  setGuideStep(0);
}
```

최종제출 화면에서도 별도 `closeSubmitGuide()`를 만들기보다
공통 `closeGuide()`를 사용해 주세요.

---

## 자동 표시 조건 정리

가이드 자동 표시는 아래 조건에서만 실행되게 해 주세요.

```js
const guideHidden =
  localStorage.getItem(`hideGuide:${authCode}`) === "true";

const shouldAutoStartGuide =
  isAuthenticated &&
  !isAdmin &&
  !guideHidden &&
  !hasDraft &&
  !hasSubmission &&
  !hasStartedEditing;
```

그리고 자동 시작은 로그인 직후 한 번만 실행해 주세요.

```js
useEffect(() => {
  if (shouldAutoStartGuide) {
    startGuideFromBeginning();
  }
}, [shouldAutoStartGuide]);
```

최종제출 화면 진입만으로 `startGuideFromBeginning()` 또는 `setGuideActive(true)`를 호출하지 마세요.

---

## 사용 방법 보기 버튼 예외

사용자가 직접 `사용 방법 보기`를 누르면
`hideGuide:${authCode}`가 true여도 가이드를 다시 볼 수 있어야 합니다.

```js
function handleOpenGuideManually() {
  setGuideActive(true);
  setGuideStep(0);
}
```

이 경우만 예외적으로 최종제출 화면 가이드도 표시될 수 있습니다.

# 3. 최종 기대 결과

| --------------- | ---------------------- |
| 되돌리기 클릭         | change_log에 undo 로그 추가 |

| undo/redo 로그 추가 | undo/redo 스택 꼬임 없음     |
| 관리자 화면          | undo/redo 기록도 확인 가능    |
| 임시저장/최종제출       | undo/redo 로그 포함 저장     |

| 상황                | 기대 결과              |
| ----------------- | ------------------ |
| 가이드 닫은 사용자        | 최종제출 화면에서도 가이드 안 뜸 |
| 다시 보지 않기 사용자      | 최종제출 화면에서도 가이드 안 뜸 |
| draft 있는 사용자      | 자동 가이드 안 뜸         |
| submission 있는 사용자 | 자동 가이드 안 뜸         |

| 전체 튜토리얼 진행 중      | 최종제출 단계에서만 표시      |

# 4. 핵심 요약

이번 수정은 두 가지입니다.

```txt
1. 되돌리기/다시실행도 change_log에 기록
2. 최종제출 화면 가이드가 독립적으로 다시 뜨는 문제 수정
```

특히 최종제출 화면 가이드는
`submitStep === "submitForm"`만 보고 자동 표시하지 말고,
전체 가이드 상태를 따라야 합니다.

```txt
guideActive가 true이고 현재 step이 submit 단계일 때만 표시
```

그리고 가이드를 닫았거나 다시 보지 않기 한 사용자는
최종제출 화면에서도 다시 보이지 않아야 합니다.

```
