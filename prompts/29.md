# 초기화 시 되돌리기/다시실행 히스토리도 함께 초기화 요청

## 현재 상태

현재 메뉴 편집 화면은 전반적으로 잘 동작합니다.  
레이아웃, 우측 편집 도구 배치, 한글 표시, 되돌리기/다시실행 기능은 현재 상태를 유지해 주세요.

다만 한 가지 수정이 필요합니다.

---

## 수정 요청

`초기화` 버튼을 눌렀을 때 메뉴 위치와 구조가 초기 상태로 돌아가는데,  
이때 `되돌리기`와 `다시실행` 히스토리도 함께 초기화되어야 합니다.

현재는 메뉴는 초기 상태로 돌아가더라도,  
되돌리기/다시실행 카운트가 남아 있는 경우가 있습니다.

초기화는 사용자가 “처음 상태로 완전히 되돌리는 작업”이므로,  
이후에는 이전 편집 이력을 다시 되돌리거나 다시 실행할 수 없어야 합니다.

---

## 기대 동작

초기화 버튼 클릭 전 예시:

```txt
되돌리기 (4)
다시실행 (10)
````

초기화 버튼 클릭 후:

```txt
되돌리기 (0)
다시실행 (0)
```

그리고 메뉴 구조는 초기 데이터 상태로 복원되어야 합니다.

---

## 구현 기준

초기화 처리 함수에서 메뉴 상태를 초기값으로 되돌릴 때,
undoStack과 redoStack도 같이 비워 주세요.

예시:

```js
function handleReset() {
  setMenuState(initialMenuState);

  setUndoStack([]);
  setRedoStack([]);

  showToast("초기 상태로 복원되었습니다.");
}
```

만약 히스토리를 별도 객체로 관리하고 있다면 아래처럼 처리해 주세요.

```js
function handleReset() {
  setMenuState(initialMenuState);

  setHistory({
    undoStack: [],
    redoStack: [],
  });

  showToast("초기 상태로 복원되었습니다.");
}
```

---

## 주의 사항

이번 수정에서는 아래 항목은 변경하지 말아 주세요.

```txt
레이아웃 변경 금지
버튼 위치 변경 금지
우측 편집 도구 박스 구조 변경 금지
되돌리기/다시실행 기본 로직 변경 금지
리스트/비주얼 편집 동작 변경 금지
한글 문구 변경 금지
```

수정 범위는 오직 다음 하나입니다.

```txt
초기화 버튼 실행 시 undoStack, redoStack도 함께 초기화
```

## 최종 기대 결과

| 동작             | 기대 결과               |
| -------------- | ------------------- |
| 메뉴 여러 번 수정     | 되돌리기/다시실행 카운트 정상 표시 |
| 초기화 클릭         | 메뉴가 초기 상태로 복원       |
| 초기화 직후         | 되돌리기 (0), 다시실행 (0)  |
| 초기화 이후 되돌리기 클릭 | 아무 작업도 발생하지 않음      |
| 초기화 이후 다시실행 클릭 | 아무 작업도 발생하지 않음      |

---

## 핵심 요약

초기화는 단순히 메뉴 데이터만 되돌리는 것이 아니라,
편집 이력까지 모두 비우는 완전 초기화로 동작해야 합니다.

따라서 `초기화` 실행 시 다음 3가지를 함께 처리해 주세요.

```txt
1. 메뉴 상태를 초기 상태로 복원
2. 되돌리기 히스토리 초기화
3. 다시실행 히스토리 초기화
```
