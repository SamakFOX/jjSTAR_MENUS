# Visual Editor Mode 전용 드래그 제한 수정 요청

## 중요 전제

현재 **List Mode는 정상 동작하고 있으므로 절대 수정하지 마세요.**

이번 수정 범위는 **Visual Editor Mode 전용**입니다.

- List Mode drag/drop 로직 수정 금지
- List Mode reorder 로직 수정 금지
- List Mode 상태 업데이트 함수 수정 금지
- List Mode 검증 로직 수정 금지

Visual Editor Mode에서만 발생하는 드래그/드롭 문제를 수정해주세요.

---

# 핵심 수정 방향

현재 Visual Editor Mode에서 Level 2 메뉴를 드래그하면  
Level 3 메뉴 쪽으로 들어가려는 모션이나 하위 삽입 판정이 발생합니다.

하지만 **Level 2는 Level 3의 하위로 들어갈 일이 절대 없습니다.**

따라서 Level 2를 드래그할 때는  
아래 두 가지 동작만 가능해야 합니다.

```txt
1. 같은 Level 2끼리 순서 정렬
2. 다른 Level 1 아래로 이동
````

그 외 동작은 UI 모션 자체가 나오지 않도록 막아주세요.

---

# Level 2 드래그 동작 제한

## Level 2에서 허용되는 동작

Level 2 메뉴를 잡고 드래그했을 때 허용되는 동작은 아래뿐입니다.

| 동작                           | 허용 여부 |
| ---------------------------- | ----- |
| 같은 Level 2끼리 순서 변경           | 허용    |
| 같은 Level 1 안에서 Level 2 순서 변경 | 허용    |
| 다른 Level 1 아래로 이동            | 허용    |
| Level 3 아래로 들어가기             | 금지    |
| 다른 Level 2 아래로 들어가기          | 금지    |
| Level 3과 하위 삽입 모션 발생         | 금지    |
| Level 4 생성                   | 절대 금지 |

---

# Level 2 드래그 시 UI 동작

Level 2를 드래그 중일 때는
**Level 3 항목이 drop target처럼 반응하면 안 됩니다.**

## Level 2를 잡고 있을 때 Level 3 항목에서는 금지

* hover highlight 표시 금지
* 하위 삽입 가능 표시 금지
* move group 표시 금지
* 들여쓰기 preview 금지
* Level 3 아래로 들어가는 듯한 animation 금지
* placeholder 생성 금지
* drop해도 아무 변화 없어야 함

즉, Level 2를 드래그 중일 때 Level 3은
**완전히 무시되는 drop target**이어야 합니다.

---

# Level 2 드래그 시 허용되는 drop target

Level 2를 드래그 중일 때 drop target은 아래로 제한합니다.

```txt
Level 1 영역
같은 parentId를 가진 Level 2 사이의 정렬 위치
다른 Level 1 아래의 Level 2 정렬 위치
```

Level 3은 drop target 후보에서 제외해주세요.

# Visual Mode 전용 판정 함수 예시

List Mode에 영향이 가지 않도록
Visual Mode 전용 함수로 작성해주세요.

```js
function getVisualAllowedDropModes(dragItem, targetItem) {
  // Level 1
  if (dragItem.level === 1) {
    return {
      canReorder: targetItem.level === 1,
      canMoveAsChild: false,
    };
  }

  // Level 2
  if (dragItem.level === 2) {
    return {
      // Level 2끼리만 순서 정렬 가능
      canReorder: targetItem.level === 2,

      // Level 2는 Level 1 아래로만 이동 가능
      canMoveAsChild: targetItem.level === 1,
    };
  }

  // Level 3
  if (dragItem.level === 3) {
    return {
      // Level 3끼리만 순서 정렬 가능
      canReorder: targetItem.level === 3,

      // Level 3은 Level 2 아래로만 이동 가능
      canMoveAsChild: targetItem.level === 2,
    };
  }

  return {
    canReorder: false,
    canMoveAsChild: false,
  };
}
```

# Level 2 전용 방어 코드

특히 Level 2는 아래 조건을 강하게 적용해주세요.

```js
function canVisualLevel2Drop(dragItem, targetItem, dropMode) {
  if (dragItem.level !== 2) return true;

  // Level 2는 Level 3과 어떤 drop 상호작용도 하면 안 됨
  if (targetItem.level === 3) {
    return false;
  }

  // Level 2를 하위로 넣는 경우는 Level 1 아래만 허용
  if (dropMode === 'move-as-child') {
    return targetItem.level === 1;
  }

  // Level 2 순서 정렬은 Level 2끼리만 허용
  if (dropMode === 'reorder') {
    return targetItem.level === 2;
  }

  return false;
}
```

# Drop 처리 전 필수 검증

실제 상태 변경 전에 반드시 검증해주세요.

```js
function handleVisualDrop(dragItem, targetItem, dropMode) {
  // Visual Editor Mode 전용
  // List Mode에서는 사용하지 않음

  if (dragItem.level === 2) {
    const isAllowed = canVisualLevel2Drop(dragItem, targetItem, dropMode);

    if (!isAllowed) {
      console.warn('Invalid Level 2 visual drop blocked:', {
        dragLevel: dragItem.level,
        targetLevel: targetItem.level,
        dropMode,
        dragTitle: dragItem.title,
        targetTitle: targetItem.title,
      });

      return;
    }
  }

  // 기존 Visual Mode 검증
  if (dropMode === 'move-as-child') {
    if (!canVisualDropAsChild(dragItem, targetItem)) {
      return;
    }
  }

  if (dropMode === 'reorder') {
    if (!canVisualReorder(dragItem, targetItem)) {
      return;
    }
  }

  applyVisualDrop(dragItem, targetItem, dropMode);
}
```

# Level 2 드래그 중 hover 처리

Level 2를 드래그 중이고 target이 Level 3이면
hover 상태도 만들지 말아주세요.

```js
function shouldShowVisualDropPreview(dragItem, targetItem, dropMode) {
  // Level 2는 Level 3과 상호작용하지 않음
  if (dragItem.level === 2 && targetItem.level === 3) {
    return false;
  }

  if (dropMode === 'move-as-child') {
    return canVisualDropAsChild(dragItem, targetItem);
  }

  if (dropMode === 'reorder') {
    return canVisualReorder(dragItem, targetItem);
  }

  return false;
}
```

# Level 2 드래그 동작 요약

Level 2 메뉴를 드래그할 때는 아래처럼 동작해야 합니다.

```txt
Level 2 → Level 2 사이에 놓기 = 순서 변경
Level 2 → 다른 Level 1 영역에 놓기 = 해당 Level 1 하위로 이동
Level 2 → Level 3 위에 놓기 = 아무 반응 없음
Level 2 → Level 3 아래로 들어가는 모션 = 표시 금지
Level 2 → Level 2 하위로 들어가는 모션 = 표시 금지
```

# 완료 기준

아래 테스트가 모두 통과해야 합니다.

| 테스트                                | 기대 결과                   |
| ---------------------------------- | ----------------------- |
| Level 2를 같은 Level 2 사이로 드래그        | 순서 변경 가능                |
| Level 2를 같은 Level 1 내부의 다른 위치로 드래그 | 순서 변경 가능                |
| Level 2를 다른 Level 1 영역으로 드래그       | 다른 Level 1 하위로 이동 가능    |
| Level 2를 Level 3 위로 드래그            | hover / drop preview 없음 |
| Level 2를 Level 3 위에 drop           | 아무 변화 없음                |
| Level 2가 Level 3 아래로 들어가는 애니메이션    | 발생하면 안 됨                |
| Level 2가 Level 2 아래로 들어가는 애니메이션    | 발생하면 안 됨                |
| 이동 후 전체 메뉴 구조                      | Level 1, 2, 3까지만 존재     |
| 이동 후 전체 메뉴 구조                      | Level 4 절대 생성 금지        |
| List Mode 테스트                      | 기존처럼 정상 동작해야 함          |
