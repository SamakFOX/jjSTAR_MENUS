# Next.js Hydration 오류 수정 요청

## 문제 상황

현재 Next.js 프로젝트에서 로그인된 상태로 새로고침하면 Hydration 오류가 발생합니다.

로그아웃 상태에서 홈 화면을 새로고침하면 정상입니다.  
하지만 로그인 또는 인증 완료 상태에서 새로고침하면 아래와 같은 오류가 발생합니다.

```txt
Hydration failed because the server rendered HTML didn't match the client.
````

오류 화면에서는 서버와 클라이언트가 서로 다른 HTML 구조를 렌더링한 것으로 표시됩니다.

```diff
+ <div className="min-h-screen flex flex-col bg-[#f5f6fb]">
- <main className="max-w-4xl mx-auto px-6 py-20 flex flex-col items-center text-center">
```

즉, 서버에서는 인증 전 화면인 `<main>` 구조를 렌더링하고,
클라이언트에서는 인증 후 화면인 `<div>` 구조를 렌더링하면서 Hydration 충돌이 발생하는 것으로 보입니다.

---

## 추정 원인

인증 상태를 `localStorage`, `sessionStorage`, `window` 등 브라우저 전용 API를 통해 바로 판단하고 있는 것이 원인으로 보입니다.

예상되는 문제 코드 예시는 다음과 같습니다.

```jsx
const [isAuthenticated, setIsAuthenticated] = useState(
  localStorage.getItem("authCodeVerified") === "true"
);
```

또는

```jsx
const isAuthenticated =
  typeof window !== "undefined" &&
  localStorage.getItem("authCodeVerified") === "true";
```

서버 렌더링 시점에는 `localStorage`를 읽을 수 없기 때문에 서버는 기본적으로 인증되지 않은 상태로 판단합니다.
반면 클라이언트에서는 새로고침 후 `localStorage`에 남아 있는 인증값을 읽고 인증된 화면을 렌더링합니다.

그 결과 서버 HTML과 클라이언트 HTML이 달라져 Hydration 오류가 발생합니다.

---

## 수정 목표

로그인 상태에서 새로고침해도 Hydration 오류가 발생하지 않도록 수정해 주세요.

핵심 목표는 다음과 같습니다.

1. 서버 렌더링 결과와 클라이언트의 첫 렌더링 결과가 달라지지 않도록 처리
2. `localStorage` 기반 인증값은 반드시 클라이언트 마운트 이후에만 읽기
3. 인증 전/후 화면의 최상위 구조가 급격히 달라지지 않도록 정리
4. 로그인 상태 새로고침 시에도 정상적으로 인증 후 화면으로 이동
5. 로그아웃 후 새로고침 시에는 정상적으로 인증 전 화면 유지

---

## 필수 수정 사항

### 1. 인증 상태 초기값을 localStorage에서 바로 읽지 않기

아래와 같은 방식은 사용하지 말아 주세요.

```jsx
const [isAuthenticated, setIsAuthenticated] = useState(
  localStorage.getItem("authCodeVerified") === "true"
);
```

대신 인증 상태의 초기값은 안전하게 `false`로 두고,
`useEffect` 안에서만 `localStorage`를 읽어 주세요.

```jsx
const [mounted, setMounted] = useState(false);
const [isAuthenticated, setIsAuthenticated] = useState(false);

useEffect(() => {
  const savedAuth = localStorage.getItem("authCodeVerified");

  if (savedAuth === "true") {
    setIsAuthenticated(true);
  }

  setMounted(true);
}, []);
```

---

### 2. mounted 처리 추가

클라이언트에서 인증 상태 확인이 끝나기 전에는 화면을 렌더링하지 않거나, 안전한 로딩 화면을 보여 주세요.

```jsx
if (!mounted) {
  return null;
}
```

또는 필요하다면 로딩 UI를 사용할 수 있습니다.

```jsx
if (!mounted) {
  return (
    <div className="min-h-screen bg-[#f5f6fb]" />
  );
}
```

---

### 3. 최상위 HTML 구조를 동일하게 유지

현재는 인증 전 화면과 인증 후 화면의 최상위 태그가 서로 다른 것으로 보입니다.

문제가 되는 구조 예시:

```jsx
if (!isAuthenticated) {
  return (
    <main className="max-w-4xl mx-auto px-6 py-20 flex flex-col items-center text-center">
      인증 화면
    </main>
  );
}

return (
  <div className="min-h-screen flex flex-col bg-[#f5f6fb]">
    로그인 후 화면
  </div>
);
```

위 구조는 서버와 클라이언트 렌더링 결과가 달라질 가능성이 높습니다.

가능하면 아래처럼 최상위 wrapper를 항상 동일하게 유지해 주세요.

```jsx
return (
  <div className="min-h-screen bg-[#f5f6fb]">
    {!mounted ? null : !isAuthenticated ? (
      <main className="max-w-4xl mx-auto px-6 py-20 flex flex-col items-center text-center">
        인증 화면
      </main>
    ) : (
      <div className="min-h-screen flex flex-col">
        로그인 후 화면
      </div>
    )}
  </div>
);
```

## 인증 성공 처리

인증 성공 시에는 상태와 `localStorage`를 함께 업데이트해 주세요.

```jsx
const handleLoginSuccess = () => {
  localStorage.setItem("authCodeVerified", "true");
  setIsAuthenticated(true);
};
```

현재 프로젝트에서 사용하는 키 이름이 `authCodeVerified`가 아니라면,
기존 인증 저장 키 이름에 맞춰 적용해 주세요.

---

## 로그아웃 처리

로그아웃 시에는 저장된 인증값을 반드시 제거해 주세요.

```jsx
const handleLogout = () => {
  localStorage.removeItem("authCodeVerified");
  setIsAuthenticated(false);
};
```

만약 인증 관련 키가 여러 개라면 모두 제거해 주세요.

예시:

```jsx
localStorage.removeItem("authCodeVerified");
localStorage.removeItem("isAuthenticated");
localStorage.removeItem("user");
```

---

## 수정 시 주의 사항

다음 코드는 렌더링 중 직접 사용하지 않도록 해 주세요.

```jsx
localStorage.getItem(...)
sessionStorage.getItem(...)
window
document
Date.now()
Math.random()
new Date().toLocaleString()
```

특히 아래와 같은 조건부 렌더링은 Hydration 오류를 만들 수 있으므로 피해야 합니다.

```jsx
if (typeof window !== "undefined") {
  return <div>클라이언트 화면</div>;
}

return <main>서버 화면</main>;
```

## 기대 동작

수정 후 아래 동작이 모두 정상이어야 합니다.

| 상황             | 기대 결과                       |
| -------------- | --------------------------- |
| 로그아웃 상태에서 새로고침 | 인증 전 홈 화면 정상 표시             |
| 로그인 상태에서 새로고침  | Hydration 오류 없이 로그인 후 화면 표시 |
| 인증 성공          | 인증 후 화면으로 전환                |
| 로그아웃 클릭        | 인증값 삭제 후 인증 전 화면 표시         |
| 다시 새로고침        | 인증 전 화면 유지                  |
| 브라우저 콘솔        | Hydration 오류 없음             |

---

## 핵심 요약

현재 오류는 로그인 상태를 브라우저 저장소에서 바로 읽으면서 서버 렌더링 HTML과 클라이언트 렌더링 HTML이 달라져 발생하는 것으로 보입니다.

따라서 다음 방식으로 수정해 주세요.

1. 인증 상태 초기값은 `false`
2. `localStorage`는 `useEffect` 안에서만 읽기
3. `mounted` 상태 추가
4. 인증 전/후 최상위 wrapper 구조 통일
5. 로그아웃 시 인증 관련 localStorage 값 완전 삭제

이 수정은 기존 UI 디자인이나 기능을 크게 바꾸지 않고, Hydration 오류만 해결하는 방향으로 적용해 주세요.

```
