# 로그인 화면/비주얼모드/프로필 이미지 수정 요청

## 수정 범위

이번 수정은 아래 3가지에만 한정해 주세요.

1. 로그인 화면 인증코드 입력박스 폰트 수정
2. 로그인 화면에서 Enter 키 입력 시 입장 처리
3. 비주얼모드 Level 1 라인 두께/가림 문제 수정
4. 미리보기 및 비주얼모드 프로필 이미지 경로 확인 및 적용

기존에 정상 동작 중인 드래그앤드랍 로직, 히스토리, 인증 처리, 레이아웃 구조는 건드리지 말아 주세요.

---

# 1. 로그인 화면 입력박스 폰트 수정

## 현재 문제

Pretendard WOFF2 폰트를 적용한 이후, 로그인 화면의 인증코드 입력박스 폰트가 이상하게 보입니다.

특히 인증코드 입력값이 일반 UI 텍스트와 다르게 어색한 고정폭/기본 브라우저 폰트처럼 보입니다.

## 수정 요청

로그인 화면의 인증코드 입력박스에도 전역 Pretendard 폰트가 정상 적용되도록 수정해 주세요.

### 적용 대상

```txt
로그인 화면 인증코드 input
placeholder
입력된 인증코드 텍스트
````

### 기대 스타일

```css
font-family: var(--font-pretendard), system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
letter-spacing: normal;
```

현재 인증코드 입력값에 `font-mono`, `monospace`, `tracking-widest`, `letter-spacing` 등이 적용되어 있다면 제거하거나 완화해 주세요.

## 예시 수정

```jsx
<input
  className="font-sans tracking-normal"
/>
```

또는 CSS로:

```css
.auth-code-input {
  font-family: var(--font-pretendard), system-ui, sans-serif;
  letter-spacing: normal;
}
```

# 2. 로그인 화면에서 Enter 키로 입장 가능하게 수정

## 현재 요청

인증코드를 입력한 뒤 버튼을 클릭하지 않아도,
키보드에서 Enter를 누르면 `입장하기` 버튼을 누른 것과 동일하게 동작해야 합니다.

## 기대 동작

```txt
인증코드 입력
→ Enter 키 입력
→ 인증 확인 실행
→ 인증 성공 시 메인 화면 진입
```

## 구현 방법

가능하면 로그인 입력 영역을 `<form>`으로 감싸고 `onSubmit`으로 처리해 주세요.

예시:

```jsx
<form onSubmit={handleLoginSubmit}>
  <input
    value={authCode}
    onChange={(e) => setAuthCode(e.target.value)}
  />

  <button type="submit">
    입장하기
  </button>
</form>
```

```js
function handleLoginSubmit(e) {
  e.preventDefault();
  handleLogin();
}
```

만약 form 구조를 쓰기 어렵다면 input에 `onKeyDown`을 추가해 주세요.

```jsx
<input
  value={authCode}
  onChange={(e) => setAuthCode(e.target.value)}
  onKeyDown={(e) => {
    if (e.key === "Enter") {
      e.preventDefault();
      handleLogin();
    }
  }}
/>
```

단, 중복 실행 방지를 위해 로딩 중에는 Enter가 여러 번 실행되지 않도록 처리해 주세요.

```js
if (isLoading) return;
```

# 3. 비주얼모드 Level 1 라인이 너무 얇아진 문제 수정

## 현재 문제

비주얼모드에서 상단 Level 1 메뉴가 나오는 라인이 너무 얇아졌습니다.

화면상으로는 메뉴 오버레이 또는 다른 레이어가 Level 1 라인을 가리는 것처럼 보입니다.

특히 아래 영역이 얇게 눌려 보입니다.

```txt
상단 파란색 Level 1 메뉴 바
star 학생지원 / 학사정보 / 대학생활 / 진로개발 / 취업지원 ...
```

## 수정 요청

비주얼모드에서 Level 1 메뉴 바가 기존 높이와 두께를 유지하도록 수정해 주세요.

### 확인할 것

```txt
Level 1 nav bar height
Level 1 item height
line-height
padding-y
overflow hidden 여부
메뉴 오버레이 z-index
상단 header / overlay / drag layer의 position
```

## 기대 상태

Level 1 메뉴 바는 다음처럼 충분한 높이를 가져야 합니다.

```txt
아이콘/드래그 핸들/메뉴명이 세로 중앙 정렬
텍스트가 눌리거나 잘리지 않음
파란색 바 높이가 기존처럼 유지됨
오버레이에 의해 가려지지 않음
```

## CSS 방향

필요하면 Level 1 nav bar에 최소 높이를 명시해 주세요.

```css
.visual-level1-bar {
  min-height: 34px;
  height: 34px;
  overflow: visible;
  position: relative;
  z-index: 20;
}
```

Level 1 아이템도 동일하게 높이와 정렬을 보장해 주세요.

```css
.visual-level1-item {
  min-height: 34px;
  display: flex;
  align-items: center;
}
```

오버레이가 Level 1 메뉴 바를 덮고 있다면 z-index를 조정해 주세요.

```css
.visual-menu-overlay {
  z-index: 30;
}

.visual-level1-bar {
  z-index: 40;
}
```

단, 드래그 중인 카드/DragOverlay가 필요한 경우에는 드래그 오버레이만 더 높은 z-index를 사용해도 됩니다.

```css
.drag-overlay {
  z-index: 100;
}
```

# 4. 미리보기/비주얼모드 프로필 이미지 경로 확인

## 현재 문제

미리보기와 비주얼모드에서 프로필 이미지가 정확히 적용되어야 합니다.

현재 프로젝트 파일 구조에서 프로필 이미지 파일은 아래 위치에 있습니다.

```txt
src/data/Profile_JeyJey.png
```

## 수정 요청

미리보기 화면과 비주얼모드 화면에서 사용하는 프로필 이미지 경로를 확인하고,
아래 파일이 정상적으로 표시되도록 수정해 주세요.

```txt
src/data/Profile_JeyJey.png
```

## 적용 대상

```txt
미리보기 화면의 좌측 프로필 이미지
비주얼모드의 좌측 프로필 이미지
```

## Next.js에서 적용 방식

이미지가 `src/data` 내부에 있으므로 직접 경로 문자열로 `/data/Profile_JeyJey.png`처럼 접근하면 안 될 수 있습니다.

가능하면 import 방식으로 사용해 주세요.

예시:

```jsx
import profileJeyJey from "@/data/Profile_JeyJey.png";
import Image from "next/image";

<Image
  src={profileJeyJey}
  alt="제이제이 프로필 이미지"
  width={96}
  height={96}
  className="rounded-full object-cover"
/>
```

만약 `next/image`를 사용하지 않는 구조라면 일반 img에서도 import된 src를 사용할 수 있습니다.

```jsx
import profileJeyJey from "@/data/Profile_JeyJey.png";

<img
  src={profileJeyJey.src}
  alt="제이제이 프로필 이미지"
  className="profile-image"
/>
```

## 주의

현재 파일 위치가 `src/data/Profile_JeyJey.png`이므로,
아래처럼 public 기준 경로로 쓰면 깨질 수 있습니다.

```jsx
<img src="/Profile_JeyJey.png" />
<img src="/data/Profile_JeyJey.png" />
```

이 방식은 파일이 `public` 폴더에 있을 때만 안정적으로 동작합니다.

현재 위치를 유지한다면 import 방식으로 적용해 주세요.
