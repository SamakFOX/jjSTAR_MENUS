# 비주얼모드 Drag & Drop 단순화 요청

## 작업 목표

비주얼모드 Drag & Drop 로직이 너무 복잡해지면서 불필요한 하이라이트, 금지 표시, 부모 삽입 판정, 드롭존 계산 문제가 계속 발생하고 있습니다.

이제 비주얼모드는 아래처럼 단순하게 구현해 주세요.

```txt
비주얼모드 Drag & Drop = 무조건 순서 정렬 중심
````

복잡한 기능을 덧붙이지 말고, 레벨별로 허용된 정렬/이동만 처리해 주세요.

---

## 제거할 것

현재 드래그 중 표시되는 노란 하이라이트는 제거해 주세요.

```txt
노란 하이라이트 제거
금지 표시 제거
복잡한 부모 삽입 표시 제거
Level 2를 Level 2 안에 넣는 것처럼 보이는 UI 제거
Level 3을 Level 3 안에 넣는 것처럼 보이는 UI 제거
```

비주얼모드에서는 사용자가 “이 안으로 넣는다”는 느낌을 받으면 안 됩니다.

대신 항상 아래 느낌이어야 합니다.

```txt
같은 레벨끼리 순서를 바꾼다
또는 허용된 부모 그룹으로 이동한 뒤 그 안에서 순서를 바꾼다
```

---

# 1. 전체 원칙

비주얼모드에서 드래그 동작은 레벨별로 고정합니다.

```txt
Level 1 → 상단 파란 Level 1 메뉴 줄에서만 순서 정렬
Level 2 → 현재 Level 1 내부에서 Level 2 순서 정렬 또는 다른 Level 1 아래로 이동 후 정렬
Level 3 → 다른 Level 2 아래로 이동 후 Level 3 순서 정렬
```

이 외 동작은 만들지 마세요.

---

# 2. Level 1 동작

## 허용 동작

Level 1은 상단 파란 메뉴 줄 안에서만 순서 정렬합니다.

예시:

```txt
[star 학생지원] [학사정보] [대학생활] [진로개발] [취업지원]
```

Level 1을 잡으면 이 파란 메뉴 바 안에서만 좌우 순서 변경이 가능해야 합니다.

## 금지/비활성

Level 1은 절대 아래 영역으로 내려가지 않습니다.

```txt
Level 2 영역으로 이동 금지
Level 3 영역으로 이동 금지
본문 영역으로 이동 금지
다른 메뉴의 하위로 이동 금지
```

단, 별도의 금지표시를 띄울 필요는 없습니다.

그냥 파란 Level 1 라인 밖에서는 드롭 후보가 없도록 처리해 주세요.

---

# 3. Level 2 동작

## 허용 동작

Level 2는 두 가지 동작만 허용합니다.

```txt
1. 현재 Level 1 내부에서 Level 2끼리 순서 정렬
2. 다른 Level 1으로 이동한 뒤 그 Level 1 내부에서 Level 2끼리 순서 정렬
```

예시:

```txt
학사정보 > 수강관리
```

를 잡고 다른 Level 1인

```txt
star 학생지원
```

으로 이동하면, `star 학생지원` Level 1이 열리고 그 안의 Level 2 목록에서 순서를 정할 수 있어야 합니다.

## 중요한 원칙

Level 2는 절대 Level 2 안으로 들어가지 않습니다.

```txt
Level 2 → Level 2 자식으로 삽입 금지
Level 2 → Level 3 영역으로 이동 금지
```

하지만 금지표시를 띄우지 말고, 애초에 그런 후보를 만들지 마세요.

즉, Level 2 드래그 중에는:

```txt
Level 1 탭/레이블 = 부모 전환용
Level 2 목록 = 순서 정렬용
Level 3 목록 = 무시
```

---

# 4. Level 3 동작

## 허용 동작

Level 3은 아래 동작만 허용합니다.

```txt
1. 같은 Level 2 내부에서 Level 3끼리 순서 정렬
2. 같은 Level 1 안의 다른 Level 2 아래로 이동 후 Level 3 순서 정렬
3. 다른 Level 1 안의 Level 2 아래로 이동 후 Level 3 순서 정렬
```

즉, Level 3은 다른 Level 2 아래로 이동할 수 있습니다.

예시:

```txt
학사정보 > 수강관리 > 수강편람
```

을

```txt
대학생활 > 공동체
```

아래로 옮길 수 있습니다.

이 경우 최종 구조는 다음처럼 되어야 합니다.

```txt
대학생활
 └ 공동체
    └ 수강편람
```

## 중요한 원칙

Level 3은 절대 Level 1 바로 아래로 들어가지 않습니다.

```txt
Level 3 → Level 1 바로 아래 이동 금지
Level 3 → Level 3 자식으로 삽입 금지
```

하지만 이 역시 금지표시를 띄우지 말고, 애초에 드롭 후보에서 제외해 주세요.

즉, Level 3 드래그 중에는:

```txt
Level 1 탭/레이블 = Level 1 전환용
Level 2 목록 = 부모 선택/전환용
Level 3 목록 = 순서 정렬용
```

---

# 5. 노란 하이라이트 제거

현재 특정 부모 그룹이 노란색으로 강조되는 UI가 있습니다.

이제 이 하이라이트는 필요 없습니다.

```txt
노란 하이라이트 제거
부모 박스 강조 제거
삽입 대상처럼 보이는 배경 강조 제거
```

비주얼모드는 “어떤 박스 안으로 넣는” 방식이 아니라,
항상 “해당 레벨의 목록에서 순서를 정하는” 방식으로 보여야 합니다.

필요한 시각 피드백은 아래 정도면 충분합니다.

```txt
드래그 중인 카드 표시
삽입될 위치 placeholder 표시
기존 아이템이 밀리는 표시
```

---

# 6. 금지표시 제거

금지표시도 제거해 주세요.

현재 금지표시나 잘못된 하이라이트 때문에 사용자가 오히려 헷갈립니다.

수정 후에는:

```txt
허용된 트랙만 반응
허용되지 않는 영역은 아무 반응 없음
```

이면 됩니다.

---

# 7. 드롭 후보 단순화

드래그 시작 시 아이템의 level을 보고, 그 level에 맞는 후보만 계산해 주세요.

```js
function getVisualDragMode(item) {
  if (item.level === 1) return "LEVEL_1_SORT";
  if (item.level === 2) return "LEVEL_2_SORT";
  if (item.level === 3) return "LEVEL_3_SORT";
}
```

각 모드별 후보:

```txt
LEVEL_1_SORT
→ 상단 Level 1 목록만 후보

LEVEL_2_SORT
→ 현재 activeLevel1의 Level 2 목록만 후보
→ 다른 Level 1 탭 위에 정확히 hover하면 activeLevel1 변경
→ 변경된 Level 1의 Level 2 목록에서 정렬

LEVEL_3_SORT
→ 현재 activeLevel2의 Level 3 목록만 후보
→ 다른 Level 2 위에 hover하면 activeLevel2 변경
→ 다른 Level 1의 Level 2로 이동하려면 Level 1 전환 후 Level 2 선택
```

---

# 8. Level 1 전환 기준

Level 2 또는 Level 3을 드래그 중 다른 Level 1로 이동할 수 있어야 합니다.

다만 Level 1 전환은 너무 민감하면 안 됩니다.

기준:

```txt
실제 마우스 포인터가 Level 1 탭 DOM 영역 안에 있을 때만 전환 후보
탭 주변 여백에서는 전환하지 않음
DragOverlay 충돌 기준으로 전환하지 않음
150~250ms 정도 hover delay 적용
```

---

# 9. Level 2 전환 기준

Level 3을 드래그 중에는 Level 2 위에 올라갔을 때 해당 Level 2가 부모 후보가 됩니다.

단, 이것도 복잡한 하이라이트 없이 처리해 주세요.

```txt
마우스 포인터가 Level 2 카드 영역 안에 있음
→ 해당 Level 2의 Level 3 목록을 정렬 대상으로 변경

마우스가 Level 2 밖에 있음
→ 변경 없음
```

Level 2 전체 박스를 노란색으로 칠하지 않아도 됩니다.

필요하면 아주 약한 outline 정도만 사용하고, 가능하면 placeholder 중심으로 보여주세요.

---

# 10. 프리뷰와 실제 드롭 결과 일치

가장 중요한 원칙입니다.

```txt
드래그 중 보이는 placeholder 위치 = 실제 드롭 후 위치
```

드래그 중 계산한 값을 그대로 drop에 사용해 주세요.

```txt
previewParentId
previewIndex
```

드롭 시점에 별도로 다시 계산해서 위치가 달라지면 안 됩니다.

---

# 11. 드롭 판정 영역

드롭 판정은 너무 좁게 잡지 말아 주세요.

현재처럼 카드의 가운데 좁은 영역에 닿아야만 반응하면 불편합니다.

각 레벨의 정렬 트랙 전체를 기준으로 계산해 주세요.

```txt
Level 1 → 파란 Level 1 메뉴 줄 전체
Level 2 → 현재 Level 1의 Level 2 목록 전체
Level 3 → 현재 Level 2의 Level 3 목록 전체
```

카드 사이 빈 공간에서도 가장 가까운 index를 계산해야 합니다.

---

# 12. 다중 행 처리

비주얼모드는 메뉴가 여러 행/여러 열로 배치됩니다.

따라서 첫 번째 행만 후보로 보면 안 됩니다.

```txt
모든 visible item의 DOM rect를 수집
X/Y 좌표를 함께 사용
아래쪽 행도 후보에 포함
카드 사이 공간도 가장 가까운 위치로 계산
```

---

# 13. 히스토리 연동

기존 히스토리 구조는 그대로 유지합니다.

```txt
드래그 시작 → 히스토리 변화 없음
드래그 중 preview 변경 → 히스토리 변화 없음
드롭 취소 → 히스토리 변화 없음
드롭 성공 → commitMenuChange(nextMenuState) 1회 호출
```

직접 `setMenuState()`를 호출하지 말고 기존 공통 함수를 사용해 주세요.

```js
commitMenuChange(nextMenuState);
```

# 최종 기대 동작

| 드래그 대상                        | 허용 동작                                |
| ----------------------------- | ------------------------------------ |
| Level 1                       | 파란 Level 1 메뉴 줄에서 좌우 순서 정렬만          |
| Level 2                       | 현재 Level 1의 Level 2 목록에서 순서 정렬       |
| Level 2 → 다른 Level 1          | 해당 Level 1로 전환 후 그 안에서 Level 2 순서 정렬 |
| Level 3                       | 현재 Level 2의 Level 3 목록에서 순서 정렬       |
| Level 3 → 다른 Level 2          | 해당 Level 2 아래로 이동 후 Level 3 순서 정렬    |
| Level 3 → 다른 Level 1의 Level 2 | Level 1 전환 후 해당 Level 2 아래로 이동 가능    |
| 잘못된 영역                        | 아무 반응 없음                             |
| 금지표시                          | 사용하지 않음                              |
| 노란 하이라이트                      | 사용하지 않음                              |
| 드롭 성공                         | 히스토리 +1                              |

# 핵심 요약

비주얼모드 Drag & Drop은 더 이상 복잡한 삽입/금지/하이라이트 방식으로 구현하지 말고,
아래처럼 단순한 순서정렬 방식으로 구현해 주세요.

```txt
Level 1 = 파란 Level 1 줄에서 순서 정렬만
Level 2 = Level 1 아래에서 Level 2 순서 정렬만
Level 3 = Level 2 아래에서 Level 3 순서 정렬만
```

다른 부모로 이동이 필요한 경우에도,
부모를 선택한 뒤 그 부모의 해당 레벨 목록 안에서 순서 정렬하는 방식으로만 처리해 주세요.

이상한 기능을 덧붙이지 말고,
허용된 레벨 트랙 안에서 정렬만 되도록 단순하게 구현해 주세요.

```
