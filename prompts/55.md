# 가이드 Step 5 / Step 7 / Step 9 위치 및 애니메이션 재수정 요청

## 작업 목표

현재 가이드 단계 중 Step 5, Step 7, Step 9에서 아직 표시 문제가 있습니다.

```txt
Step 5: 핸들 가이드 설명창 위치와 포인터 애니메이션 레이어가 여전히 이상함
Step 7: 메뉴 설명 확인 단계에서 설명창/툴팁이 하이라이트와 겹침
Step 9: 등록관리 중분류 예시는 좋지만, 핸들을 움직이라는 포인터 애니메이션이 사라짐
````

이번 수정은 **가이드 UI 표시 방식만 수정**하는 작업입니다.
기존 메뉴 편집 기능, 실제 드래그앤드랍, 저장, 제출, 로그 기능은 건드리지 말아 주세요.

# 1. Step 5 핸들 가이드 재수정

## 현재 문제

Step 5 `핸들을 잡고 드래그` 단계에서 아직 문제가 있습니다.

현재 증상:

```txt
설명창이 하이라이트된 아이템 위에 겹침
포인터/마우스 애니메이션이 하이라이트 네모 뒤에서 동작함
하이라이트된 목록 위에 포인터가 명확하게 보이지 않음
사용자가 실제로 핸들을 누르는 장면을 보기 어려움
```

---

## 수정 요청

Step 5에서는 설명창을 **하이라이트 영역 오른쪽**으로 보내 주세요.

기존처럼 위/아래에 두면 하이라이트 영역과 겹치거나 포인터 애니메이션 경로를 가릴 수 있습니다.

### Step 5 설명창 위치

```txt
Step 5 설명창 위치: 하이라이트된 메뉴 아이템의 오른쪽
```

화면 오른쪽 공간이 부족할 때만 예외적으로 왼쪽 또는 위쪽으로 배치해 주세요.

우선순위:

```txt
1순위: 대상 오른쪽
2순위: 대상 왼쪽
3순위: 대상 위
4순위: 대상 아래
```

---

## Step 5 포인터 애니메이션 레이어 수정

Step 5의 포인터/마우스 애니메이션은 반드시
**하이라이트 네모보다 위에 표시**되어야 합니다.

현재는 하이라이트 영역 뒤에서 움직이는 것처럼 보여서 안 보입니다.

아래 z-index 순서를 반드시 지켜 주세요.

```txt
dim overlay: 9000
하이라이트 네모: 9010
포인터/마우스 애니메이션: 9025
가이드 설명창: 9030
가이드 버튼: 9040
```

CSS 예시:

```css
.guide-highlight {
  position: fixed;
  z-index: 9010;
}

.guide-pointer,
.guide-cursor,
.guide-hand-animation {
  position: fixed;
  z-index: 9025;
  pointer-events: none;
}

.guide-popover {
  position: fixed;
  z-index: 9030;
}
```

---

## Step 5 애니메이션 위치

Step 5의 포인터 애니메이션은 반드시 **핸들 위치에서 시작**해야 합니다.

```txt
1. 포인터가 핸들 중앙으로 이동
2. 핸들을 누르는 모션
3. 위/아래로 짧게 드래그하는 모션
4. 포인터는 하이라이트 네모보다 위에 보임
5. 실제 아이템은 이동하지 않음
```

아이템 전체가 움직이는 것처럼 보이면 안 됩니다.
핸들을 누르고 움직이는 느낌만 보여주세요.

---

## Step 5 기대 결과

```txt
설명창은 오른쪽에 있어 하이라이트 영역을 가리지 않음
포인터 애니메이션은 하이라이트된 목록 위에 선명하게 보임
사용자가 "점 모양 핸들을 잡으면 되는구나"를 바로 이해함
```

# 2. Step 7 메뉴 설명 확인 단계 재수정

## 현재 문제

Step 7 `메뉴 설명 확인` 단계에서 문제가 있습니다.

현재 증상:

```txt
가이드 설명창이 하이라이트 영역과 겹침
설명창이 하이라이트 뒤로 가려져 보임
툴팁 위치가 어색하거나 제대로 보이지 않음
hover 했을 때만 툴팁이 나와야 하는데 시연 흐름이 부자연스러움
```

---

## 수정 요청

Step 7에서는 설명창이 하이라이트와 겹치지 않도록 **오른쪽으로 밀어서 배치**해 주세요.

### Step 7 설명창 위치

```txt
Step 7 설명창 위치: 하이라이트된 메뉴명 오른쪽
```

오른쪽 공간이 부족하면 위쪽 또는 왼쪽으로 이동해도 됩니다.

우선순위:

```txt
1순위: 대상 오른쪽
2순위: 대상 위
3순위: 대상 왼쪽
4순위: 대상 아래
```

---

## Step 7 툴팁 표시 조건

툴팁은 항상 떠 있으면 안 되고,
**포인터 애니메이션이 메뉴명 위로 hover했을 때만 표시**되어야 합니다.

원하는 흐름:

```txt
1. 처음에는 툴팁 없음
2. 포인터 아이콘이 메뉴명 쪽으로 이동
3. 포인터가 메뉴명 위에 도착
4. 이때 메뉴 설명 툴팁 표시
5. 조건 충족 후 완료 표시
```

즉:

```txt
포인터 hover 전: desc tooltip 숨김
포인터 hover 중: desc tooltip 표시
포인터 hover 종료 또는 단계 종료: desc tooltip 숨김
```

---

## Step 7 가이드 설명창은 꺼지면 안 됨

Step 7에서 가이드 설명창 자체가 사라지면 안 됩니다.

```txt
가이드 설명창 유지
desc tooltip은 hover 시에만 표시
둘이 겹치면 설명창을 오른쪽/위쪽으로 이동
```

---

## Step 7 추천 구현

가이드 전용 hover 상태를 따로 두는 것을 추천합니다.

```js
const isGuideTooltipStep =
  guideActive && currentGuideStep?.id === "menu-tooltip";

const shouldShowGuideDescTooltip =
  isGuideTooltipStep && guidePointerReachedTarget;
```

기존 tooltip 표시 조건에 추가:

```jsx
<DescTooltip
  visible={isHovered || shouldShowGuideDescTooltip}
  desc={item.desc}
/>
```

단, Step 7이 끝나면 반드시 `shouldShowGuideDescTooltip`은 false가 되어야 합니다.

---

## Step 7 기대 결과

```txt
처음에는 메뉴 설명 툴팁이 안 보임
포인터가 메뉴명으로 이동해서 hover하는 순간 툴팁 표시
가이드 설명창은 오른쪽으로 배치되어 하이라이트와 겹치지 않음
설명창이 하이라이트 뒤로 가려지지 않음
```

# 3. Step 9 비주얼모드 포인터 애니메이션 복구

## 현재 상태

Step 9에서 비주얼모드 예시 대상을 `등록관리` 중분류 메뉴로 바꾼 것은 좋습니다.

현재 개선된 점:

```txt
등록관리 중분류 메뉴가 예시 대상으로 잡혀서 가시성이 좋아짐
레벨1 파란 배경보다 훨씬 잘 보임
```

하지만 새로운 문제가 있습니다.

```txt
핸들을 움직이라는 마우스/포인터 애니메이션이 사라짐
```

---

## 수정 요청

Step 9에서도 Step 5처럼
**등록관리 메뉴의 핸들을 잡고 움직이는 포인터 애니메이션**을 다시 표시해 주세요.

---

## Step 9 대상

```txt
대상 메뉴: 등록관리
레벨: L2 / 중분류
대상 위치: 등록관리 메뉴의 점 모양 핸들
```

---

## Step 9 애니메이션 방식

```txt
1. 등록관리 메뉴 전체 또는 핸들을 하이라이트
2. 포인터가 등록관리의 점 모양 핸들로 이동
3. 핸들을 누르는 모션
4. 좌우 또는 살짝 이동하는 드래그 모션
5. 실제 menuState는 변경하지 않음
```

비주얼모드에서는 실제 배치가 가로/그리드 구조이므로
애니메이션 방향은 상황에 맞게 좌우 또는 짧은 대각선 이동으로 보여주세요.
