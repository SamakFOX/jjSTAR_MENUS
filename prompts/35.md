# 비주얼모드 Drag & Drop 개선 요청: @dnd-kit 기반 레벨별 이동 구조 적용

## 작업 목표

현재 비주얼모드의 드래그앤드랍 사용성이 불안정합니다.

특히 다음 문제가 있습니다.

- Level 2를 이동하려고 할 때 금지 표시가 자주 뜸
- 금지 표시가 실제 마우스 위치가 아닌 엉뚱한 곳에 표시됨
- 드래그 중 보이는 위치와 실제 드롭 결과가 다름
- Level 2를 다른 Level 1 아래로 이동하거나 같은 Level 1 안에서 정렬할 때 조작감이 불안정함
- 사용자가 정확히 어디에 드롭되는지 예측하기 어려움

따라서 비주얼모드의 Drag & Drop 구조를 `@dnd-kit` 기반으로 개선하고 싶습니다.

---

## 가장 중요한 방향

현재처럼 모든 위치를 드롭 후보로 보고, 잘못된 위치에 금지 표시를 띄우는 방식은 피해주세요.

대신 **드래그를 시작한 메뉴의 Level에 따라 가능한 이동 영역만 활성화**하는 방식으로 변경해 주세요.

핵심 원칙:

```txt
Level 1을 잡으면 Level 1 정렬만 가능
Level 2를 잡으면 Level 2 정렬/다른 Level 1로 이동만 가능
Level 3을 잡으면 Level 3 정렬/다른 Level 2로 이동만 가능
````

잘못된 영역은 금지 표시를 띄우는 것이 아니라,
처음부터 드롭 후보에서 제외해 주세요.

# 1. 레벨별 드래그 모드

드래그 시작 시점에 드래그 대상의 level을 확인하고, 드래그 모드를 고정해 주세요.

```js
function getVisualDragMode(dragItem) {
  if (dragItem.level === 1) return "LEVEL_1_SORT";
  if (dragItem.level === 2) return "LEVEL_2_SORT_OR_MOVE";
  if (dragItem.level === 3) return "LEVEL_3_SORT_OR_MOVE";
  return null;
}
```

드래그 중에는 이 모드에 맞는 드롭존만 활성화해야 합니다.

```txt
LEVEL_1_SORT
→ Level 1 트랙만 활성화

LEVEL_2_SORT_OR_MOVE
→ 현재 활성 Level 1의 Level 2 트랙만 활성화
→ 다른 Level 1 레이블 hover 시 해당 Level 1로 전환

LEVEL_3_SORT_OR_MOVE
→ 현재 활성 Level 2의 Level 3 트랙만 활성화
→ 다른 Level 2 hover 시 해당 Level 2로 전환
```

# 2. Level 1 드래그 동작

## 허용 동작

Level 1 메뉴를 잡았을 때는 Level 1 메뉴끼리만 정렬되게 해주세요.

예시:

```txt
[star 학생지원] [학사정보] [대학생활] [진로개발]
```

Level 1을 잡으면 위 Level 1 레이블/탭 영역에서만 이동할 수 있어야 합니다.

## 비활성 영역

Level 1 드래그 중에는 아래 영역을 드롭 후보로 보지 않습니다.

```txt
Level 2 영역
Level 3 영역
본문 빈 공간
하위 메뉴 카드
```

## 기대 동작

```txt
Level 1 드래그 시작
→ Level 1 정렬 모드 진입
→ Level 1 트랙만 활성화
→ 다른 영역은 드롭 후보에서 제외
→ 금지표시 불필요
```

---

# 3. Level 2 드래그 동작

## 허용 동작

Level 2 메뉴를 잡았을 때는 Level 2 정렬/이동 모드로 동작해야 합니다.

허용되는 동작:

```txt
1. 현재 Level 1 안에서 Level 2 순서 정렬
2. 다른 Level 1 레이블 위로 이동하면 해당 Level 1로 전환
3. 전환된 Level 1 내부에서 Level 2 순서 정렬
```

예시:

```txt
취업지원 > 취업정보
```

Level 2 메뉴를 잡고 상단의 다른 Level 1 레이블인

```txt
정보공유
```

위로 이동하면, 비주얼모드가 정보공유 Level 1로 전환되어야 합니다.

그 후 정보공유 내부의 Level 2 목록 사이에서 원하는 위치에 드롭할 수 있어야 합니다.

## 중요

Level 2를 잡았을 때는 Level 3 영역을 드롭 대상으로 계산하지 마세요.

```txt
Level 2 드래그 중 Level 3 영역에 올라감
→ 금지 표시를 띄우지 않음
→ Level 3은 드롭 후보가 아님
→ 현재 Level 2 정렬 후보만 유지
```

## 기대 동작

```txt
Level 2 드래그 시작
→ Level 2 정렬/이동 모드 진입
→ 현재 Level 1의 Level 2 트랙 활성화
→ 다른 Level 1 레이블 hover 시 activeLevel1 변경
→ 변경된 Level 1의 Level 2 트랙 활성화
→ 원하는 위치에 placeholder 표시
→ 드롭 시 해당 위치로 이동
```

---

# 4. Level 3 드래그 동작

## 허용 동작

Level 3 메뉴를 잡았을 때는 Level 3 정렬/이동 모드로 동작해야 합니다.

허용되는 동작:

```txt
1. 같은 Level 2 내부에서 Level 3 순서 정렬
2. 다른 Level 2 아래로 이동
3. 이동한 Level 2 내부에서 Level 3 순서 정렬
```

예시:

```txt
커뮤니티 > 설문조사
```

Level 3 메뉴를 잡고 다른 Level 2인

```txt
졸업관리
```

위로 이동하면, 졸업관리 하위의 Level 3 목록이 드롭 후보가 되어야 합니다.

## 비활성 영역

Level 3 드래그 중에는 아래 영역을 드롭 후보로 보지 않습니다.

```txt
Level 1 바로 아래
Level 1 빈 공간
다른 Level 3의 하위
본문 빈 공간
```

## 기대 동작

```txt
Level 3 드래그 시작
→ Level 3 정렬/이동 모드 진입
→ 현재 Level 2의 Level 3 트랙 활성화
→ 다른 Level 2 hover 시 해당 Level 2의 Level 3 트랙 활성화
→ 원하는 위치에 placeholder 표시
→ 드롭 시 해당 Level 2 아래로 이동
```

# 5. 금지 표시 제거 또는 최소화

현재 금지 표시가 마우스 위치와 다른 곳에 표시되어 사용성을 해치고 있습니다.

따라서 가능하면 비주얼모드에서 금지 표시는 제거해 주세요.

대신 다음 방식으로 처리해 주세요.

```txt
허용되는 드롭존만 활성화
허용되지 않는 영역은 드롭 후보에서 제외
드롭 후보가 아닌 영역에서는 아무 반응 없음
```

정말 필요한 경우에만 토스트로 안내할 수 있습니다.

예시:

```txt
이 위치로는 이동할 수 없습니다.
```

하지만 일반적인 드래그 중에는 금지 아이콘이 계속 뜨지 않는 것이 좋습니다.

# 6. 드롭 프리뷰

드래그 중에는 실제 들어갈 위치가 명확하게 보여야 합니다.

## 기대 UI

```txt
아이템이 들어갈 위치에 placeholder 표시
기존 아이템은 placeholder 기준으로 밀림
드롭 후 실제 위치는 프리뷰와 동일
```

가장 중요한 원칙:

```txt
프리뷰 위치 = 실제 드롭 위치
```

드래그 중 계산한 `previewParentId`, `previewIndex`와
드롭 시 적용하는 `finalParentId`, `finalIndex`가 같은 기준이어야 합니다.

# 7. 드롭 위치 계산 방식

드롭 위치는 마우스 좌표와 대상 카드의 중앙값을 기준으로 계산해 주세요.

## 가로 정렬인 경우

```txt
마우스 X 좌표 < 대상 카드 중앙 X
→ 대상 카드 앞에 삽입

마우스 X 좌표 >= 대상 카드 중앙 X
→ 대상 카드 뒤에 삽입
```

## 세로 정렬인 경우

```txt
마우스 Y 좌표 < 대상 카드 중앙 Y
→ 대상 카드 앞에 삽입

마우스 Y 좌표 >= 대상 카드 중앙 Y
→ 대상 카드 뒤에 삽입
```

비주얼모드의 Level 1/Level 2 카드가 가로 배치라면 X 좌표 기반 계산을 우선 사용해 주세요.

줄바꿈되는 카드 그리드 구조라면 X/Y 좌표를 함께 고려해 주세요.

# 8. @dnd-kit 적용 구조 제안

비주얼모드 컴포넌트는 Client Component여야 합니다.

```js
"use client";
```

기본 구조 예시:

```jsx
import {
  DndContext,
  DragOverlay,
  PointerSensor,
  useSensor,
  useSensors,
} from "@dnd-kit/core";

import {
  SortableContext,
  horizontalListSortingStrategy,
  verticalListSortingStrategy,
} from "@dnd-kit/sortable";

function VisualEditor() {
  const sensors = useSensors(
    useSensor(PointerSensor, {
      activationConstraint: {
        distance: 6,
      },
    })
  );

  return (
    <DndContext
      sensors={sensors}
      onDragStart={handleVisualDragStart}
      onDragOver={handleVisualDragOver}
      onDragEnd={handleVisualDragEnd}
      onDragCancel={handleVisualDragCancel}
    >
      {/* Level 1 SortableContext */}
      {/* Level 2 SortableContext */}
      {/* Level 3 SortableContext */}

      <DragOverlay>
        {/* 드래그 중인 카드 미리보기 */}
      </DragOverlay>
    </DndContext>
  );
}
```

# 9. 레벨별 SortableContext 분리

레벨별로 SortableContext를 분리해 주세요.

```txt
Level 1 SortableContext
Level 2 SortableContext
Level 3 SortableContext
```

예시:

```jsx
<SortableContext
  items={level1Items.map((item) => item.id)}
  strategy={horizontalListSortingStrategy}
>
  {level1Items.map((item) => (
    <SortableVisualMenuItem key={item.id} item={item} />
  ))}
</SortableContext>
```

Level 2는 현재 활성 Level 1 기준으로만 렌더링합니다.

```jsx
<SortableContext
  items={activeLevel2Items.map((item) => item.id)}
  strategy={horizontalListSortingStrategy}
>
  {activeLevel2Items.map((item) => (
    <SortableVisualMenuItem key={item.id} item={item} />
  ))}
</SortableContext>
```

Level 3은 현재 활성 Level 2 기준으로만 렌더링합니다.

```jsx
<SortableContext
  items={activeLevel3Items.map((item) => item.id)}
  strategy={horizontalListSortingStrategy}
>
  {activeLevel3Items.map((item) => (
    <SortableVisualMenuItem key={item.id} item={item} />
  ))}
</SortableContext>
```

실제 화면 구조에 따라 `verticalListSortingStrategy` 또는 커스텀 strategy를 사용해도 됩니다.

# 10. 드래그 시작 처리

드래그 시작 시 다음 정보를 저장해 주세요.

```js
function handleVisualDragStart(event) {
  const activeId = event.active.id;
  const dragItem = findMenuItemById(activeId);

  if (!dragItem) return;

  const dragMode = getVisualDragMode(dragItem);

  setVisualDragState({
    activeId,
    dragItem,
    dragMode,
    sourceParentId: dragItem.parentId,
    sourceIndex: getIndexInParent(dragItem),
    previewParentId: dragItem.parentId,
    previewIndex: getIndexInParent(dragItem),
  });
}
```

# 11. 드래그 오버 처리

드래그 중에는 dragMode에 맞는 대상만 처리해 주세요.

```js
function handleVisualDragOver(event) {
  const { active, over } = event;

  if (!over || !visualDragState) return;

  const { dragItem, dragMode } = visualDragState;

  if (dragMode === "LEVEL_1_SORT") {
    updateLevel1Preview(event);
    return;
  }

  if (dragMode === "LEVEL_2_SORT_OR_MOVE") {
    updateLevel2Preview(event);
    return;
  }

  if (dragMode === "LEVEL_3_SORT_OR_MOVE") {
    updateLevel3Preview(event);
    return;
  }
}
```

잘못된 레벨의 over 대상은 무시합니다.

```js
if (!isAllowedVisualOverTarget(dragMode, over)) {
  return;
}
```

# 12. Level 2의 Level 1 hover 전환

Level 2를 드래그 중일 때 다른 Level 1 레이블 위에 올라가면, 해당 Level 1로 전환되어야 합니다.

```js
function updateLevel2Preview(event) {
  const overMeta = getDroppableMeta(event.over);

  if (overMeta.type === "LEVEL_1_LABEL") {
    setActiveLevel1Id(overMeta.level1Id);

    setVisualDragState((prev) => ({
      ...prev,
      previewParentId: overMeta.level1Id,
      previewIndex: getEndIndexOfLevel2(overMeta.level1Id),
    }));

    return;
  }

  if (overMeta.type === "LEVEL_2_ITEM") {
    const targetParentId = overMeta.parentId;
    const targetIndex = calculateInsertIndex(event);

    setVisualDragState((prev) => ({
      ...prev,
      previewParentId: targetParentId,
      previewIndex: targetIndex,
    }));
  }
}
```

중요:

```txt
Level 2 드래그 중에는 Level 3 대상은 무시
Level 1 레이블은 부모 전환용으로만 사용
실제 삽입은 Level 2 트랙에서만 처리
```

# 13. Level 3의 Level 2 hover 전환

Level 3을 드래그 중일 때 다른 Level 2 위에 올라가면 해당 Level 2 하위로 이동할 준비를 해야 합니다.

```js
function updateLevel3Preview(event) {
  const overMeta = getDroppableMeta(event.over);

  if (overMeta.type === "LEVEL_2_ITEM") {
    setActiveLevel2Id(overMeta.level2Id);

    setVisualDragState((prev) => ({
      ...prev,
      previewParentId: overMeta.level2Id,
      previewIndex: getEndIndexOfLevel3(overMeta.level2Id),
    }));

    return;
  }

  if (overMeta.type === "LEVEL_3_ITEM") {
    const targetParentId = overMeta.parentId;
    const targetIndex = calculateInsertIndex(event);

    setVisualDragState((prev) => ({
      ...prev,
      previewParentId: targetParentId,
      previewIndex: targetIndex,
    }));
  }
}
```

중요:

```txt
Level 3 드래그 중에는 Level 1 바로 아래로 이동하지 않음
Level 3 드래그 중에는 Level 3 하위로 들어가지 않음
Level 2는 부모 전환 대상으로 사용
```

# 14. 드롭 처리

드롭 시에는 드래그 중 계산한 preview 정보를 그대로 사용해 주세요.

```js
function handleVisualDragEnd(event) {
  if (!visualDragState) return;

  const {
    dragItem,
    previewParentId,
    previewIndex,
    sourceParentId,
    sourceIndex,
  } = visualDragState;

  if (
    sourceParentId === previewParentId &&
    sourceIndex === previewIndex
  ) {
    clearVisualDragState();
    return;
  }

  const nextMenuState = moveMenuItem({
    menuState,
    itemId: dragItem.id,
    targetParentId: previewParentId,
    targetIndex: previewIndex,
  });

  if (!nextMenuState) {
    clearVisualDragState();
    return;
  }

  commitMenuChange(nextMenuState);
  clearVisualDragState();
}
```

중요:

```txt
드롭 성공 시 commitMenuChange 1회만 호출
드롭 실패 시 히스토리 변화 없음
드래그 취소 시 히스토리 변화 없음
프리뷰 변경 중에는 히스토리 변화 없음
```

# 15. 같은 부모 내 index 보정

같은 부모 안에서 순서 이동할 때는 기존 위치에서 제거 후 삽입되므로 index 보정이 필요합니다.

예시:

```txt
기존 순서: A B C D
B를 D 뒤로 이동
```

처리:

```txt
1. B 제거 → A C D
2. D 뒤에 B 삽입 → A C D B
```

주의:

```txt

# 17. 히스토리 연동

비주얼모드 드래그앤드랍은 기존 공통 히스토리와 연결되어야 합니다.

성공한 드롭만 히스토리에 저장합니다.

```txt
드래그 시작 → 히스토리 변화 없음
드래그 중 → 히스토리 변화 없음
드롭 취소 → 히스토리 변화 없음
잘못된 위치 → 히스토리 변화 없음
같은 위치 드롭 → 히스토리 변화 없음
드롭 성공 → commitMenuChange(nextMenuState) 1회 호출
```

직접 `setMenuState()`를 호출하지 말고 기존 공통 함수를 사용해 주세요.

```js
commitMenuChange(nextMenuState);
```

비주얼모드 외 다른 기능은 변경하지 말아 주세요.

```
