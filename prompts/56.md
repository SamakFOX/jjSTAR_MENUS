# 가이드 위치/진행 방식 재수정 요청

## 현재 문제

가이드 수정 요청을 했지만 실제 화면에서는 여전히 같은 문제가 반복되고 있습니다.

현재 문제는 아래와 같습니다.

```txt
1. Step 1에서 [건너뛰기] 버튼이 있어 사용자가 실제 가이드를 보지 않고 건너뛰기 쉬움
2. Step 5 핸들 설명창 위치가 계속 하이라이트 영역과 겹침
3. Step 7 메뉴 설명 확인 단계에서 설명창이 하이라이트 영역과 겹침
4. Step 9 비주얼모드 단계에서 핸들 조작 마우스 애니메이션이 없음
5. 전체적으로 설명창이 강조 영역 아래/위에 애매하게 겹쳐서 정작 봐야 할 요소를 가림
````

이번에는 가이드 설명창 위치를 자동 계산에 맡기지 말고,
**각 Step별로 강제 배치 규칙을 명확히 적용**해 주세요.

---

# 가장 중요한 조건

기존 기능은 절대 건드리지 말아 주세요.

```txt
로그인/인증
메뉴 편집 기능
리스트모드 드래그앤드랍
비주얼모드 드래그앤드랍
임시저장/자동저장
미리보기
최종 제출 저장
변경 로그 저장
툴팁(desc) 데이터 구조
undo/redo
초기화
관리자 화면
Supabase 저장 구조
```

이번 작업은 **가이드 UI 위치, 진행 방식, 포인터 애니메이션만 수정**하는 작업입니다.

# 1. Step 1에서 건너뛰기 제거

## 현재 문제

Step 1에서 `다음` 버튼이 비활성화되어 있고 `건너뛰기` 버튼이 보입니다.

이러면 사용자는 자연스럽게 `건너뛰기`를 누르게 됩니다.

## 수정 요청

Step 1에서는 `건너뛰기` 버튼을 없애 주세요.

그리고 Step 1에서 `다음` 버튼을 누르면
자동으로 `메뉴 편집하기` 버튼을 누른 것과 동일하게 처리해서 편집화면으로 전환되게 해 주세요.

즉, Step 1은 아래처럼 동작해야 합니다.

```txt
Step 1: 메뉴 편집 시작 안내
사용자 클릭: [다음]
동작:
1. 편집모드로 전환
2. Step 2로 이동
3. 편집화면 기본 UI 가이드 표시
```

## Step 1 버튼 구성

기존:

```txt
[건너뛰기] [다음 비활성화]
```

수정 후:

```txt
[다음]
```

또는 필요하면:

```txt
[닫기] [다음]
```

단, `건너뛰기`라는 문구는 제거해 주세요.

# 2. Step 5 핸들 가이드 위치 강제 수정

## 현재 문제

Step 5 `핸들을 잡고 드래그` 단계에서 설명창이 계속 하이라이트된 메뉴 영역과 겹칩니다.

또한 포인터 애니메이션이 하이라이트 뒤에 깔리거나 보이지 않습니다.

## 수정 요청

Step 5에서는 자동 위치 계산을 사용하지 말고,
**설명창을 무조건 하이라이트 대상 오른쪽으로 배치**해 주세요.

```txt
Step 5 설명창 위치: target 오른쪽 고정
```

오른쪽 공간이 부족한 경우에만 왼쪽으로 이동합니다.

```txt
Step 5 위치 우선순위:
1. 오른쪽
2. 왼쪽
3. 위
4. 아래
```

## 금지

Step 5에서는 아래 위치 배치를 하지 마세요.

```txt
하이라이트 영역 위에 겹치기 금지
하이라이트 영역 아래에 겹치기 금지
하이라이트된 아이템 내부에 설명창 겹치기 금지
포인터 애니메이션 경로 위에 설명창 배치 금지
```

## 포인터 애니메이션 필수

Step 5에는 반드시 핸들 조작 포인터 애니메이션이 보여야 합니다.

```txt
1. 포인터가 핸들 중앙으로 이동
2. 핸들을 누르는 모션
3. 위/아래로 짧게 움직이는 모션
4. 실제 데이터는 변경하지 않음
```

## z-index 강제

```txt
하이라이트: 9010
포인터 애니메이션: 9025
설명창: 9030
가이드 버튼: 9040
```

포인터가 하이라이트 뒤에 깔리면 안 됩니다.

# 3. Step 7 메뉴 설명 확인 위치 강제 수정

## 현재 문제

Step 7 `메뉴 설명 확인` 단계에서 설명창이 하이라이트된 메뉴 영역과 계속 겹칩니다.

그리고 hover tooltip을 보여줘야 하는 단계인데, 설명창과 하이라이트가 겹쳐서 사용자가 어디를 봐야 하는지 알기 어렵습니다.

## 수정 요청

Step 7도 자동 위치 계산을 사용하지 말고,
**설명창을 무조건 하이라이트 대상 오른쪽으로 배치**해 주세요.

```txt
Step 7 설명창 위치: target 오른쪽 고정
```

오른쪽 공간이 부족한 경우에는 위쪽으로 이동합니다.

```txt
Step 7 위치 우선순위:
1. 오른쪽
2. 위
3. 왼쪽
4. 아래
```

## 금지

```txt
설명창이 하이라이트 영역과 겹치면 안 됨
설명창이 메뉴명 위를 덮으면 안 됨
설명창이 tooltip 위치를 가리면 안 됨
```

## hover tooltip 동작

Step 7에서는 tooltip이 항상 떠 있으면 안 됩니다.

아래 흐름으로 처리해 주세요.

```txt
1. 처음에는 tooltip 없음
2. 포인터 애니메이션이 메뉴명 위로 이동
3. 포인터가 메뉴명 위에 도착했을 때만 tooltip 표시
4. 포인터가 떠나거나 Step이 끝나면 tooltip 숨김
```

## 가이드 설명창은 유지

Step 7에서 가이드 설명창은 꺼지면 안 됩니다.

```txt
가이드 설명창 유지
hover tooltip만 조건부 표시
설명창과 tooltip이 겹치면 설명창을 오른쪽 또는 위로 이동
```

# 4. Step 9 비주얼모드 핸들 애니메이션 복구

## 현재 문제

Step 9에서 `등록관리` 중분류 메뉴를 대상으로 잡은 것은 좋습니다.

하지만 `등록관리` 메뉴의 핸들을 움직이는 마우스 애니메이션이 사라졌습니다.

## 수정 요청

Step 9에는 반드시 포인터/마우스 애니메이션을 다시 넣어 주세요.

대상은 아래와 같습니다.

```txt
대상 메뉴: 등록관리
대상 레벨: L2 / 중분류
대상 위치: 등록관리 메뉴의 점 모양 핸들
```

## Step 9 애니메이션

```txt
1. 등록관리 메뉴 또는 핸들 하이라이트
2. 포인터가 등록관리 핸들 중앙으로 이동
3. 포인터가 핸들을 누르는 모션
4. 좌우 또는 대각선으로 짧게 드래그하는 모션
5. 실제 menuState는 변경하지 않음
```

## Step 9 설명창 위치

Step 9 설명창은 등록관리 메뉴와 겹치면 안 됩니다.

```txt
Step 9 설명창 위치 우선순위:
1. 오른쪽
2. 왼쪽
3. 위
4. 아래
```

비주얼모드에서는 설명창이 메뉴 목록을 덮지 않도록,
등록관리 카드와 분리된 위치에 배치해 주세요.

# 5. 전체 가이드 위치 계산 규칙 수정

현재 가이드 설명창이 전반적으로 강조 영역과 겹칩니다.

따라서 모든 단계에서 아래 원칙을 지켜 주세요.

## 공통 원칙

```txt
설명창은 강조되는 target과 겹치면 안 됨
설명창은 포인터 애니메이션 경로와 겹치면 안 됨
설명창은 클릭해야 하는 버튼을 덮으면 안 됨
설명창은 tooltip을 가리면 안 됨
```

## 자동 위치 계산 시 필수 체크

설명창 위치를 계산할 때 아래 충돌 검사를 반드시 해 주세요.

```js
function isOverlapping(rectA, rectB) {
  return !(
    rectA.right < rectB.left ||
    rectA.left > rectB.right ||
    rectA.bottom < rectB.top ||
    rectA.top > rectB.bottom
  );
}
```

위치 계산 후:

```txt
popoverRect와 targetRect가 겹치면 다른 위치로 재계산
popoverRect와 pointerPathRect가 겹치면 다른 위치로 재계산
popoverRect와 tooltipRect가 겹치면 다른 위치로 재계산
```

# 7. 포인터 애니메이션 공통 규칙

Step 5와 Step 9에는 포인터 애니메이션이 반드시 있어야 합니다.

## 필수 조건

```txt
포인터는 target 핸들의 중앙 좌표에서 시작
포인터는 하이라이트보다 위에 표시
포인터는 실제 클릭 이벤트를 발생시키지 않음
포인터는 실제 drag 이벤트를 발생시키지 않음
포인터는 CSS animation 또는 requestAnimationFrame으로만 표현
```

## 좌표 기준

```js
const handleRect = handleElement.getBoundingClientRect();

const pointerStart = {
  x: handleRect.left + handleRect.width / 2,
  y: handleRect.top + handleRect.height / 2,
};
```

# 8. Step 1 완료 방식 변경

Step 1에서 사용자가 다음을 누르면, 실제 버튼 클릭을 기다리지 않고 편집모드로 전환해 주세요.

```js
function handleGuideNext() {
  if (currentGuideStep.id === "start-edit") {
    enterEditMode();
    goToGuideStep("edit-basic-toolbar");
    return;
  }

  goToNextGuideStep();
}
```

이때 실제 편집모드 진입 로직은 기존 `메뉴 편집하기` 버튼에서 사용하는 함수를 재사용해 주세요.

```txt
새로운 편집 진입 로직 만들지 말고 기존 함수 재사용
```

# 핵심 요약

이번 수정의 핵심은 아래입니다.

```txt
1. Step 1에서 건너뛰기 제거
2. Step 1의 다음 버튼으로 바로 편집화면 진입
3. Step 5 설명창은 오른쪽 고정
4. Step 5 포인터 애니메이션은 하이라이트 위에 표시
5. Step 7 설명창은 오른쪽 고정
6. Step 7 tooltip은 포인터 hover 시에만 표시
7. Step 9 등록관리 핸들 포인터 애니메이션 복구
8. 모든 설명창은 강조되는 target과 겹치지 않게 배치
```
