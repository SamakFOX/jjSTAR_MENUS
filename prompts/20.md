# Drag Preview는 before인데 실제 Drop은 after로 들어가는 문제 수정 요청

## 현재 문제

드래그 중 preview는 특정 카드의 왼쪽 또는 앞쪽에 들어갈 것처럼 표시되지만,  
실제로 마우스를 놓으면 해당 카드의 오른쪽 또는 뒤쪽에 삽입됩니다.

예시:

- `학생회 활동`을 `대학생활 상담` 앞쪽 위치로 드래그
- preview는 `대학생활 상담` 앞에 들어갈 것처럼 표시됨
- 실제 drop 후에는 `대학생활 상담` 뒤, 즉 오른쪽 위치로 들어감

이 문제는 잠깐 기다려도 동일하게 발생합니다.

따라서 단순한 state 업데이트 지연 문제가 아니라,  
**preview position과 실제 apply position이 서로 다르게 처리되는 문제**로 보입니다.

---

# 핵심 원인 추정

현재 문제는 아래 중 하나일 가능성이 큽니다.

```txt
1. preview에서는 position = 'before'로 계산됨
2. 실제 drop 적용에서는 position = 'after'로 처리됨
3. preview와 applyDrop이 서로 다른 before/after 판정 로직을 사용함
4. Visual grid에서 왼쪽/오른쪽 판정이 반대로 들어감
5. insertIndex 계산 시 before/after가 뒤집힘
6. drop target은 맞지만 position 값이 적용 중에 사라지거나 기본값 after로 바뀜
````

즉, targetItem 자체는 맞게 잡히는데
**targetItem의 앞에 넣을지 뒤에 넣을지 판정이 어긋나는 문제**입니다.

---

# 중요 전제

List Mode와 Visual Mode 모두 점검해주세요.

다만 기존 레벨 제한 규칙은 유지해야 합니다.

```txt
Level 1은 하위로 들어갈 수 없음
Level 2는 Level 1 아래에만 위치
Level 3은 Level 2 아래에만 위치
Level 4 생성 금지
```

---

# 핵심 수정 방향

preview에서 계산된 `position` 값을 실제 drop에서도 그대로 사용해야 합니다.

```txt
preview position이 before면 실제 drop도 before
preview position이 after면 실제 drop도 after
```

절대 drop 시점에 position을 다시 임의로 바꾸거나
기본값으로 after를 넣으면 안 됩니다.

# Drop Intent에 position을 반드시 포함

```js
const dropIntent = {
  dragId,
  targetId,
  dropMode,
  position, // 'before' | 'after' | 'inside'
  newParentId,
};
```

이 `position` 값은 아래 세 군데에서 동일해야 합니다.

```txt
1. preview 표시
2. drop 가능 여부 검증
3. 실제 insertIndex 계산
```

# before / after 판정 로직

Visual Mode처럼 가로 카드 배치가 있는 경우에는
Y축이 아니라 X축 기준으로 before/after를 판정해야 합니다.

```js
function getHorizontalDropPosition(pointerX, targetRect) {
  const middleX = targetRect.left + targetRect.width / 2;

  return pointerX < middleX ? 'before' : 'after';
}
```

세로 리스트에서는 Y축 기준을 사용합니다.

```js
function getVerticalDropPosition(pointerY, targetRect) {
  const middleY = targetRect.top + targetRect.height / 2;

  return pointerY < middleY ? 'before' : 'after';
}
```

Visual Mode의 Level 2 카드처럼 가로 그리드로 배치된 경우에는
반드시 X축 기준을 우선 사용해주세요.

# 실제 insertIndex 계산

`position`이 before일 때는 targetIndex 그대로 삽입해야 합니다.

```js
const insertIndex =
  intent.position === 'after'
    ? targetIndex + 1
    : targetIndex;
```

즉:

```txt
before = targetIndex
after = targetIndex + 1
```

이 규칙이 반대로 적용되어 있으면 현재처럼 오른쪽으로 들어갑니다.

---

# 반드시 확인할 부분

아래와 같은 코드가 있으면 문제가 생길 수 있습니다.

```js
// 잘못된 예시
const insertIndex = targetIndex + 1;
```

또는:

```js
// 잘못된 예시
const insertIndex =
  intent.position === 'before'
    ? targetIndex + 1
    : targetIndex;
```

이 경우 before와 after가 뒤집혀서,
preview는 앞에 보이는데 실제로는 뒤에 들어갑니다.

# movingGroup 제거 후 targetIndex 재계산

기존 위치에서 movingGroup을 제거한 뒤 targetIndex를 계산해야 합니다.

```js
function applyDropByIntent(intent, nodes) {
  const movingGroup = getMovingGroup(intent.dragId, nodes);
  const movingIds = new Set(movingGroup.map((node) => node.id));

  const remainingNodes = nodes.filter((node) => !movingIds.has(node.id));

  const targetIndex = remainingNodes.findIndex(
    (node) => node.id === intent.targetId
  );

  if (targetIndex === -1) return nodes;

  const insertIndex =
    intent.position === 'after'
      ? targetIndex + 1
      : targetIndex;

  return [
    ...remainingNodes.slice(0, insertIndex),
    ...movingGroup,
    ...remainingNodes.slice(insertIndex),
  ];
}
```

# Visual Mode 그리드 순서 기준 통일

Visual Mode는 화면상 카드가 그리드로 보이기 때문에
DOM 순서, 데이터 배열 순서, 화면상 위치가 다를 수 있습니다.

따라서 아래 중 하나로 통일해야 합니다.

## 방법 A: 화면상 순서 기준

preview도 화면상 순서 기준,
drop도 화면상 순서 기준으로 처리합니다.

```js
const visualOrderedNodes = getNodesSortedByVisualPosition();
```

## 방법 B: 데이터 배열 순서 기준

렌더링도 데이터 배열 순서대로 하고,
preview도 데이터 배열 순서대로 표시합니다.

중요한 것은 둘 중 하나로 통일하는 것입니다.

```txt
preview 기준과 실제 insert 기준이 다르면 안 됨
```

# 디버깅 로그 추가 요청

preview 시점과 drop 적용 시점에 아래 값을 비교해주세요.

```js
console.log('Preview intent:', {
  dragTitle: dragItem.title,
  targetTitle: targetItem.title,
  position: intent.position,
  targetId: intent.targetId,
  newParentId: intent.newParentId,
});
```

```js
console.log('Apply intent:', {
  dragTitle: dragItem.title,
  targetTitle: targetItem.title,
  position: intent.position,
  targetId: intent.targetId,
  newParentId: intent.newParentId,
});
```

그리고 insertIndex도 찍어주세요.

```js
console.log('Insert index:', {
  position: intent.position,
  targetIndex,
  insertIndex,
});
```

정상이라면:

```txt
position: before
targetIndex: 5
insertIndex: 5
```

이어야 합니다.

문제가 있다면 아마 이렇게 나올 가능성이 큽니다.

```txt
position: before
targetIndex: 5
insertIndex: 6
```

이 경우 before인데 after처럼 들어가는 것입니다.

# 완료 기준

| 테스트                                 | 기대 결과                    |
| ----------------------------------- | ------------------------ |
| 카드 왼쪽/before preview 위치에 drop       | 실제로도 target 앞에 삽입        |
| 카드 오른쪽/after preview 위치에 drop       | 실제로도 target 뒤에 삽입        |
| 기다린 뒤 drop                          | preview와 실제 위치 일치        |
| 바로 drop                             | preview와 실제 위치 일치        |
| `학생회 활동`을 `대학생활 상담` 앞 preview에 drop | 실제로도 앞에 삽입               |
| `학생회 활동`을 `대학생활 상담` 뒤 preview에 drop | 실제로도 뒤에 삽입               |
| Level 2 이동                          | 부모 Level 1 유지 또는 올바르게 변경 |
| Level 3 이동                          | 부모 Level 2 유지 또는 올바르게 변경 |
| Level 4 생성                          | 절대 없음                    |

# 최종 요약

이번 증상은 state 지연보다는
**preview의 before/after 위치와 실제 insertIndex 계산이 반대로 적용되는 문제**로 보입니다.

핵심은 아래입니다.

```txt
preview에서 before로 보이면 실제 drop도 before
preview에서 after로 보이면 실제 drop도 after

특히 insertIndex 계산은 반드시 아래 규칙을 따라야 합니다.

```txt
before = targetIndex
after = targetIndex + 1
```

그리고 Visual Mode의 가로 카드 배치에서는
before/after 판정을 Y축이 아니라 X축 기준으로 처리해주세요.
