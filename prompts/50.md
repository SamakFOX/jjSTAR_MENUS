# 사용자 가이드(튜토리얼) 추가 개선 요청 수정본

## 작업 목표

현재 가이드는 이전보다 많이 개선되었지만, 아직 일부 설명 위치와 단계 순서, 문구 구성이 실제 사용 흐름과 맞지 않는 부분이 있습니다.

이번 수정에서는 아래 내용을 반영해 주세요.

```txt
1. Step 3 상단 우측 버튼 설명 문구 수정
2. Step 3에서는 "사용 방법 보기" 설명 제거
3. Step 6 아이템별 편집 버튼 설명을 아이콘 + 버튼명 + 설명 구조로 정리
4. 기존 가이드 개선 조건은 유지
5. 기존 기능은 절대 건드리지 않기
```

---

## 가장 중요한 조건

기존에 이미 잘 동작하는 기능은 절대 건드리지 말아 주세요.

```txt
로그인/인증
메뉴 편집 기능
리스트모드 드래그앤드랍
비주얼모드 드래그앤드랍
임시저장/자동저장
미리보기
최종 제출 저장
변경 로그 저장
툴팁(desc) 기능
undo/redo
초기화
관리자 화면
Supabase 저장 구조
```

이번 작업은 **가이드 문구와 가이드 흐름 일부 수정**만 대상입니다.

가이드 때문에 실제 데이터가 변경되면 안 됩니다.

```txt
가이드 표시만으로 menuState 변경 금지
가이드 표시만으로 commitMenuChange 호출 금지
가이드 표시만으로 undo/redo 증가 금지
가이드 표시만으로 change_log 추가 금지
가이드 표시만으로 임시저장/자동저장 실행 금지
```

---

# 1. 전체 튜토리얼 흐름 유지

전체 튜토리얼 흐름은 아래 구조를 유지해 주세요.

```txt
1. 첫 화면에서 "메뉴 편집하기" 안내
2. 사용자가 편집하기를 누름
3. 편집 화면 진입 직후, 먼저 편집 UI를 순서대로 설명
4. 상단 버튼 설명
5. 우측 패널 "편집 도구" 설명
6. 리스트모드 핸들/드래그 설명
7. 아이템별 편집 버튼 설명
8. 메뉴 hover 툴팁 설명
9. 비주얼모드 설명
10. 최종 제출 화면 설명
11. 완료 후 처음 로그인한 상태 화면으로 복귀
```

---

# 2. 수정된 가이드 단계

## Step 1. 첫 화면: 메뉴 편집하기 버튼 설명

### 대상

```txt
메뉴 편집하기 버튼
```

### 설명 예시

```txt
메뉴를 직접 바꾸려면 먼저 "메뉴 편집하기"를 눌러 주세요.
```

### 완료 조건

```txt
사용자가 실제로 "메뉴 편집하기" 버튼 클릭
```

---

## Step 2. 편집 화면 진입 후: 상단 기본 UI 설명

### 대상

```txt
편집 완료
리스트 편집
화면에서 편집
초기화
```

### 설명 예시

```txt
편집 화면에서는 리스트 편집과 화면에서 편집 모드를 오가며 메뉴를 조정할 수 있어요.
초기화 버튼은 현재 배치를 처음 상태로 되돌립니다.
```

이 단계는 사용자가 `메뉴 편집하기`를 누른 직후 바로 나와야 합니다.

---

## Step 3. 상단 우측 버튼 설명

### 대상

```txt
임시저장
미리보기
최종 제출
```

### 중요 수정사항

기존 Step 3에서 `사용 방법 보기`도 함께 설명하고 있었다면 제거해 주세요.

`사용 방법 보기` 버튼은 우측 편집 도구 영역에 이미 있으므로,
Step 3에서는 설명하지 않아도 됩니다.

---

### Step 3 설명 문구

아래 문구로 정리해 주세요.

```txt
임시저장: 작업 내용 임시 저장
미리보기: 현재 구성을 실제 화면처럼 확인
최종제출: 의견 추가 및 제출
```

### 설명창 표시 예시

```txt
상단 우측 버튼으로 작업을 저장하거나, 현재 메뉴 구성을 미리보고, 최종 제출 단계로 이동할 수 있어요.

임시저장: 작업 내용 임시 저장
미리보기: 현재 구성을 실제 화면처럼 확인
최종제출: 의견 추가 및 제출
```

---

## Step 4. 우측 패널 편집 도구 설명

### 대상

```txt
우측 패널 "편집 도구"
```

### 설명 대상

```txt
대분류 추가
대분류 순서변경
되돌리기
다시실행
대분류만 보기
중분류까지 보기
소분류까지 보기
사용 방법 보기
```

### 설명 예시

```txt
우측 편집 도구에서는 대분류 추가, 대분류 순서 변경, 되돌리기/다시실행, 표시 레벨 조절을 할 수 있어요.
사용 방법 보기를 누르면 이 가이드를 다시 볼 수 있습니다.
```

---

## Step 5. 리스트모드 핸들 설명

### 대상

```txt
각 메뉴 왼쪽의 점 모양 드래그 핸들
```

### 설명 예시

```txt
왼쪽 점 모양 핸들을 잡고 위아래로 드래그하면 메뉴 순서를 바꿀 수 있어요.
```

### 위치 요청

이 단계에서는 설명창을 대상 요소 **위쪽**에 배치해 주세요.

```txt
핸들 단계: 설명창은 위쪽
```

### 애니메이션 요청

드래그 예시는 아이템 전체가 아니라 **핸들 위치**에서 시작해야 합니다.

```txt
1. 핸들 영역 pulse 강조
2. 포인터 아이콘이 핸들 위로 이동
3. 포인터가 핸들을 누르는 모션
4. 핸들에서 위/아래로 살짝 드래그하는 모션
5. 실제 아이템은 이동하지 않음
```

---

## Step 6. 아이템별 편집 버튼 설명

### 대상

```txt
아이템 우측 편집 버튼 영역
```

### 중요 수정사항

Step 6 설명은 아래처럼 **아이콘 + 버튼명 + 설명** 구조로 작성해 주세요.

```txt
(아이콘) 버튼명: 설명
```

---

### Step 6 설명 문구 예시

```txt
오른쪽 아이콘 버튼으로 메뉴를 이동하거나 수정, 추가, 삭제할 수 있어요.
```

### 버튼별 설명 구조

아래 형태로 설명창 안에 표시해 주세요.

```txt
← 이전 이동: 현재 메뉴를 앞쪽으로 이동합니다.
→ 다음 이동: 현재 메뉴를 뒤쪽으로 이동합니다.
✎ 이름 수정: 메뉴 이름을 변경합니다.
＋ 하위 메뉴 추가: 현재 메뉴 아래에 새 하위 메뉴를 추가합니다.
🗑 삭제: 현재 메뉴를 삭제합니다.
```

또는 실제 사용 중인 아이콘 컴포넌트에 맞춰 아래처럼 표시해도 됩니다.

```txt
(←) 이전 이동: 현재 메뉴를 앞쪽으로 이동합니다.
(→) 다음 이동: 현재 메뉴를 뒤쪽으로 이동합니다.
(연필) 이름 수정: 메뉴 이름을 변경합니다.
(＋) 하위 메뉴 추가: 현재 메뉴 아래에 새 하위 메뉴를 추가합니다.
(휴지통) 삭제: 현재 메뉴를 삭제합니다.
```

### 추가 안내 문구

```txt
각 아이콘 위에 마우스를 올리면 기능 이름도 확인할 수 있어요.
```

---

## Step 7. 메뉴 hover 툴팁 설명

### 대상

```txt
desc가 있는 메뉴명
```

### 설명 예시

```txt
메뉴명이 헷갈릴 때는 이름 위에 마우스를 올려 설명 툴팁을 확인해 보세요.
```

### 위치 요청

이 단계에서는 설명창을 대상 요소 **위쪽**에 배치해 주세요.

```txt
툴팁 단계: 설명창은 위쪽
```

### 시각 연출 요청

```txt
1. 포인터 아이콘이 특정 메뉴명으로 이동
2. 해당 메뉴명이 hover 상태처럼 강조
3. desc 툴팁이 실제처럼 표시
4. 설명창에는 "어려운 메뉴는 이렇게 설명을 확인하면 됩니다" 안내
```

---

## Step 8. 비주얼모드 설명

### 대상

비주얼모드에서는 레벨1 상단 탭보다
**레벨2 또는 레벨3 메뉴**를 대상으로 설명해 주세요.

### 이유

```txt
레벨1 영역은 파란 배경이라 파란 하이라이트/포인터가 잘 보이지 않음
레벨2/레벨3 메뉴가 더 잘 보이고 조작 방식도 이해하기 쉬움
```

### 설명 예시

```txt
비주얼모드에서도 점 모양 핸들을 잡고 원하는 위치에 놓으면 됩니다.
대분류는 상단 탭 순서만, 중분류와 소분류는 허용된 계층 안에서 이동할 수 있어요.
```

---

## Step 9. 최종 제출 화면 설명

### 대상

```txt
최종 제출 확인 화면
```

### 설명 대상

```txt
메뉴 구성 의도 textarea
취소 버튼
제출 버튼
```

### 설명 예시

```txt
여기에 왜 이렇게 메뉴를 배치했는지 간단히 적어 주세요.
현재 메뉴 구성과 변경 로그가 함께 제출됩니다.
```

```txt
취소: 제출하지 않고 이전 화면으로 돌아갑니다.
제출: 현재 메뉴 배치를 최종 제출합니다.
```

### 주의

가이드가 실제 제출을 자동으로 실행하면 안 됩니다.

```txt
가이드가 자동으로 제출 버튼 클릭 금지
가이드가 실제 submissions 생성 금지
가이드가 실제 intention 저장 금지
```

# 3. 설명창 배치 공통 원칙

설명창은 대상 요소를 가리지 않도록 배치해 주세요.

## 기본 배치 우선순위

```txt
1순위: 대상 요소 위
2순위: 대상 요소 아래
3순위: 대상 요소 오른쪽
4순위: 대상 요소 왼쪽
```

## 예외 규칙

```txt
핸들 단계: 설명창은 위쪽
툴팁 단계: 설명창은 위쪽
아이템별 편집 버튼 단계: 설명창은 대상 우측 또는 위쪽
비주얼모드 단계: 설명창은 대상 영역을 가리지 않는 위치
최종 제출 단계: 입력창/버튼을 가리지 않는 아래쪽 또는 옆쪽
```

# 4. 가이드 레이어 / 포인터 / 애니메이션 가시성 개선

가이드 관련 요소가 서로 가리지 않도록 z-index를 정리해 주세요.

## 권장 z-index

```txt
기본 UI: z-index 0~100
dim overlay: 9000
하이라이트: 9010
포인터/손가락 애니메이션: 9020
설명창(popover): 9030
가이드 버튼/조작부: 9040
```

## 중요

```txt
포인터는 항상 하이라이트보다 위
설명창은 항상 하이라이트보다 위
설명창은 기존 UI/다른 레이어 뒤에 깔리면 안 됨
```

# 5. 가이드 버튼 문구

현재처럼 작은 화살표만 있는 버튼보다 명확한 텍스트 버튼으로 표시해 주세요.

## 버튼 형태

```txt
[이전] [다음]
```

마지막 단계:

```txt
[이전] [완료]
```

첫 단계:

```txt
[건너뛰기] [다음]
```

또는:

```txt
[이전(비활성)] [다음]
```

`다시 보지 않기`는 하단 좌측에 유지해 주세요.

# 6. 사용 방법 보기 버튼

`사용 방법 보기` 버튼은 우측 편집 도구 영역에 유지해 주세요.

다만 Step 3의 상단 우측 버튼 설명에서는 `사용 방법 보기`를 제외해 주세요.

## 요구사항

```txt
우측 편집 도구 영역에서 사용 방법 보기 버튼 유지
클릭 시 전체 튜토리얼을 처음부터 다시 시작
Step 3 상단 버튼 설명에서는 사용 방법 보기 제외
```

# 7. 튜토리얼 완료 후 복귀 화면

## 현재 문제

튜토리얼이 끝난 뒤 최종 제출 화면에 그대로 멈춰 있으면
사용자가 실제 작업을 시작하기 어렵습니다.

## 수정 요청

전체 튜토리얼이 완료되면 맨 처음 로그인했을 때의 기본 화면으로 돌아가게 해 주세요.

```txt
튜토리얼 완료
→ 가이드 종료
→ 최종 제출 화면에서 빠져나옴
→ 로그인 직후 기본 첫 화면 상태로 복귀
```

## 기대 결과

가이드 종료 후 사용자는 바로 아래 흐름으로 실제 작업을 시작할 수 있어야 합니다.

```txt
메뉴 편집하기
→ 실제 편집 시작
```

# 8. 인터랙션 기반 진행 유지

중요한 단계는 사용자가 직접 조작해야 다음 단계로 넘어가는 구조를 유지해 주세요.

## 예시

```txt
메뉴 편집하기 버튼 클릭
핸들 hover 또는 클릭
편집 아이콘 영역 hover
메뉴명 hover로 툴팁 확인
비주얼모드 탭 클릭
최종 제출 버튼 클릭으로 제출 확인 화면 진입
```

단, 최종 제출 단계에서는 실제 제출까지 강제하지 않습니다.

---

# 9. 최종 기대 결과

개선 후 기대 흐름은 아래와 같습니다.

```txt
1. 첫 화면에서 "메뉴 편집하기" 가이드 시작
2. 편집하기를 누르면 편집 UI 기본 설명부터 시작
3. 상단 우측 버튼 설명
   - 임시저장: 작업 내용 임시 저장
   - 미리보기: 현재 구성을 실제 화면처럼 확인
   - 최종제출: 의견 추가 및 제출
```
